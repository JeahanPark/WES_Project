# UI Review 스킬 설계서

> **작성일**: 2026-04-18
> **목적**: UI 비주얼 검수 전담 스킬. 스크린샷 기반으로 시각적 문제를 발견하고 리포트한다. 수정은 하지 않는다.

---

## 1. 개요

### 역할
- UI의 시각적 품질을 검수하는 전담 스킬
- MCP `u_screenshot` 도구로 스크린샷을 찍고, 체크리스트 기반으로 판정
- **리포트만 출력** — 코드/프리팹 수정은 절대 하지 않음

### 호출 방식
- `/ui-review` — 현재 씬의 UI 전체 검수
- `/ui-review CraftPopup` — 특정 UI만 검수

### 사용 시점
- 리소스 구현 후 1차 검수 (레이아웃, 정렬, 크기 중점)
- QA 단계 2차 검수 (실제 데이터 상태에서 극단값, 전환 중점)
- 다른 세션 작업물을 검수만 할 때 단독 호출

---

## 2. 체크리스트

| # | 항목 | 확인 내용 |
|---|------|-----------|
| 1 | 텍스트 오버플로우 | 텍스트가 프레임/박스 밖으로 삐져나옴, 잘림, 줄바꿈 깨짐 |
| 2 | 정렬/간격 | 요소 간 정렬 어긋남, 간격 불균일 |
| 3 | 크기/비율 | 버튼/패널 크기 이상, 이미지 비율 왜곡 |
| 4 | 겹침/비침 | UI 요소 겹침, 팝업 뒤 UI 비침(딤 처리 누락) |
| 5 | 폰트/색상 일관성 | 같은 역할인데 크기/색상이 다름 |
| 6 | 극단값 대응 | 긴 텍스트, 큰 숫자, 빈 상태에서 레이아웃 붕괴 |
| 7 | 상태별 시각 | 버튼 normal/pressed/disabled 상태 구분 |
| 8 | 전환 이상 | 팝업 열기/닫기 시 깜빡임, 잔상 |
| 9 | 해상도 대응 | 16:9 기준 레이아웃 유지 |

### 1차 vs 2차 중점

| | 1차 (리소스 구현 후) | 2차 (QA 단계) |
|---|---|---|
| 모드 | 에디터 or 플레이 | 플레이 모드 |
| 데이터 | 기본/더미 상태 | 실제 게임 데이터 |
| 중점 항목 | 1~5 (레이아웃, 정렬, 크기, 겹침, 일관성) | 6~9 (극단값, 상태, 전환, 해상도) |

## 3. 실행 흐름

```
1. 대상 파악
   ├── 인자 있음 → 해당 UI만 검수
   └── 인자 없음 → 현재 씬에서 열 수 있는 UI 전체 검수

2. 스크린샷 촬영
   ├── 에디터 모드: 씬에서 UI 확인 → u_screenshot
   └── 플레이 모드: u_play 진입 → UI 열기 → u_screenshot → 다음 UI

3. 검수
   └── 스크린샷을 체크리스트 9개 항목으로 판정

4. 리포트 출력
```

## 4. 리포트 형식

```markdown
## UI Review 결과

**대상**: CraftPopup, InventoryPopup, PlayerStatusHUD
**모드**: 플레이 모드
**검수 유형**: 2차 (QA 단계)

### ✅ 통과
- 정렬/간격: 정상
- 폰트/색상: 일관성 유지
- 해상도: 16:9 정상

### ❌ 문제 발견
1. [CraftPopup] 텍스트 오버플로우
   - 아이템명이 10자 이상일 때 프레임 밖으로 삐져나옴
   - 스크린샷: (첨부)

2. [PlayerStatusHUD] 겹침
   - HP바와 Cold바 간격 부족으로 겹침
   - 스크린샷: (첨부)

### 📋 요약
- 통과: 7/9
- 문제: 2/9
```

## 5. 제약사항

- **수정 금지**: 코드, 프리팹, Inspector 값 등 일체 수정하지 않음
- **리포트 전용**: 발견한 문제를 정리해서 보고만 함
- **MCP 도구**: `u_screenshot`, `u_play`(플레이 모드 진입/종료), `u_console`(에러 확인)만 사용
