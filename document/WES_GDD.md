## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **게임명** | 야생 생존 탈출 |
| **장르** | 탑뷰 생존 / 협동 / 탈출 |
| **플레이 인원** | 1 ~ 4인 협동 |
| **플랫폼** | PC (Steam) |
| **세션 방식** | 저장 없음 — 세션 종료 시 진행도 초기화 |
| **참고 게임** | Don't Starve Together (게임 시스템), Escape from Duckov (인벤토리 UI/UX), Warcraft3 야생 탈출 UMS |

### 핵심 콘셉트

- 야생 환경에서 제한된 자원으로 **생존**
- 플레이어들이 협력하여 맵 반대편 **목표지점까지 탈출**
- 캐릭터 선택 없음 — 모든 플레이어 동일한 기본 캐릭터
- 저장 없는 로그라이크 세션 — 매 판이 새로운 도전

## 2. 스토리

폭풍 속에서 난파된 그들은 검은 파도에 삼켜지듯 섬의 해변으로 떠밀려왔다.

손에 쥔 지도에는 단 하나의 표식 — 섬 반대편의 **안전한 마을**이 그려져 있다.

혹한과 맹수, 끝없는 밤을 견디며 그들은 그 표식을 향해 나아가야 한다.

주위를 둘러보니 모래 위에는 썩어가는 어망과 부서진 부두 잔해만이 남아 있고, 절벽 위 등대에는 아무 불빛도 보이지 않는다.

## 3. 씬 구성

### 3.1 Intro Scene

- 게임 실행 시 최초 진입 화면
- 구성 오브젝트: `IntroController`, `Canvas`, `Main Camera`, `UICamera`
- UI 요소:
  - 게임 로고
  - START 버튼
  - OPTIONS 버튼
- 저장 / 불러오기 UI 없음

### 3.2 Lobby Scene

- 구성 오브젝트: `LobbyController`, `Canvas`, `Main Camera`, `UICamera`
- 최대 4인 플레이어 슬롯
- 방 생성 / 방 참가
- Ready / Start 버튼
- 방장만 Start 가능

### 3.3 InGame Scene

- 실제 게임 플레이 씬
- 맵 전환 없음 — `GameState`로 흐름 제어
- 라운드 종료(탈출 성공 / 전멸) 후 로비로 복귀

**씬 내 오브젝트 목록**

| 오브젝트 | 역할 |
|----------|------|
| `InGameController` | 씬 전체 흐름 관리, 워커 조율 |
| `InGamePlayWorker` | 플레이어 스폰, 아이템 드롭 |
| `InGameAreaWorker` | 몬스터 스폰 영역 관리 |
| `InGameColliderWorker` | 충돌 판정 및 데미지 처리 |
| `InGameHUDWorker` | HUD(HP, Cold 바) 관리 |
| `InGameObjectDataWorker` | 캐릭터/인벤토리 레지스트리 |
| `InGameSpawnWorker` | 네트워크 오브젝트 스폰/풀링 |
| `InGameWorldUIWorker` | 월드 체력바 UI 관리 |
| `AreaTest1` | 몬스터 스폰 영역 (테스트) |
| `StartPosition1~4` | 플레이어 초기 스폰 위치 |
| `Plane` | 기본 맵 지형 |
| `CinemachineCamera` | 플레이어 추적 카메라 |

## 5. 스탯 시스템

### 스탯 목록

| 스탯 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| **HP** | float | 100 | 체력. 0 이하 시 사망 |
| **HPRegen** | float | 0.5 / sec | 초당 HP 회복량. 최대 HP 초과 불가 |
| **ATK** | int | 10 | 공격력. 공격 시 기준 피해량 |
| **DEF** | int | 3 | 방어력. 받는 피해 감소 |
| **MoveSpeed** | float | 5.0 | 이동 속도 |
| **Cold** | float | 100 | 체온. 0 이하 시 추가 피해 (기획 예정) |

### 피해 공식

```
최종 피해 = ATK - DEF
최소 피해 = 1 (방어력이 공격력 이상이어도 최소 1 피해)
```

### HP

- 범위: `0 ~ 100`
- 자연 회복 없음 — `HPRegen`으로만 회복
- `HP <= 0` 시 사망

### HPRegen

- 초당 HP 회복
- 전투 중에도 적용
- 최대 HP 초과 불가

### Cold (체온)

- 현재: 제작 조건 체크용으로만 사용 (`CraftConditionType.MaxCold / MinCold`)
- 추후 기획 예정: 체온 감소 시스템, 동사 피해 등

## 6. 게임 흐름

```
게임 시작
    ↓
Intro Scene (START 버튼)
    ↓
Lobby Scene (방 생성 / 참가 → 방장 START)
    ↓
InGame Scene
    ↓
플레이어 스폰 (StartPosition1~4)
    │
    ├─ 생존: 이동 / 전투 / 자원 수집 / 제작 / 건물 배치
    │
    ├─ [탈출 성공] 목표지점 도달 → 클리어 화면 → 로비 복귀
    │
    └─ [전멸] 모든 플레이어 사망 → 게임 오버 화면 → 로비 복귀
```

## 7. 플레이어 시스템

### 스폰

- 씬 시작 시 `InGamePlayWorker`가 `StartPosition1~4` 위치에 플레이어 스폰
- 플레이어는 모두 동일한 `Player.prefab` 사용

### 카메라

- `CinemachineCamera`로 플레이어 추적
- 마우스 휠 줌: 거리 5 ~ 30

### 아이템 수집

- **클릭 수집**: Raycast로 클릭한 `WorldDropItem` 수집
- **자동 수집**: 반경 2m 내 아이템 자동 수집
- 수집 시 인벤토리에 추가, 네트워크 동기화

---

## 8. 전투 시스템

### 기본 공격

- `마우스 좌클릭` 시 공격 애니메이션 재생
- `InGameColliderWorker`에서 구체 충돌 판정
- 공격 반경: **1.5m**
- 동시 타격 최대: **3 타겟**

### 피해 처리

```
1. 공격 발생 → InGameColliderWorker.CreateCollider()
2. 반경 내 적 탐색
3. TakeDamage(ATK) 호출
4. 최종 피해 = ATK - DEF (최소 1)
5. HP 감소 → NetworkVariable 동기화
6. HP <= 0 → 사망 처리
```

### 사망

- 플레이어 사망: 리스폰 또는 게임 오버 (미정)
- 몬스터 사망: 드롭 아이템 스폰 → 리스폰 타이머 시작

## 9. 인벤토리 시스템

### 구조

- `InventoryRegistry`: 플레이어별 인벤토리 데이터 관리
- `ItemData`: 아이템 인스턴스 (ItemInfo + 현재 개수)

### 스택 규칙

| 조건 | 동작 |
|------|------|
| `IsStackable = true` | MaxStack까지 스택 |
| `IsStackable = false` | 슬롯 1개에 1개만 |

### UI — 그리드 인벤토리 (Escape from Duckov 스타일)

- `I` 키로 `InventoryPopup` 열기/닫기
- **그리드(칸) 기반** 슬롯 배치 (스크롤 리스트 → 그리드로 변경)
- 슬롯 수 **확장 가능** (초기 N칸, 업그레이드로 증가)
- 드래그 앤 드롭으로 아이템 이동:
  - 인벤토리 슬롯 간 이동 (순서 변경)
  - 인벤토리 → 퀵슬롯 등록
  - 퀵슬롯 간 순서 변경
  - 퀵슬롯에서 빼기
- 아이템 선택 시 상세 정보 패널 표시

### 퀵슬롯

- `1 ~ 8` 단축키로 해당 슬롯 아이템 즉시 사용
- HUD 하단에 8칸 슬롯 표시
- **모든 아이템 타입 등록 가능** (소비/장비/건물)
- 건물 아이템 사용 시 배치 모드 진입

### 제작 조건 (CraftConditionInfo)

| 조건 타입 | 설명 |
|-----------|------|
| `None` | 조건 없음 |
| `MaxCold` | 체온이 특정값 이하일 때만 제작 가능 |
| `MinCold` | 체온이 특정값 이상일 때만 제작 가능 |

### 제작 흐름

```
1. CraftPopup 열기 (HUD CraftHUDTab)
2. 카테고리 탭 선택 (건물 / 아이템)
3. 레시피 선택 → CraftDetailPanel에 재료 표시
4. 재료 보유 여부 확인
5. 조건 확인 (Cold 등)
6. 제작 버튼 클릭
7. 인벤토리에서 재료 차감
8. 결과 아이템 인벤토리에 추가
```

> **현재 상태**: 구현 완료. 재료 차감 + 결과 아이템 인벤토리 추가 동작.

## 12. 몬스터 시스템

### 현재 몬스터 (3종)

| ID | 이름 | HP | 드롭 | 위치 |
|----|------|-----|------|------|
| 1 | Test01Monster | 100 | 나무 1~3, 돌 1 | 해안가 |
| 2 | 늑대 | 80 | 가죽 1~2, 뼈 0~1 | 숲 |
| 3 | 골렘 | 250 | 돌 3~5, 철광석 1~3, 뼈 1~2 | 산지 |

### AI 상태머신

```
Idle ──(배회 시작)──→ Walk
Walk ──(피격)──→ Hit ──(회복)──→ Idle
Hit  ──(HP <= 0)──→ Death
```

| 상태 | 설명 |
|------|------|
| `Idle` | 제자리 대기 |
| `Walk` | 배회 이동 |
| `Hit` | 피격 모션 |
| `Death` | 사망 → 드롭 → 리스폰 타이머 |

### 스폰 시스템

- `InGameAreaWorker`가 `MonsterSpawnArea[]` 관리
- `WorldAreaInfo`의 `MaxCount`만큼 스폰 유지
- 사망 후 `RespawnDelay` 경과 시 재스폰

## 13. 탈출 시스템

### 탈출 조건

- 목표지점(`EscapePoint`) 오브젝트에 **플레이어가 도달**하면 탈출 성공
- 현재: 개인 탈출 방식 (먼저 도달한 플레이어부터 탈출)
- 추후 엔딩 다양화 예정

### 탈출 흐름

```
1. 맵 특정 위치에 EscapePoint 배치
2. 플레이어가 EscapePoint 반경 진입 감지
3. 탈출 처리 (해당 플레이어 씬에서 제거 또는 대기 상태)
4. 전원 탈출 또는 탈출 인원 기록
5. 클리어 화면 표시 → 로비 복귀
```

> **현재 상태**: 구현 완료.

## 14. 네트워크 구조

| 항목 | 내용 |
|------|------|
| **라이브러리** | Unity Netcode for GameObjects (NGO) |
| **연결 방식** | Steam Relay 서버 |
| **서버 구조** | 방장 = Host = Server |
| **권한** | 방장(서버)이 모든 게임 로직의 권한(authority) 보유 |
| **클라이언트** | 입력 요청만 서버에 RPC로 전달 |

### 동기화 항목

| 데이터 | 방식 |
|--------|------|
| HP | `NetworkVariable<float>` |
| Cold | `NetworkVariable<float>` |
| 아이템 수집 | `ServerRpc` |
| 데미지 | `ServerRpc` |
| 몬스터 스폰/사망 | 서버 권한 |

## 15. 데이터 구조 (CSV)

모든 게임 데이터는 `Assets/CSVInfo/` 의 CSV 파일로 관리.
`InfoLoader.cs`가 Reflection으로 파싱 → `InfoManager`의 List에 저장.

| CSV 파일 | Info 클래스 | 용도 |
|---------|-------------|------|
| `ItemInfo.csv` | `ItemInfo` | 아이템 정의 |
| `CraftInfo.csv` | `CraftInfo` | 제작 레시피 |
| `CraftMaterialInfo.csv` | `CraftMaterialInfo` | 제작 재료 |
| `CraftConditionInfo.csv` | `CraftConditionInfo` | 제작 조건 |
| `BuildingInfo.csv` | `BuildingInfo` | 건물 정의 |
| `MonsterInfo.csv` | `MonsterInfo` | 몬스터 정의 |
| `DropSourceInfo.csv` | `DropSourceInfo` | 드롭 소스 연결 |
| `DropTableItemInfo.csv` | `DropTableItemInfo` | 드롭 아이템 확률/수량 |
| `WorldAreaInfo.csv` | `WorldAreaInfo` | 스폰 영역 설정 |
| `WorldAreaMonsterInfo.csv` | `WorldAreaMonsterInfo` | 영역별 몬스터 |
| `WorldObjectInfo.csv` | `WorldObjectInfo` | 환경 오브젝트 |

### 아이템 ID 체계

| 범위 | 카테고리 | 예시 |
