# 이동 비용 — 기획서

> **버전**: v0.1 (director)
> **작성**: 2026-06-08
> **분기**: ④ 이동 ("진행을 산다")
> **상위**: [CORE §4.4](../../CORE_자원투자_트레이드오프_설계.md) · [출시_9시간_관통구조](../출시_9시간_관통구조/기획.md)
> **선행(구현 완료)**: 시간/낮밤(`DayNightWorker`), Cold 실질화(`ColdDamageWorker`)
> **성격**: director — 이동 비용의 *구조*·트레이드오프까지. 소모 계수·지역 배수는 level-design, 코드는 client.

---

## 1. 한줄 컨셉

**이동에 시간·체온 비용을 붙여, "지금 진행할까 / 머물러 준비할까"를 매 순간의 결정으로 만든다.**

> CORE 4분기 중 마지막 구멍. 현재 비용 없는 자유 이동 → 이동이 *자원을 지불하는 행위*가 되어야 ④분기가 점화된다.

## 2. 디자인 골 (Why)

- **④ 이동 축 점화.** "위치를 바꾸면 새 자원·새 위험·탈출 진척을 얻지만, *대가가 있다*."
- **9시간 종단의 추진력.** 마을로 가려면 이동해야 하고, 이동엔 비용이 드니, "더 갈까 / 준비를 더 할까"가 9시간 내내 반복.
- **신규 스탯 없이 기존 시스템 활용**(CORE 4.4): 시간(낮밤)·체온(Cold)을 이동 비용의 통화로 재사용. 피로/스태미나 신규 도입 회피.
- 설원의 "이동비용 극대"(출시_9시간 §4)가 이 시스템의 절정.

## 3. 안티 골 (What this is NOT)

- ❌ **신규 스탯(피로·스태미나) 도입 금지.** 기존 시간·Cold로만(CORE 4.4).
- ❌ **이동을 단순 페널티로만 만들기 금지.** 반드시 *결정*("이 체온을 이동에 쓸까")으로 환원.
- ❌ **이동 봉쇄 금지.** 비용이 크되, 이동 자체가 *불가능*해선 안 됨(준비하면 갈 수 있다).
- ❌ 자동 길찾기/순간이동 등 비용 우회 수단.

## 4. 플레이어 경험 흐름

```
목표 방향으로 이동 필요 인지
   ↓
비용 점검: "지금 출발하면 체온이 버틸까? 밤 오기 전 도착할까?"
   ↓
결정: 지금 진행(이동 비용 지불) vs 머물러 준비(모닥불·체온 비축 후)
   ↓
이동 중: 체온 가속 소모 + 시간 경과(밤·날씨 진행) + 새 지역 위험 노출
   ↓
적중: 무사 도착 / 빗나감: 길 위에서 동사·야간 피습 → "준비를 더 할걸"(후회)
```

## 5. 핵심 디자인 결정

### 5.1 이동 비용 = 체온 가속 + 시간·위험 노출

| 비용 통화 | 작동 |
|---|---|
| **체온(Cold) 가속 소모** | 이동 중 Cold가 평소보다 빨리 감소. 정지 시 회복/유지. 지역별 배수(설원 최대) |
| **시간 경과** | 이동에 걸리는 실시간 동안 낮밤·날씨가 진행 → "밤 오기 전 도착" 압박(0순위와 직결) |
| **위험 노출** | 이동 = 새 지역/구간 진입 = 새 위협. 머물면 안전하나 진척 0 |

> 핵심은 **체온 가속**. 이동이 Cold를 "지불 통화"로 소모해, 모닥불·체온포션·방한복(③안전/자원)과 직접 경쟁한다.

### 5.2 지역별 이동 비용 배수

- 깊은 지역일수록 이동 체온 소모↑. **설원 = 극대**(방한 테크 없이 이동 시 빠른 동사).
- 날씨 연동: 눈보라·비 중 이동은 체온 소모 추가 가속(날씨 `weatherMul`과 결합).
- 시간대 연동: 야간 이동은 체온·위험 모두 가중.

### 5.3 비용 완화 = 다른 분기 투자

- **방한복**(④/자원): 설원 이동 체온 소모 감소 → 모피(설원 드롭) 선투자.
- **모닥불 동선**(③안전): 이동 경로에 모닥불을 놓아 체온 보급소 확보 → 나무 소모.
- → 이동 비용은 *다른 3분기에 미리 투자*함으로써 줄일 수 있다. 이게 4분기 상호작용.

## 6. 콘텐츠 분량 (수치는 level-design)

- 신규 콘텐츠 거의 없음(기존 Cold·시간 재사용). 추가는 **이동 상태에서의 Cold 소모 배수** 파라미터 + 지역별 배수 테이블.
- 방한복·이동용 소비재는 아이템/제작에서 이미 커버.

## 7. 톤 가이드

- 이동은 *결단*이어야 한다 — 따뜻한 모닥불을 떠나 추운 어둠으로 발을 내딛는 무게.
- "한 발 더 갈 때마다 체온이 깎인다"는 체감이 야생의 냉혹함(톤)과 일치.

## 8. 밸런스 / 튜닝 포인트

- **이동 중 Cold 소모 배수**: 정지 대비 ×N. 너무 크면 이동 공포증, 작으면 비용 무의미.
- **지역별 배수**: 해변 ~1.0 → 설원 최대(예 ×2.5~3).
- **날씨·야간 가중**: 눈보라 이동 = 추가 가속.
- 난이도 공식 `EnvPressure`의 `coldDrain`이 *이동 상태에서 가속*되도록 반영.
- 거짓 선택 방지: "항상 머무는 게 정답"이 되지 않게 — 머물면 자원 고갈·밤 심화로 진행 압박도 상승.

## 9. 엣지 케이스 (게임플레이 차원)

- **이동 중 Cold 0 도달**: 동사 피해(밤_추위 시스템) → 이동 중단·복귀 결정 강제.
- **협동 이동**: 빠른/느린 플레이어 분산 시 체온 관리 분업.
- **되돌아가기**: 앞 지역 회귀도 동일 비용 — 무의미한 왕복 억제.
- **모닥불 의존**: 경로상 모닥불 배치로 비용 회피 가능하나, 나무 소모로 다른 분기 압박.

## 10. 향후 확장 (Backlog)

- 지형 경사에 따른 이동 비용(오르막 = 체온/시간 추가) — 지형경사 시스템과 결합.
- 탈것/지름길(고비용 우회) — 단 안티골(비용 우회) 주의.
