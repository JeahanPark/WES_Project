# R3 콘텐츠 채우기 — 스코프 기획서

> **버전**: v0.1 (director — R3 콘텐츠 스코프)
> **작성**: 2026-06-21
> **상위**: [출시_9시간_관통구조](../출시_9시간_관통구조/기획.md) · [CORE_자원투자_트레이드오프_설계.md](../../CORE_자원투자_트레이드오프_설계.md)
> **연관(중복 금지·참조만)**: [몬스터](../몬스터/기획.md) · [아이템](../아이템/기획.md) · [제작레시피](../제작레시피/기획.md) · [도면해금](../도면해금/기획.md) · [level-design 수치명세](../../level-design/출시_지역별_수치명세/수치명세.md)
> **성격**: R1(시스템 토대)·R2(6지역 공간) 위에 **실제 콘텐츠를 채우는 슬라이스 스코프**. 신규 시스템 설계가 아니라, 기존 설계 문서들을 *구현 가능한 단위로 묶고 우선순위를 매기는* 결정 문서.
> **핵심 제약**: 몬스터/동물 메쉬 없음 → **비주얼은 백로그, 기능(행동·수치·레시피)은 R3**. placeholder 구동으로 게임플레이를 먼저 완성한다.

---

## 1. 한줄 컨셉

**메쉬·아이콘 없이도 굴러가는 "코어 루프 콘텐츠"(도구 등급·몬스터 행동·자원/제작/도면·6지역 수치)를 R3에서 먼저 박고, 비주얼은 placeholder로 대체해 9시간 트레이드오프 루프를 *기능적으로 완주 가능하게* 만든다.**

> R3의 성공 기준 = "메쉬가 다 박힌 게임"이 아니라 "**해변→폐허까지 placeholder로 끝까지 플레이되며 4분기 결정이 실제로 작동**하는 게임".

## 2. 디자인 골 (Why)

- **코어 루프부터 가동**: 4분기(효율/테크/안전/이동) 중 R1에서 시스템만 깔린 ①효율(도구 등급)을 **실아이템으로 점화**해, 자원→투자처 매핑([아이템 §2])이 처음으로 *플레이 가능*하게 한다.
- **불완전 정보의 실체화**: 몬스터 "행동 패턴"(무리·돌진·독·은신·날씨강화)이 CORE의 불완전 정보 척추. 메쉬가 없어도 **행동(FSM/파라미터)** 만으로 이 척추를 세운다.
- **6지역 종단 완주**: R2의 빈 공간에 [수치명세]의 값을 흘려넣어 해변→폐허가 끊김 없이 이어지는 진행 게이트를 완성.
- **비주얼-기능 분리로 리스크 격리**: 외부 메쉬 확보(designer 백로그)가 늦어도 게임플레이 검증·QA·밸런스는 R3에서 진행 가능.

## 3. 안티 골 (What this is NOT)

- ❌ **신규 시스템 설계.** R3은 R1/R2 시스템과 기존 설계 문서를 *채우는* 단계. 새 4분기 축·새 코어 메커닉 추가 금지.
- ❌ **메쉬·정밀 아이콘 대기.** 비주얼 자산을 기다리며 멈추지 않는다. placeholder(캡슐/색구분/임시 아이콘)로 기능 먼저.
- ❌ **콘텐츠 양 인플레.** 자원 ~9종·레시피 ~25~32·도면 5~10 상한 엄수(CORE 안티골·출시_9시간 §6). R3에서 상한을 넘기지 않는다.
- ❌ **수치 직접 확정.** HP/ATK/드롭률/밀도 *실수치*는 [수치명세](../../level-design/출시_지역별_수치명세/수치명세.md) 소관. R3 director는 *어떤 콘텐츠를·어느 분량·어느 순서로* 만들지의 스코프만 결정.
- ❌ **6지역 전부를 한 번에 완성.** 슬라이스로 끊어 d0~d1 먼저 검증 후 확장(아래 §5 우선순위).

## 4. 플레이어 경험 흐름 (R3 완료 시점 기준)

```
세션 시작(placeholder 비주얼)
  ├─ 해변: 맨손/석기로 채집 학습 — "도구 업글하면 더 빨리 모은다"는 갈증 발생
  ├─ 숲: 늑대 무리(placeholder 캡슐 떼) 추격 — "한 마리 보이면 더 있다"의 행동 위협 체감
  │     └─ 첫 결정: 모은 철광석을 [철 도구]에 쓸까(효율) [철검]에 쓸까(전투)
  ├─ 늪지~산지: 독·은신·돌진·공중 행동으로 위협 갱신 + 도면 주워 제작 옵션 확장
  ├─ 설원: 모피 드롭으로 방한복 게이트 — 행동(체온부여) + 자원게이트 동시
  └─ 폐허: 다수 소모전 + 보스 페이즈 → 탈출 최종 베팅
```

핵심 감정: **비주얼이 임시여도 "결정의 무게"는 진짜다.** R3은 그 무게가 작동하는지를 placeholder로 먼저 증명한다.

## 5. 핵심 디자인 결정 — R3 스코프 4축 + 슬라이스 순서

### 5.1 R3 콘텐츠 4축 (달성 가능 우선순위 = 메쉬 의존 최소순)

| 축 | 내용 | 메쉬/아이콘 의존 | R3 분량 |
|---|---|---|---|
| **① 도구 등급 실아이템** | 나무/돌 도구 → 철 도구 → 고철 도구. R1 도구등급 시스템을 실제 아이템·레시피로 활성화 | **낮음**(아이콘만, placeholder 가능) | 도구 **3등급 ×주요 종류**(예: 도끼·곡괭이 라인 → 4~6 아이템) |
| **② 몬스터 행동 패턴** | 무리/돌진/독/원거리(공중)/은신/날씨강화/보스페이즈 — FSM·파라미터로 | **없음**(캡슐 placeholder로 행동 검증 가능) | 행동 패턴 **6종 + 보스 페이즈 1** |
| **③ 자원·제작·도면 확장** | 자원 ~9종 / 레시피 ~25~32 / 도면 5~10장 (기존 8 레시피·3 도면 → 확대) | **중간**(결과물 아이콘, placeholder 가능) | 자원 +3 / 레시피 +17~24 / 도면 +2~7 |
| **④ 6지역 몬스터·자원 수치** | [수치명세] 값을 CSV로 흘려넣어 6지역 게이트 완성 | **없음**(순수 데이터) | 6지역 전 d0~d5 |

> **읽는 법**: 위→아래로 갈수록 *비주얼 의존이 늘거나 작업량이 큼*. ②④는 메쉬 0 의존 → **최우선 착수**. ①은 아이콘만 → 다음. ③은 결과물 아이콘 다수 → placeholder로 기능 먼저, 아이콘은 백로그.

### 5.2 슬라이스 순서 (client3 비용 반영 — quick win 먼저)

> **client3 비용 정정 반영**: 도구 실아이템 = R1-T3 채집배수 이미 연결 → **코드 0, 데이터만(S) = quick win**. 몬스터 행동 = 현 FSM에 추격/공격 상태 자체가 없음(평화 배회만) → 신규 L~XL, client 권고로 **Chase+Attack 2상태 + BehaviorType enum 분기**로 L→M 축소, DetectRange=0 디폴트로 비파괴. 도면 월드스폰 = 획득→해금은 연결, 자연 스폰만 미연결 = M.

| 슬라이스 | 범위 | client3 비용 | 검증 목표 | 비주얼 |
|---|---|---|---|---|
| **R3-A (quick win)** | **도구 등급 실아이템(①)** + 해변·숲(d0~d1) 수치(④, 최소 컬럼) | **S(데이터만)** | "효율 vs 전투" 첫 결정 즉시 작동 | placeholder |
| **R3-B (행동 코어)** | 몬스터 추격/공격 도입(Chase+Attack 2상태+BehaviorType) — 무리·돌진(②) + 숲·늪 위협 활성 | **M(L→M 축소)** | 평화 배회 → 실제 위협 전환, 무리/돌진 체감 | placeholder |
| **R3-C (확장)** | 도면 월드스폰(M) + 독·은신·공중·날씨강화·보스(②) + 산~폐허(d3~d5) + 레시피/방한복/최종장비(③) | M+ | 탐험 보상 루프 + 게이트 + 보스 + 최종 베팅 | placeholder |

> **v0.1→v0.2 변경**: 도구(S)를 단독 R3-A로 분리(즉시 가치), 행동패턴(M)을 R3-B로 독립. client3의 "도구=quick win, 행동=신규비용" 정정을 반영해 **가치/비용비 높은 것부터**. 각 슬라이스 끝 QA 통과 후 다음.

### 5.2.1 director 결정 (client3 질문 4건 응답)

| 질문 | director 결정 | 근거(게임의도) |
|---|---|---|
| **행동패턴 종수·대상지역** | R3-B: **무리·돌진 2종**(숲·늪). R3-C: 독·은신·공중·날씨강화·보스. **총 6+보스 유지하되 슬라이스 분산**. client 권고대로 Chase+Attack 2상태에 BehaviorType enum으로 6행동 분기 | 무리·돌진이 "첫 위협" 학습에 가장 적합(출시_9시간 §4 숲). 나머지는 후반 지역 도입에 묶임 |
| **추격 leash 정책** | **leash 있음** — 스폰 영역 + DetectRange 이탈 시 복귀(배회로). leash 거리 ≈ 영역 반경 ×1.5 | 무한 추격은 야생 톤 깨고 도주 전술(④이동 분기) 무력화. "지역 벗어나면 따돌린다"가 이동 베팅의 일부 |
| **도면 중복정책** | **중복 무효**(이미 해금된 종류면 효과 없음/줍히지 않음) — [도면해금 §9] 계승 | 도면=해금 트리거지 비축 자원 아님(인플레 안티골). 중복은 게임적 무해 처리 |
| **평화몬스터 반격 여부** | **반격함**(공격받으면 Chase 전환). 단 DetectRange=0이라 먼저 안 덤빔 | 갯게·갈매기도 때리면 반격해야 "온보딩이지만 살아있는 세계". 비파괴 디폴트(DetectRange=0)와 양립 — client 권고 구조 그대로 수용 |

### 5.3 비주얼 = 백로그, 기능 = R3 (명시)

| 콘텐츠 | R3에서 하는 것(기능) | 백로그로 미루는 것(비주얼) |
|---|---|---|
| 몬스터 6종+보스 | 행동 FSM·HP/ATK·드롭·스폰 | 메쉬·애니메이션 (designer: Synty/polyperfect 외부자산) |
| 도구 3등급 | 등급별 채집 배수·레시피·소비 | 정밀 도구 아이콘 (임시 아이콘으로 구동) |
| 자원 +3·결과물 | 아이템 정의·드롭·제작 | 자원/결과물 아이콘·프리팹 |
| 6지역 | 수치·게이트·스폰 데이터 | 지역 환경 디테일(R2 공간 위) |

> placeholder 원칙: **색·형태로 구분만 되면 OK**(늑대=회색 캡슐 떼, 멧돼지=갈색 큰 캡슐, 독개구리=녹색 등). 행동이 다르면 비주얼 같아도 플레이 구분 가능 — 이게 "행동으로 위협" 설계의 부수 효과.

### 5.4 R4/R5로 미루는 것 (R3 비포함)

| 미룸 대상 | 이유 | 행선지 |
|---|---|---|
| 전 몬스터 메쉬·애니메이션 | 외부자산 확보·리깅 시간 | R4(designer 백로그) |
| 정밀 아이콘·프리팹 세트 | 기능 검증 후 일괄 | R4 |
| 표지(협동 정보공유) 기술 | ②안전 보조 — 코어 아님, 후순위 | R4/R5 |
| 도면 희귀도 등급 | 변주 강화용 — 출시 후 | R5(도면해금 §10) |
| 엔딩 분기(탈출 인원/방식) | 콘텐츠 완주 후 | R5(출시_9시간 §10) |
| 시드 변주·리플레이 강화 | 코어 안정화 후 | R5 |
| **도구 종류별 채집배수 분리**(도끼=벌목/곡괭이=채광) | ①효율 *내부*의 미시 최적화 — 코어 4분기 트레이드오프를 강화하지 않고 관리 부담만 늘림 | R5(도구종류확장) |

> **R3-A 도구 결정 (2026-06-21)**: R3은 **등급만**(ToolTier 1/2/3, 보유 최대 등급 합산) 활성. 도끼/곡괭이는 동일 채집배수를 공유 — 종류별 분리 X. 이유: 종류 분리는 ①효율을 두 갈래로 쪼개 결정을 미시화할 뿐, "같은 자원을 4분기 중 어디에"라는 코어 충돌을 강화하지 못함. 등급 자체(철광석을 도구 vs 철검)가 이미 핵심 트레이드오프이므로 R3-A는 거기에 집중.

## 6. 콘텐츠 분량 (수치 포함 — 상한 엄수)

| 콘텐츠 | 현재 | R3 목표 | 상한(근거) |
|---|---|---|---|
| **도구 등급 아이템** | 0 | **3등급 × 도끼·곡괭이 = 4~6종** | ①효율 점화. 종류 과확장 금지(채집행위 수만큼) |
| **몬스터 행동 패턴** | 공통 FSM | **6종 + 보스 페이즈** | 무리·돌진·독·공중·은신·날씨강화 ([몬스터 §3]) |
| **몬스터 개체** | 3종 | **지역별 2종 = ~12 + 보스** | [몬스터 §2] 상한 |
| **원재료 자원** | ~8 | **~9** | [아이템 §3] 상한(인플레 금지) |
| **제작 레시피** | 8 | **~25~32** | [제작레시피 §3] |
| **도면(해금 트리거)** | 3 | **5~10장** | [도면해금 §6] |
| **6지역 수치 적용** | d0 일부 | **d0~d5 전 적용** | [수치명세] 전 표 |

> 분량은 *전부 신규 창작*이 아니라 **기존 설계 문서의 카탈로그를 구현으로 옮기는 것**. R3에서 상한을 새로 늘리지 않는다.

## 7. 톤 가이드

- placeholder여도 **톤은 유지**: 캡슐 색은 어둡고 칙칙한 야생 팔레트(회색/갈색/탁녹/창백), 화사한 원색 금지.
- 도구 등급 업글은 "성장 팡파레"가 아니라 **건조한 효율 향상**(도면해금 §7 톤 계승). "철 도끼를 만들었다" 한 줄.
- 몬스터 행동이 톤을 만든다: 무리의 포위·은신의 불안·돌진의 공포 — 비주얼이 임시여도 *행동의 정서*는 Don't Starve의 외롭고 위험한 야생.

## 8. 밸런스 / 튜닝 포인트 (director는 의도, 수치는 level-design)

- **도구 등급 회수 곡선**: 도구는 *장기* 보상. 위급 시 안전(덫)이 급한데 도구에 베팅 → 빗나갈 수 있음([제작레시피 §5]). 효율-안전 충돌이 살아있어야.
- **행동 위협 ≠ HP 인플레**: EnemyScale은 HP/ATK 배수지만, 체감 위협의 주는 **행동**이어야([몬스터 §4]). 같은 HP라도 무리·은신이 더 무섭게.
- **게이트 자원 보장**: 설원 모피 드롭 70%+ (진행 막힘 0) — [수치명세 §9.2].
- **4분기 거짓선택 방지**: 도구 등급(①)이 항상 정답/항상 낭비가 안 되게 — placeholder 단계 QA에서 경로별 클리어 시도([수치명세 §9.3]).
