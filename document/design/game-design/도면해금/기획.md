## 1. 한줄 컨셉

> **제작 레시피를 "세션 시작부터 전부 보유"에서 "기본 5종 + 월드에서 도면을 주워 여는 장비 3종"으로 바꿔, 탐험이 곧 제작 옵션의 확장이 되게 한다.**

---

## 2. 디자인 골 (Why)

| 골 | 설명 |
|---|---|
| **② 테크 축 점화** | 4분기 중 유일하게 "전무"였던 테크 해금을 최소 형태로 실현. "투자(탐험 시간)해서 새 옵션을 연다"는 결정이 처음으로 발생 |
| **탐험-보상 사이클 강화** | 월드를 도는 행위에 "도면을 찾을지도"라는 능동적 동기 부여. 단순 자원 수집을 넘어선 탐험 이유 |
| **시간 압박과의 결합** | 밤이 오기 전 도면을 찾으러 더 멀리 갈지 vs 안전하게 돌아올지 — 0순위(낮밤)가 만든 시간 긴장을 ② 결정으로 흘려보냄 |
| **세션 변주 = 리플레이성** | 매 세션 도면 분포가 달라져 "이번 판엔 철검을 일찍 열었다 / 갑옷을 못 찾았다"는 서사 차이 발생 |
| **협동 역할 분담** | 4인 협동에서 "도면 탐색 담당"이라는 분업 역할이 생김 (코어 비전 시나리오 C) |

---

## 3. 안티 골 (What this is NOT)

- ❌ **테크 트리 아님.** 전제-노드 의존 구조 금지. 평면 카탈로그(5~8장 수준). 도면 A를 열어야 B를 여는 식 없음.
- ❌ **영구 진행 아님.** 해금은 **세션 내 한정**. 세션 종료 시 모두 초기화. 다음 판은 다시 기본 5종부터. (코어 비전 안티골 §10 준수)
- ❌ **연구/제작으로 도면 만들기 아님.** 도면은 오직 월드에서 **줍는다**. 도면을 제작하는 레시피는 없음.
- ❌ **자원 종류 인플레이션 아님.** "도면"은 사용 즉시 사라지는 해금 트리거지, 비축·거래되는 일반 자원이 아님.
- ❌ **신규 제작 콘텐츠 추가 아님.** 이번 슬라이스는 **기존 8종의 재배치**일 뿐. 새 레시피·새 아이템을 만들지 않는다.
- ❌ **밸런스 전면 개편 아님.** 기존 레시피의 재료/결과 수치는 건드리지 않는다.

## 4. 플레이어 경험 흐름

```
세션 시작
  └─ 제작창 열면: 기본 5종 제작 가능 / 장비 3종은 "잠김(도면 필요)" 상태로 보임
       │
       ├─ 낮: 자원 채집하며 월드 탐험
       │     └─ 월드 곳곳에 도면이 드롭/배치되어 있음 (탐험 보상)
       │
       ├─ 도면 발견 → 줍기 → "○○ 도면 해금!" 알림
       │     └─ 제작창에서 해당 장비가 잠금 해제, 즉시 제작 가능
       │
       ├─ 시간 압박 결정: "철검 도면은 산지 쪽에 있을 것 같다.
       │     밤 오기 전 다녀올까, 아니면 지금 가진 걸로 버틸까?"
       │
       └─ 세션 종료: 모든 해금 초기화. 다음 판은 다시 기본 5종부터.
```

핵심 감정 곡선: **"제작창의 잠긴 칸"이 갈증을 만들고 → 탐험이 그 갈증을 푸는 행위가 되며 → 밤이라는 마감이 탐험을 도박으로 만든다.**

## 5. 핵심 디자인 결정

### 5.1 기본 / 도면 해금 레시피 지정 (실측 CraftInfo.csv 기준)

현재 8종을 **생존 기초 5종(상시) + 장비 3종(도면)**으로 가른다. 기준: "생존에 필수인가(기초)" vs "전투 우위를 사는 선택재인가(장비)".

| 그룹 | ID | 이름 | 분류 근거 |
|---|---|---|---|
| **기본 5종 (상시 보유)** | 1 | 모닥불 | 체온 생존 필수 — Cold 시스템과 직결, 없으면 밤 생존 불가 |
| | 2 | 회복 포션 | 기초 HP 회복 — 생존 기반 |
| | 3 | 체온 포션 | 기초 체온 회복 — Cold 대응 기반 |
| | 4 | 붕대 | 저비용 HP 회복 — 초반 생존 기반 |
| | 8 | 횃불 | 기초 조명 — 밤 탐험 기반 |
| **도면 해금 3종** | 5 | 나무 방패 | 전투 방어 우위 — 선택적 강화재 |
| | 6 | 철검 | 전투 공격 우위 — 선택적 강화재 |
| | 7 | 가죽 갑옷 | 전투 방어 우위 — 선택적 강화재 |

> **이유**: 기초 5종은 "없으면 죽는" 생존 루프라 잠그면 세션 초반이 너무 가혹해진다. 장비 3종은 "있으면 전투가 유리해지는" 선택재라 잠금-해금의 보상 곡선에 딱 맞는다. 또한 장비 3종(공격1·방어2)은 "철검 먼저? 갑옷 먼저?"의 우선순위 결정도 만든다.

### 5.2 도면 = 월드 획득 아이템

- 도면은 월드에 배치/드롭되는 **획득 아이템**. 주우면 대응 레시피 1종이 즉시 해금.
- 도면 1장 = 레시피 1종 해금 (1:1 매핑). 도면 종류 3가지 (방패 도면 / 철검 도면 / 갑옷 도면).
- **도면은 인벤토리 슬롯을 점유하지 않는 즉시 해금형**을 권장. 줍는 순간 소비되고 해금 상태만 남음 (자원 인플레이션 안티골 + 인벤토리 압박 방지).

### 5.3 세션 내 한정 (영구 진행 금지)

- 해금 상태는 **현재 세션에만 유효**. 세션 종료/재시작 시 전부 기본 5종으로 리셋.
- 저장/계정 귀속 없음. 코어 비전 §10 안티골("영구 진행 금지") 절대 준수.

### 5.4 멀티플레이 — 해금은 **개인(플레이어별)**

| 옵션 | 결정 | 이유 |
|---|---|---|
| 개인 해금 | **채택** | 줍는 사람만 해금 |
| 파티 공유 | 미채택 | — |

- **이유**: 개인 해금이라야 "누가 도면 탐색을 맡을지"라는 분업 결정(코어 비전 시나리오 C)이 진짜 의미를 갖는다. 공유면 한 명만 주워도 전원 해금되어 분업 긴장이 사라진다.
- **단, 도면 아이템 자체는 줍기 전까지 월드에 존재**하므로, "내가 주울까 / 검사(劍士) 역할 동료에게 양보할까"라는 협동 분배 결정이 자연 발생한다 (도면이 한정 자원이기 때문).
- 이 결정은 client에게 "개인 해금이 구현 비용상 수용 가능한가"를 질의한다 (§질의 참조). 비용이 크면 폴백은 5.4-폴백.

> **5.4-폴백 (양보 가능)**: 개인 해금 구현이 무겁다면 **파티 공유 해금**도 수용. 분업 긴장은 약해지지만, "도면 자체가 월드 한정 자원"이라는 점에서 탐험 보상 가치는 유지된다. 단 1순위는 개인 해금.

### 5.5 시간/낮밤 결합 (탐험 시간압박)

- 도면 배치는 **시작 지점에서 먼 영역(숲 깊은 곳 / 산지)에 가중**을 둔다. 멀수록 좋은 도면(철검 등).
- 이로써 "밤 오기 전 도면 찾으러 더 갈까 vs 안전 복귀"가 0순위(낮밤)와 직결된 ② 결정이 된다.
- 야간에는 가시성↓로 도면 찾기가 어려워져, "낮에 미리 vs 밤에 위험 무릅쓰고"의 타이밍 결정도 발생.

## 6. 콘텐츠 분량 (수치 포함)

| 항목 | 수치 |
|---|---|
| 기본 상시 레시피 | **5종** (모닥불·회복포션·체온포션·붕대·횃불) |
| 도면 해금 레시피 | **3종** (나무방패·철검·가죽갑옷) |
| 도면 아이템 종류 | **3종** (1:1 매핑) |
| 세션당 월드 배치 도면 수 | **권장 3~5장** (3종을 다 못 찾을 수도 있게 약간의 결핍 설계 — 5장이면 중복 허용 시 여유, 3장이면 빠듯) |
| 도면 분포 가중 | 원거리 영역(숲 심부/산지)에 고급 장비 도면 가중 |
| 해금 지속 범위 | 세션 내 한정 (영구 0) |
| 해금 단위 | 플레이어 개인별 (멀티) |
| **출시 확대 (2026-06-08)** | 6지역 관통에 맞춰 도면 **5~10장**으로 확대 — 장비 3종 + 도구 등급·덫·방한복 도면 추가. 원거리 지역(설원·폐허)에 고급 도면 가중. 분포: [맵_지역](../맵_지역/기획.md)·[월드오브젝트](../월드오브젝트/기획.md) |

> **결핍 설계 노트**: 세션당 도면을 "3종 전부 확정 등장"으로 하면 탐험이 의무 체크리스트가 된다. **3~4장 무작위 배치 + 일부 종류 누락 가능**으로 "이번 판엔 갑옷 도면을 못 찾았다"는 결핍 서사를 만든다. 정확한 등장 확률/개수는 §8 튜닝 포인트.

## 7. 톤 가이드

- WES는 **어둡고 외로운 야생 생존** (Don't Starve 계열). 도면은 "축하받는 성장 팝업"이 아니라 **버려진 잔해에서 건진 지식**의 느낌.
- 도면 비주얼/플레이버: 낡고 바랜 양피지·찢긴 설계도 톤. "누군가 여기서 죽으며 남긴 것"의 분위기.
- 해금 알림은 화려한 팡파레 금지. **절제된 한 줄 통지** ("철검 도면을 익혔다") 정도. 보상이되 외롭고 건조하게.
- 도면을 못 찾은 세션의 결핍감도 톤의 일부다 — "이번엔 변변한 무기 없이 버텨야 한다"는 압박이 게임 정체성.

## 8. 밸런스 / 튜닝 포인트

| 파라미터 | 초기 권장값 | 튜닝 방향 |
|---|---|---|
| 세션당 도면 배치 수 | 3~4장 | 너무 많으면 갈증 소멸 / 너무 적으면 좌절. 플레이테스트로 조정 |
| 종류별 등장 확률 | 균등에 가깝되 철검(원거리 가중) | 철검을 가장 찾기 어렵게 → 최고 보상 |
| 원거리 영역 가중치 | 산지 = 고급 도면 우세 | 이동(④) 리스크 대비 보상 비례 |
| 도면 발견 후 제작 비용 | **기존 레시피 수치 그대로** | 이번 슬라이스에선 미변경 (안티골) |

> 도면을 **다 못 찾는 게 정상**인 빈도가 목표. "운 좋으면 풀 무장, 운 없으면 맨몸"의 변주가 리플레이성의 핵심.

## 9. 엣지 케이스 (게임플레이 차원)

| 케이스 | 처리 |
|---|---|
| 같은 도면을 두 번 주움 | 이미 해금된 종류면 효과 없음(또는 줍히지 않음). 중복 의미 없음 — 게임적으로 무해 처리 |
| 멀티: 동료가 이미 그 종류 해금, 또 다른 동료가 같은 종류 도면 주움 | 개인 해금이므로 줍는 본인 기준으로 해금. 각자 별도 |
| 도면 없이 세션 내내 진행 | 정상 플레이. 기본 5종만으로 클리어 가능해야 함 (장비는 "있으면 유리"지 "없으면 불가"가 아니어야 — §5.1 분류 전제) |
| 세션 재시작/사망 | 해금 전부 리셋, 기본 5종 복귀 |
| 제작창에서 잠긴 레시피 선택 시도 | 제작 불가 + "도면 필요" 안내. 재료가 충분해도 잠겨있으면 제작 안 됨 |

## 10. 향후 확장 (Backlog)

- **2순위 덫·3순위 도구 등급의 해금 그릇**: 이 도면 시스템이 덫/도구 등급을 담는 그릇이 된다 (로드맵 §2). 덫 도면·도구 도면을 카탈로그에 추가하는 식으로 확장. (이번 슬라이스 범위 밖)
- 도면 카탈로그 5~8장으로 확대 (현재 3장 → 추후 신규 장비/덫 추가 시).
- 도면 희귀도 등급(평범/희귀) 도입 — 더 멀고 위험한 곳일수록 희귀 도면.
- 거래/상자 등 도면 획득 경로 다양화 (현재는 월드 배치/드롭만).

## 11. UI 시각 의도 (디자이너 → 와이어프레임 변환용)

> 본 섹션은 디자이너가 와이어프레임으로 변환할 시각 의도 가이드. 톤·레이아웃 방향만 제시(픽셀 스펙 아님).

### 11.1 도면 획득 알림

- 위치/형태: 화면 상단 또는 중앙 하단의 **절제된 토스트 한 줄**. 풀스크린 팝업·파티클 금지(톤가이드 §7).
- 문구 톤: "○○ 도면을 익혔다" 류 건조한 1줄. 느낌표 남발 금지.
- 색상: 세트의 어둡고 단순한 팔레트. 바랜 양피지색 아이콘 + 무채색 배경. 화사한 골드/네온 금지.
- 지속: 짧게 표시 후 자동 소멸 (3초 내외).

### 11.2 CraftPopup의 잠김/해금 표시

- **잠긴 레시피**: 제작 목록에 **계속 보이되** 회색/실루엣 처리 + 자물쇠 아이콘 또는 "도면 필요" 라벨.
  - 잠긴 칸을 숨기지 말 것 — "보이지만 못 만든다"는 갈증이 ② 탐험 동기의 핵심.
- **해금 레시피**: 정상 컬러로 전환. 해금 직후 짧은 강조(테두리 한 번 반짝) 정도는 허용, 과하지 않게.
- 잠긴 칸 선택 시: 재료 대신 **"이 레시피는 도면이 필요합니다"** 안내. 어디서 찾는지의 힌트 여부는 추후 결정(§10).
- 레이아웃: 기존 CraftPopup 구조 유지. 잠금 상태는 **오버레이 레이어**로만 표현(목록 재배치 금지).
