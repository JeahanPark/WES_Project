## 1. 한줄 컨셉

이미 갖춘 시스템(날씨·이동·코어텐션 오버레이·R3 콘텐츠)에 **감각의 외피**를 입혀, 플레이어가 "혹독한 야생에 혼자 버려졌다"를 *느끼게* 만든다. 새 메커니즘이 아니라 **기존 상태를 증폭하는 레이어**.

## 2. 디자인 골 (Why)

- 코어텐션(추위·밤·저체력·죽음)을 **화면뿐 아니라 소리와 문구로** 다중감각화 → 긴장이 몸에 닿게.
- 날씨/이동이 단순 수치 페널티가 아니라 **분위기 있는 사건**으로 읽히게(비·안개·눈보라 각각의 무드).
- Don't Starve의 외로움 + 건조한 야생 생존 톤을 **텍스트 한 줄 한 줄**로 못박기.
- 음원이 없어도(placeholder/무음) **게임이 완주 가능**하고, 음원이 붙으면 곧장 격이 올라가는 구조.

## 3. 안티 골 (What this is NOT)

- ❌ 새 게임 메커니즘 추가 아님. 데미지·자원·이동 규칙은 R1~R3 그대로.
- ❌ 풀보이스/대사극 아님. 내레이션은 무성(無聲)·절제·1줄.
- ❌ 화려한 오케스트라 BGM 아님. 앰비언트·미니멀·정적 위주.
- ❌ 음원 수급이 R4 완료의 게이트 아님. 음원은 분리된 백로그 트랙.
- ❌ 느낌표·과장·게임적 보상 팡파레 금지(엔딩 포함 건조).

## 4. 달성 가능 우선순위 (음원 의존 적은 순)

| # | 항목 | 담당 | 음원 의존 | R4 포함 |
|---|------|------|-----------|---------|
| ① | **코어텐션 ↔ 날씨/이동 연출 연동** (시각, 코드) | client_r4 | 없음 | ✅ 필수 |
| ② | **인게임 텍스트 확정·반영** (story v0.1 → CSV/UI 문구) | story→client | 없음 | ✅ 필수 |
| ③ | **AudioManager 시스템 골격** (무음/placeholder로 동작, 트리거 연결) | client_r4 | 없음 | ✅ 필수 |
| ④ | **실제 음원 파일** (BGM/SFX/앰비언트) | sound | **높음(외부)** | ⚠️ 가능하면, 아니면 백로그 |

## 5. 핵심 디자인 결정

### 5-1. ① 코어텐션 ↔ 날씨/이동 연출 연동 (음원 무관, 최우선)

기존 CoreTensionOverlayWorker는 Cold/HP/낮밤만 읽는다. R4는 **날씨 상태와 이동 상태**를 같은 오버레이 레이어에 추가 연동.

| 트리거(기존 상태) | 화면 연출 (오버레이 강도) | 톤 |
|---|---|---|
| 비 내림 | 화면 가장자리 빗물 얼룩 + 채도 -10% | 축축하고 무겁게 |
| 안개 | 풀스크린 옅은 회백색 베일 + 시야 비네팅 강화(I-5 재활용) | 무엇이 숨었는지 모르는 불안 |
| 눈보라 | 화면 흩날리는 눈 + 가장자리 성에 + 채도 -25% | 시야가 지워지는 혹독함 |
| 이동 중 추위 누적(Cold 하락 가속 구간) | G-2 추위 비네팅 **맥동 속도 증가** | 몸이 빠르게 식는 체감 |
| 정지/모닥불 근처(Cold 회복) | 추위 오버레이 서서히 **걷힘** | 잠깐의 안도 |

- 결정: **신규 NetworkVariable/Rpc 없음**. 이미 동기화된 날씨 상태(R1)·로컬 이동 상태를 클라이언트가 읽어 화면 연출만. (코어텐션 오버레이 원칙 계승)
- 결정: 오버레이는 **누적 가산**이 아니라 **우선순위 + 블렌드**. 눈보라+저체력+밤이 겹쳐도 화면이 검게 뭉개지지 않게 상한선 둔다(가시성 = 게임플레이 가치, 절대 양보 X).

### 5-2. ② 인게임 텍스트 (story v0.1 → 확정·반영)

story가 [텍스트.md](../../narrative/출시_관통_서사/텍스트.md)에 **전량 기집필**. R4는 신규 집필이 아니라 **확정 + client 반영**.

| 텍스트 군 | 분량 | 반영 위치 | director 확정 사항 |
|---|---|---|---|
| 지역 진입 내레이션 | 6지역 × 2줄 = 12줄 | 신규 지역 진입 토스트 | ✅ v0.1 채택 |
| 아이템 플레이버(Description) | 13종 | ItemInfo.csv Description | ✅ v0.1 채택 |
| 도면 획득 통지 | 2종(신규/중복) | 도면 토스트(G-9) | ✅ v0.1 채택 |
| 관문·이벤트 대사 | 6종 | 이벤트 트리거 UI | ✅ v0.1 채택 |
| 엔딩/게임오버 | 4결말 | 클리어/게임오버 화면 | ✅ 채택. **단 "다회차 불안" 변종(텍스트.md §6)은 R4 채택** — 저장없는 로그라이크 톤에 부합 |

- director 결정: **엔딩 "다회차 불안" 라인 채택** (`불빛 아래, 낯선 눈들이 너를 본다. 이곳은 정말 안전한가.`) — 매 판 초기화되는 세션 구조에 "탈출이 끝이 아닐 수도"라는 여운을 1줄로. 단 첫 클리어는 평이한 성공 라인, 2회차+부터 변종(빈도는 R5 튜닝).
- director 결정: **느낌표 전면 금지**. 모든 통지/내레이션 건조 1줄, UI 한 줄 ≤ 40자(story 규칙 유지).
- director 결정: 토스트 표시시간 = 지역진입 4초 / 이벤트 대사 3.5초 / 도면 통지 3초. 시각 구분은 **표시시간으로만**, 색·신규위젯 분리 안 함(기존 BlueprintToast 재활용, client 합의).
- **CSV 콤마 제약 (client 합의)**: ItemInfo.Description(CSV STRING)은 콤마 불가. 플레이버 13종 중 콤마 2건 톤유지 우회표기 확정 —
  - 약초: `쓴 잎. 상처에도, 추위에도.` → `쓴 잎. 상처에도 추위에도 듣는다.`
  - 특수약초: `늪의 독한 잎. 잘 쓰면 약, 잘못 쓰면 독.` → `늪의 독한 잎. 잘 쓰면 약 잘못 쓰면 독.`
  - 내레이션·이벤트·엔딩은 신규 토스트/화면 문구라 콤마 제약 무관 → 원문 유지.
  - **story 후속 집필 가이드**: 플레이버(ItemInfo.Description)는 처음부터 콤마 없이. 다른 텍스트는 콤마 자유.
- 후속(R5): 몬스터/아이템 **이름(Name) 컬럼** 톤 통일은 story 후속 — R4 스코프 밖(플레이버·문구만).

### 5-3. ③ AudioManager 시스템 골격 (무음으로 동작)

음원 파일이 없어도 **트리거 배선이 완성**돼 게임이 도는 상태가 R4 목표. 음원이 채워지면 데이터만 갈아끼우면 됨.

| 채널 | 역할 | placeholder 동작 |
|---|------|------------------|
| BGM | 지역/상황별 배경음 | 무음 또는 단일 톤 루프 |
| Ambient | 환경음(비/바람/안개/밤벌레) | 무음, 트리거만 발화 로그 |
| SFX | 타격·수집·제작·UI·발소리 | placeholder 비프 또는 무음 |
| Stinger | 사건 강조(보스 등장·동료 사망·탈출) | 무음 |

- 결정: 사운드 **무드 맵**을 디렉터가 제공(아래) → sound가 음원 생성 시 가이드, client가 트리거 배선 시 어떤 이벤트에 어떤 채널을 거는지 기준.
- 결정: 음량/믹스 우선순위 = **Stinger > SFX > Ambient > BGM**. 사건이 음악에 묻히지 않게.

### 5-4. ④ 실제 음원 (외부 의존 — 가능하면)

- Suno/ElevenLabs Playwright 자동화로 BGM/앰비언트/SFX 생성 시도.
- **인증/접근 불가 시 즉시 백로그**로 분리(R4 완료 막지 않음).
- 우선 생성 목록(되는 만큼): ⓐ 메인 앰비언트 루프 1종 ⓑ 눈보라/비 환경음 2종 ⓒ 보스 등장 stinger 1종 ⓓ 핵심 SFX(타격·수집) 2종. — 최소 6클립이 "음원 있음" 체감 임계.

## 7. 톤 가이드

### 7-1. 날씨별 연출 톤
- **비**: 무겁고 축축. 채도 낮추고 화면을 살짝 어둡게. "쉴 곳이 필요하다"는 압박.
- **안개**: 정적·불안. 시야를 좁혀 "무엇이 숨었는지 모른다"(내레이션과 호응). 적의 실루엣만 흐릿하게.
- **눈보라**: 가장 혹독. 시야 거의 지움 + 추위 가속. "여기서 멈춘 자들"의 공포.

### 7-2. 텍스트 톤
- 건조·절제·무성 내레이션. "먼저 온 자들의 흔적"이 바닥 정서.
- 1줄 ≤ 40자, 느낌표 금지, 감탄·과장 금지.
- 보상조차 담담하게(`철검 도면을 익혔다.` — 팡파레 없음).

### 7-3. 사운드 무드
- **BGM**: 미니멀·앰비언트·정적. 멜로디보다 질감(드론·바람). 지역이 험할수록 더 비고 더 춥게.
- **Ambient**: 환경이 주인공. 비·바람·눈보라·밤벌레·먼 짐승 소리. 외로움 강조.
- **SFX**: 건조하고 묵직. 타격은 둔탁, 수집은 작고 메마르게.
- **Stinger**: 드물게, 짧게. 보스 등장·동료 사망·탈출 순간에만. 남발 금지.
- 전체 정서: **고요 → 사건 → 고요**. 침묵이 무기. 항상 소리가 깔려있지 않게.

## 8. 밸런스/튜닝 포인트 (R5 인계)

- 오버레이 블렌드 상한선(눈보라+밤+저체력 겹칠 때 최저 가시성) — QA로 실측.
- 추위 맥동 속도 vs Cold 하락률 매핑 곡선.
- "다회차 불안" 엔딩 등장 빈도(2회차+ 몇 %).
- 사운드 믹스 음량 밸런스(채널 간), stinger 빈도.
- 안개 시 적 실루엣 가시 거리(너무 안 보이면 불공정, 너무 보이면 무드 손실).

## 9. 엣지 케이스 (게임플레이 차원)

- **오버레이 과중첩으로 시야 차단** → 가시성 상한선이 게임플레이를 항상 보호(연출 < 플레이 가치).
- **음원 없음(④ 백로그)** → 무음으로도 완주 가능, 텍스트/시각이 감각 공백 메움.
- **멀티 시 동료 사망 stinger/텍스트** → 로컬 연출, 동기화된 상태만 읽어 각 클라이언트가 표시.
- **텍스트 길이 초과(번역/이름 치환 후 40자↑)** → UI 줄바꿈 또는 …처리는 client/designer 협의(게임의도: 잘려도 첫 줄에 핵심).
- **내레이션과 SFX 동시 발생** → stinger가 내레이션 가독성 해치지 않게 타이밍 분리.

## 10. 향후 확장 (Backlog)

- 실제 음원 전량(④의 미수급분) — 외부 사이트 트랙.
- 몬스터/아이템 **이름 컬럼** 톤 통일(story 후속).
- 동적 음악(전투 진입 시 레이어 추가/감소).
- 보스별 전용 stinger·테마.
- 발소리 지면 재질별 분기(눈/물/돌).
- 음성 내레이션(현재 무성 유지가 톤 — 장기 보류 가능).
