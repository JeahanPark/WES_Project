# 날씨 시스템 — 기획서

> **연관 구현**: 시간/낮밤 시스템(`DayNightWorker`, 이미 구현) — 날씨는 이와 결합.

---

## 1. 한줄 컨셉

**예측할 수 있지만 빗나갈 수 있는 환경 변화로, 4분기 투자 결정에 "베팅과 후회"를 만드는 불완전 정보 시스템.**

## 2. 디자인 골 (Why)

- **CORE 0순위.** 4분기 트레이드오프가 "계산 가능한 최적해"가 되지 않으려면, 플레이어가 *잘못 판단할 수 있어야* 한다(CORE 6장). 날씨가 그 잘못의 여지를 만든다.
- **예측 단서 제공 + 빗나갈 여지** → "괜찮을 것 같다" 베팅 → 적중/빗나감 → **후회 루프**(CORE 재미 메커니즘).
- **시간(낮밤)과 결합**해 복합 압박. 밤+악천후 = 결정의 무게 최고조.
- 난이도 공식 `EnvPressure`의 `weatherMul` 실체. 날씨가 위협 압력 TP를 직접 끌어올린다.

## 3. 안티 골 (What this is NOT)

- ❌ **순수 랜덤**(단서 없는 기습). 학습 불가 = CORE 함정 2. 모든 악천후는 전조 단서를 동반.
- ❌ **완전 예측 가능**(긴장 소멸). 단서는 확률만 제공, 빗나감이 핵심.
- ❌ **날씨 종류 비대화.** 4~5종 핵심만(자원 인플레이션 안티골과 동일 정신).
- ❌ **단순 페널티**(그냥 불편함). 모든 날씨는 반드시 **4분기 결정으로 환원**돼야 한다.

## 4. 플레이어 경험 흐름 (베팅-후회 루프)

```
전조 단서 인지 (하늘·구름·기온)
   ↓
다음 날씨 예측 ("비 올 것 같다 / 버틸 만하다")
   ↓
4분기 투자 베팅 (안전 보강? 진행 강행? 효율 더?)
   ↓
날씨 실현 (예측 적중 or 빗나감 — 확률적)
   ↓
적중: 베팅 만족 / 빗나감: "그때 다르게 투자했어야" 후회
   ↓
다음 베팅이 더 정교해짐 (단서 학습)
```

## 5. 핵심 디자인 결정

### 5.1 날씨 종류 (4~5종)

| 날씨 | 게임플레이 영향 | 전조 단서 | 주 출현 지역 |
|------|-----------------|-----------|--------------|
| **맑음** | 기준 (영향 없음) | — | 전역(특히 초반) |
| **흐림** | 약한 시야↓, **악천후 전조 신호** | 구름 증가·빛 약화 | 전역 |
| **비** | 시야↓, 모닥불 효율↓, 체온(Cold) 가속 감소 | 흐림 지속·먹구름 | 숲·늪지 |
| **안개** | 시야 극↓, 몬스터 접근 은폐(예측 어려움) | 새벽·습한 지역·기온차 | 늪지 |
| **눈보라** | 체온 극대 압박, 시야↓, 이동 비용↑ | 기온 급강하·하늘 변색 | 설원 |

> 5종 상한. 후반 지역일수록 악천후 비중↑(난이도 곡선).

### 5.2 예측 메커니즘 (불완전 정보 핵심)

- 각 날씨는 **전조 단서**(시각·환경 신호)를 동반 — 플레이어가 읽을 수 있는 정보.
- 단서 → 다음 날씨의 **확률 분포**(예: 먹구름 = 비 70% / 유지 30%). 100%가 아니다.
- **빗나가는 30%가 후회의 원천.** 단, 학습 가능하도록 확률은 일관(같은 단서 = 같은 분포).
- 협동: 한 명이 단서를 읽고 공유 → 정보 공유 자체가 분기 결정(CORE 시나리오 C).

### 5.3 전이 규칙

- 날씨는 시간대(낮밤 페이즈)와 동기화된 틱에서 확률 전이.
- 급격한 무작위 점프 금지 — 맑음→흐림→비 같은 **단계적 악화**(전조 보장).
- 지역 진입 시 그 지역의 날씨 분포 테이블 적용.

### 5.4 4분기 연결 (가장 중요)

| 날씨 신호 | 발생하는 트레이드오프 |
|-----------|----------------------|
| **비 예보** | 모닥불·체온 보강(안전) vs 진행 강행(이동) vs 채집 더(효율) |
| **안개** | 시야가 없으니 덫/표지(안전) vs 정찰(이동)의 가치가 뒤바뀜 |
| **눈보라 임박** | 체온 자원 비축(안전) vs 자원 더 모으고 출발(효율) — 빗나가면 동사 |

→ 날씨는 페널티가 아니라 **결정 트리거**. 이 연결이 없으면 안티골 위반(단순 불편).

## 6. 콘텐츠 분량

- 날씨 **5종**(맑음/흐림/비/안개/눈보라). 각 1) 영향 2) 전조 단서 3) 지역 분포 테이블.
- 신규 비주얼/사운드 의뢰: 날씨별 화면 오버레이·앰비언스(designer/sound 영역, 후속).

## 7. 톤 가이드

- 어둡고 혹독. **날씨 자체가 위협**이며, 맑음조차 "다음이 올 것"이라는 불안을 깐다.
- 비=음울, 안개=불길·고립, 눈보라=절망적 혹한. 환경이 적인 Don't Starve 계보.

## 8. 밸런스 / 튜닝 포인트 (→ level-design 수치)

- **악천후 빈도 곡선**: 지역 깊이에 따라 악천후 확률↑ (난이도 공식과 연동).
- **예보 정확도**(빗나감 확률): 너무 정확하면 긴장 소멸, 너무 부정확하면 학습 불가. 단서별 확률 분포 튜닝.
- **영향 강도**: 각 날씨의 시야·체온·이동 계수 → 난이도 공식 `weatherMul`(예: 맑음 1.0 / 비 1.2 / 안개 1.3 / 눈보라 1.6).
- 계수·지역별 분포 테이블은 **level-design 밸런스 명세(CSV)** 영역.

## 9. 엣지 케이스 (게임플레이 차원)

- **협동 동기화**: 날씨 전이가 호스트 권한으로 전원 동일하게 보여야(서버 권한).
- **거점/실내 회피**: 모닥불·건물 근처가 날씨 일부를 상쇄하는가? (안전 분기 가치와 연결) — 결정 필요.
- **예보 못 본 플레이어**: 단서가 너무 미묘하면 불공정. 최소한의 가독성 보장.
- **날씨×야간 중첩 상한**: 밤+눈보라가 동시에 와도 진행 *불가능*은 아니어야(D>1 스파이크는 일시적).

## 10. 향후 확장 (Backlog)

- 계절 개념(장기 변동) — 단 9시간 단일 세션엔 과할 수 있음, 출시 후 검토.
- 극단 이벤트(폭풍·천둥) — 단발 고강도 위협.
- 날씨 연동 몬스터 행동(비 오면 특정 적 활성) — 불완전 정보 심화.

## 11. 참고 / 영감

- CORE 6장(불완전 정보의 원천) · CORE 9장 함정 2(랜덤 vs 학습 균형).
