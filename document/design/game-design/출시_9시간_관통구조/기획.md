## 1. 한줄 컨셉

**9시간 동안 "이 자원을 어디에 투자하지?"의 무게가 갈수록 무거워지다, 마지막엔 비가역 베팅 하나로 탈출이 갈리는 단일 세션 협동 생존.**

> 재미는 지역을 통과하는 것이 아니라, **매 순간의 4분기 자원 분배 결정**(효율/테크/안전/이동)이다. 지역·시간은 그 결정을 담는 무대일 뿐.

## 2. 디자인 골 (Why)

- **9시간 = 트레이드오프 압박의 누적 곡선.** 콘텐츠 가짓수가 아니라 **결정의 깊이·빈도·비가역성**이 시간을 채운다.
- **불완전 정보(날씨·시간·몬스터 패턴)를 9시간의 척추로 둔다.** 플레이어가 *잘못 판단할 수 있어야* 재미가 성립(CORE 6장). "덫 썼어야 했는데"의 후회 루프가 9시간 내내 살아있게.
- **4분기를 점진 개방**한다. 초반엔 단순(즉시소모 vs 비축), 후반엔 4분기가 동시에 후보이며 자원은 희소 → 결정이 치명적.
- 협동에서 4분기 = **역할 분담**(CORE 시나리오 C). 분업이 무너질 때의 텐션이 9시간을 지탱.

## 3. 안티 골 (What this is NOT) — CORE 안티골 준수

- ❌ **콘텐츠 양으로 9시간 채우기.** 자원 30+종·레시피 비대·테크 트리 거대화 금지(CORE 10장). 재미는 자원 *가짓수*가 아니라 같은 자원의 *투자 결정*.
- ❌ 영구 진행(세션 간 저장·메타 해금). 매 판 처음부터.
- ❌ 무한 샌드박스. 명확한 종착(마을 탈출).
- ❌ 분기가 거짓 선택(항상 같은 분기가 정답)이 되는 것 — 4분기 페이오프 균형 필수(CORE 9장).
- ❌ 풀 다이얼로그/서사 분기, PvP, 캐릭터 클래스 (CORE 10장).

## 4. 플레이어 경험 흐름 — 4분기 트레이드오프 압박 곡선

> 지역 = "새 위협 채우기"가 아니라 **"새로운 트레이드오프 압박 또는 새 불완전 정보를 도입하는 단위"**.
> 난이도 = 활성 분기 수 × 자원 희소성 × 불완전 정보의 변동성. (공간 깊이는 그 매개)

| # | 지역 | 체류 | 이 구간에서 **새로 켜지는 압박** | 활성 분기 | 불완전 정보 |
|---|------|------|----------------------------------|-----------|-------------|
| 0 | 난파 해변 | ~40m | **즉시소모 vs 비축** (가장 단순한 트레이드오프 입문) | 이동·획득 | 없음(학습 구간) |
| 1 | 숲 | ~1.5h | **효율 vs 안전** — 첫 위협(늑대)+야간 도입 → 도구 갈까 방어 갈까 | +안전(첫 대응) | **시간(낮↔밤)** |
| 2 | 늪지/강 | ~1.5h | **능동 안전 본격** — 덫/표지 등장 + 시야·추위로 예측 어려움 | +안전(덫/표지) | **날씨(시야·추위)** |
| 3 | 산지/폐광 | ~2h | **효율·테크 본격** — 도구 등급·도면 해금. 자원 풍부하나 고위험 | +효율·테크 | 몬스터 패턴 변동 |
| 4 | 설원 | ~1.5h | **이동 비용 극대** — 이동=체온 소모, 보급난. 4분기 동시 충돌 최고조 | 4분기 전부 | 혹한 변동·보급 |
| 5 | 폐허 | ~1.5h | **비가역 최종 베팅** — 남은 자원을 탈출 푸시 vs 안전 보강 | 4분기(자원 최소) | 최종 관문 불확실 |
| — | 마을 | — | 종착 | — | — |

**합계 ≈ 8.5~9시간.** 핵심: 뒤로 갈수록 **활성 분기는 늘고 자원은 줄어** 결정 1회의 무게가 기하급수로 커진다.

## 5. 핵심 디자인 결정

1. **4분기 점진 개방**: 모든 분기를 처음부터 주지 않는다. 지역마다 1개씩 켜서, 플레이어가 트레이드오프를 단계적으로 학습하고 후반에 동시 저글링하게.
2. **불완전 정보가 진행과 함께 강해짐**: 시간(숲)→날씨(늪)→혹한 변동(설원). 후반일수록 예측이 빗나갈 여지가 커져 베팅의 긴장 상승.
3. **자원 희소성 역곡선**: 초반은 자원 풍족(결정이 가벼움), 후반은 희소(결정이 치명적). 콘텐츠 양이 아니라 **희소성**이 후반 난이도.
4. **비가역성**: 쓴 자원은 안 돌아온다(CORE). 특히 후반 "마지막 베팅"이 엔딩을 가르도록 — 9시간의 클라이맥스.
5. **저장 없음 가혹함 완화(세션 내 한정)**: 협동 부활·지역 안전지대 회복은 두되, **진행 자체는 저장 안 됨**. 전멸=처음부터의 무게는 유지하되 단발 실수로 9시간이 날아가진 않게 튜닝.

## 6. 콘텐츠 분량 (CORE 안티골 준수 — "적은 자원, 깊은 결정")

> 현재(임시)에서 *최소한만* 늘린다. 9시간은 분량이 아니라 결정으로 채운다.

| 콘텐츠 | 현재 | 출시 목표 | 상한 근거(CORE) |
|---|---|---|---|
| 원재료 자원 | 8종 | **~10종 이내** | ❌ 30+종 금지. 같은 자원의 투자처가 많은 게 핵심 |
| 도면(테크 해금) | 0 | **5~10장** | CORE 9장 "트리 아닌 카탈로그 5~10개" |
| 능동 기술 | 0 | **덫 1~2 + 표지 1** | CORE 4.3 "처음부터 4종은 과함" |
| 도구 등급 | 0 | **2~3 단계** | 효율 분기 |
| 장비 | 6종 | **등급 트리 ~8종** | 양보다 등급 결정 |
| 몬스터 | 3종 | **지역별 1~2종 + 패턴 변동** | 가짓수보다 **행동/패턴**(불완전정보) |
| 건물/방어 | 2종 | **6~8종** | 능동 안전 분기 |
| 불완전 정보 | 0 | **시간 + 날씨** (0순위) | 이거 없으면 4분기 의미 약함 |

→ 총 아이템은 **~30종 내외**(v0.1의 40~55 철회). 9시간 = 이 적은 자원을 **언제·어디에** 쓸지의 반복 결정.

## 7. 톤 가이드

- 어둡고 외로운 야생(Don't Starve 톤). 환경이 점점 혹독(해변 온화→설원 혹한→폐허 황폐).
- **정서의 핵심 = 후회와 결단.** "그때 다르게 투자했어야"의 무게. 협동의 온기 vs 환경의 냉혹.

## 8. 난이도 공식 (정량 모델)

> 난이도를 "공간 깊이로만" 같은 정성 표현이 아니라 **계산 가능한 공식**으로 정의한다.
> 핵심: 체감 난이도는 **위협 압력 ÷ 플레이어 역량**이며, 역량은 **플레이어의 4분기 투자 결과**다 → 난이도가 코어 재미(트레이드오프)와 직접 연결된다.

### 8.1 체감 난이도

```
D(t) = TP(t) / CAP(t)

  D ≈ 1.0  : 팽팽함 (의도된 긴장 — 목표 구간)
  D > 1.0  : 압도   (잘못된 투자의 결과 → "후회")
  D < 1.0  : 여유   (과투자 — 다른 분기에 썼어야)
```

t = 진행도(경과 시간 또는 지역 깊이 d). 의도 곡선: 평균 D를 1.0 근처로 유지하되, **관문·야간·악천후에서 일시 D>1 스파이크**(불완전 정보가 만드는 변동).

### 8.2 위협 압력 TP(t)

```
TP(t) = EnemyScale(d) + EnvPressure(t) + ScarcityPressure(d)

  EnemyScale(d)      = base_enemy × (1 + k_e · d)          # 지역 깊이 d에 따른 적 HP·ATK 배수
  EnvPressure(t)     = coldDrain(t) × phaseMul(시간대) × weatherMul(날씨)   # 체온·환경 압박
  ScarcityPressure(d)= k_s / resourceDensity(d)            # 자원 희소성(후반 ↑)
```

### 8.3 플레이어 역량 CAP(t) — 4분기 투자의 산물

```
CAP(t) = w1·ToolTier + w2·GearScore + w3·TechUnlocked + w4·StockpileBuffer

  ToolTier        ← 효율 분기 투자 (도구 등급)
  GearScore       ← 효율/안전 분기 (장비)
  TechUnlocked    ← 테크 분기 (해금 도면 수)
  StockpileBuffer ← 비축 + 능동 안전(덫/표지)으로 흡수한 위협
```

→ 플레이어가 트레이드오프를 잘하면 CAP↑로 D를 1 근처 관리. 잘못 투자하면 그 구간 TP에 D>1로 압도 = **설계된 후회**.

### 8.4 director는 여기까지 / level-design이 이어받는 것

- **director(이 문서)**: 공식 **구조**와 의도 곡선(목표 D, 스파이크 지점) 정의.
- **level-design(후속 명세)**: 계수(`base_enemy, k_e, k_s, w1~w4`)·지역별 `resourceDensity(d)`·`weatherMul` 등 **실수치 튜닝**. CSV 밸런스 입력은 level-design 직접 편집 영역.
- **검증**: 시나리오별 D(t) 시뮬레이션 + QA 효과측정으로 거짓 선택(특정 분기만 정답) 여부 확인.

## 8.5 밸런스 / 튜닝 포인트

- **4분기 페이오프 균형**(CORE 함정 1): 어느 분기도 항상 정답이면 안 됨. 시나리오 검증 필수.
- **불완전 정보 변동성**(CORE 함정 2): 날씨·패턴에 진짜 변동 — 단 너무 랜덤이면 학습 불가, 단서(예보) 제공.
- **자원 희소성 곡선**: 지역별 자원 밀도 = 결정 무게의 핵심 다이얼.
- **결정 빈도**(CORE 7장): 자원 획득 직후·사이클 전환·위험 신호·세션 분기점 — 모두 4분기가 동시 후보.
- **인원 스케일**: 1인 9시간 / 4인 협동 적 밀도·자원 보정.

## 9. 엣지 케이스 (게임플레이 차원)

- **솔로 4분기 저글링**(CORE 함정 4): 솔로는 줄여서 가능해야. 4인 분담은 덤.
- **협동 1인 이탈/디스커넥트**: 남은 인원으로 9시간 지속 + 난이도 재스케일.
- **전멸 시 처음부터**: 완충(부활/안전지대) 강도가 출시 체감 좌우.
- **거짓 선택 방지**: 각 지역에서 "한 분기만 정답"이 되지 않는지 시나리오 검증.
- **후반 보급/회귀**: 앞 지역 회귀 허용 여부 — 일방향 게이트로 막을지.

## 10. 향후 확장 (Backlog)

- **지역별 상세 기획** — 각 지역 = 독립 문서(도입 압박·자원·관문·불완전정보·서사·레벨명세).
- **4분기 후속 설계서** — CORE 11장 순서(시간/날씨 0순위 → 도면 → 덫 → 도구등급 → 이동비용).
- **엔딩 분기**(탈출 인원/방식) · 시드 변주(리플레이).
