## 1. 컨셉 — "적은 자원, 깊은 결정"

- CORE 안티골 준수: **자원 가짓수로 9시간을 채우지 않는다.** 재미는 **같은 자원을 4분기 중 어디에 투자할지**의 결정.
- 원재료는 **~9종 상한**(현재 6종 + 지역별 3). 대신 각 자원이 **여러 투자처**를 가져 결정이 무겁다.
- 후반 지역일수록 자원 **희소** → 결정이 치명적(난이도 공식 `ScarcityPressure`).

## 3. 원재료 자원 (출시 ~9종, 지역별 분포)

| 자원 | 주 획득 지역 | 획득 방식 | 희소성 |
|------|--------------|-----------|--------|
| 나무 | 해변·숲 | 채집 | 풍부(초반) |
| 돌 | 해변·산지 | 채집 | 풍부 |
| 약초 | 숲 | 채집 | 보통 |
| 가죽 | 숲(늑대) | 드롭 | 보통 |
| 뼈 | 숲·산지 | 드롭 | 보통 |
| 철광석 | 산지·폐광 | 채굴 | 적음(중반) |

## 5. 도구 등급 — ① 효율 분기 (신규)

> CORE에서 "전무"였던 ①효율 축을 도구 등급으로 점화.

- **본질**: 같은 시간·노력으로 더 많이 채집. "미래 효율을 산다".
- **등급**: 1단계(맨손/석기) → 2단계(철 도구, 철광석 소비) → 3단계(고철 도구, 폐허).
- **트레이드오프**: 철광석을 *도구*(효율)에 쓸까 *철검*(전투)에 쓸까 → §2 매핑의 핵심 충돌.
- **후반 희소성 대응**: 도구 등급이 높을수록 희박한 후반 자원을 그나마 효율적으로 — 단 도구에 투자한 자원만큼 다른 분기를 포기.
- 구체 채집 배수·비용은 제작레시피 문서 + level-design 튜닝.

## 6. 희소성 곡선 (난이도 공식 연동)

- 지역 깊이 `d`에 따라 `resourceDensity(d)` 감소 → `ScarcityPressure(d) = k_s / resourceDensity(d)` 증가.
- 초반(해변·숲): 자원 풍부 = 결정 가벼움(학습). 후반(설원·폐허): 희소 = 결정 치명적.
- 지역별 자원 밀도 수치 = **level-design 밸런스 명세**.
