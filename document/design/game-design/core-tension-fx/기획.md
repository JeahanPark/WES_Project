## 1. 한줄 컨셉

> **이미 돌아가는 코어 텐션(추위·시간압박·도면해금)을, 신규 시스템 0개로 "화면이 말하게" 만들어 플레이어 경험으로 완성한다.**

세 시스템은 로직상 완성됐지만 플레이어는 그 긴장을 눈으로 못 느낀다. 이 기획은 그 **전달 갭만** 닫는다. 새 메커니즘을 추가하지 않는다.

---

## 2. 디자인 골 (Why)

| 골 | 설명 |
|---|---|
| **코어 재미의 전달** | 자원투자 트레이드오프의 긴장은 이미 시스템에 있다. 그러나 "느껴지지 않으면 없는 것"이다. 연출은 코어 재미를 비로소 재미로 만든다. |
| **불완전 정보의 시각 단서화** | 낮밤은 타이머가 아니라 환경 톤으로 인지된다(시간 기획 §5-2). 추위는 숫자가 아니라 화면이 파래지는 것으로 인지된다. 연출이 곧 게임의 "정보 채널"이다. |
| **결정 타이밍에 정서 Beat 부여** | 황혼=귀환 압박, 추위 2단계=죽음의 예감, 도면 해금=탐험 보상. 각 코어 결정 시점에 화면이 감정을 입혀 결정의 무게를 키운다. |
| **세 연출의 톤 통합** | 세 시스템이 각자 다른 룩으로 따로 놀면 톤이 깨진다. 추위(청)·밤(남)·해금(양피지)을 한 정서 팔레트 아래 묶는다. |
| **온보딩의 씨앗** | 연출이 곧 학습 단서다(스케줄 §C 의존). "화면이 파래지면 위험하다"를 말 없이 가르친다. |

## 3. 안티 골 (What this is NOT)

- ❌ **신규 시스템 금지.** 추위/시간/도면 로직에 손대지 않는다. 새 스탯·새 메커니즘·새 위협 0개. 이 기획은 **출력 레이어만** 다룬다.
- ❌ **연출 = 장식 아님.** "예쁘게 꾸미기"가 아니라 코어 텐션의 전달이 목적. 코어와 무관한 파티클·화려한 트랜지션은 안티골.
- ❌ **자원/시각 인플레이션 금지.** 세트의 어둡고 단순한 톤 유지. 화면을 정보로 채우지 않는다 — 침묵과 절제가 톤의 일부.
- ❌ **수치·구현 방식 미지정.** 알파 값·전환 시간 등 구체 수치 방향은 제시하되, 셰이더/렌더 파이프라인/코드 결합 방식은 client 영역.
- ❌ **디지털 타이머·게이지 숫자 노출 금지.** 시간은 하늘로, 추위는 화면 물듦으로만. "남은 시간 5:42" 같은 정확 수치는 불완전 정보를 죽인다.
- ❌ **즉사 연출·점프스케어 금지.** 추위도 해금도 점진·절제. 갑작스러운 풀스크린 플래시·사운드 스팅 없음(추위는 "서서히 파래지는 세상", 추위 기획 §7).
- ❌ **밸런스 변경 금지.** 단계 임계값(Cold 30/60/90)·페이즈 길이는 기존 확정값 그대로. 연출은 그 위에 얹힌다.

## 4. 플레이어 경험 흐름

세 연출이 한 세션 안에서 어떻게 겹쳐 흐르는지 — 따로가 아니라 같은 화면 위에 합성된다.

```
[낮] 화면: 빛바랜 회청. 추위 오버레이 없음. 평온하되 긴장의 바닥음.
  │  플레이어: 탐험·채집. "도면 찾으러 더 갈까?"
  │
  ├─ [도면 발견] 양피지 토스트 한 줄 + 제작창 잠긴 칸이 정상색으로.
  │     → 어두운 화면에 잠깐 스민 건조한 보상. 팡파레 없음.
  │
[황혼] 하늘이 붉은 주황으로 물들기 시작. 환경 톤이 경고색으로 전환.
  │  → "돌아가야 하나?" 시간압박이 색으로 체감됨. 추위 오버레이는 아직 없음.
  │
[밤] 화면 짙은 남색 + 시야 축소. 모닥불 꺼지면 →
  │  추위 청색 오버레이가 밤 톤 위에 겹쳐 차오르기 시작.
  │     Cold 1단계: 가장자리에 옅은 청색 성에.
  │     Cold 2단계: 청색 짙어지고 외곽 어두워짐. "죽음이 가까워진다."
  │     Cold 3단계: 흰빛 서리 + 느린 펄스. 세상이 얼어붙음. 그러나 HP 1 보호 = "아직 살 수 있다"의 시각 잔불.
  │  → 모닥불 재점화 / 새벽 도래 → 오버레이가 서서히 걷힘.
  │
[새벽] 서리 걷히고 하늘 희뿌옇게. "또 버텼다"의 차가운 안도.
```

**핵심**: 세 연출은 독립 채널이되 한 화면에 합성된다. 밤 톤(환경) + 추위 오버레이(생존 위협) + 해금 토스트(탐험 보상)가 동시에 떠도 서로 안 싸우게 톤·레이어 우선순위를 정한다(§5.4).

## 5. 핵심 디자인 결정

### 5.1 두 위험 채널의 분리 — 추위(청) vs 전투(적)

감사 §6에서 박은 원칙: **HUD 위험 채널은 2종으로 분리**한다.

| 채널 | 색 | 의미 | 본 기획 범위 |
|---|---|---|---|
| 추위 | 청색→흰 서리 | 환경 위협(Cold) | ✅ 이번 (G-2) |
| 전투 | 적색 비네팅 | 피격·저체력(HP) | 감사 G-13, 본 기획 범위 밖(언급만) |

> 추위와 전투를 같은 적색으로 뭉치면 플레이어가 위협의 종류를 구분 못 한다. **청=추위, 적=전투**의 색 약속이 게임 내내 일관돼야 한다. 본 기획은 추위 채널만 봉인하되, 전투 채널과 색이 충돌하지 않게 청색을 못박는다.

### 5.2 추위 오버레이 — 3단계 점진 (G-2)

기존 추위 기획 §5 단계표를 **연출 관점에서** 확정한다(수치는 기존 확정값 참조, 변경 없음).

| 단계 | Cold | 화면 정서 | 연출 방향 |
|---|---|---|---|
| 정상 | 0~29 | 평온 | 오버레이 없음 |
| 1단계 경고 | 30~59 | 불안의 시작 | 화면 가장자리에 옅은 청색 비네트. 중앙 시야는 깨끗 — "아직 괜찮지만 시작됐다" |
| 2단계 약틱 | 60~89 | 죽음의 예감 | 청색 짙어지고 비네트가 안쪽으로 번짐. 화면 외곽 어두워짐(혈관 수축 메타포) |
| 3단계 강틱 | 90~100 | 얼어붙음 | 흰빛 서리가 화면을 덮음. 느린 펄스(1.5초 주기)로 "심장이 식어가는" 리듬. 단 완전 불투명 금지 — **알파 상한 0.85 캡**(코드 안전망)으로 중앙 시야 일부 항상 보존. HP 1 보호처럼 "살 수 있다"의 시각 여백 유지 |

> **차폐 방지 확정(client v0.1 결정)**: 단계3 알파 상한을 **0.85로 코드 캡**한다(공정성 천장). 그 천장 안에서 designer가 텍스처 투명 여백으로 자연화. 캡=공정성 보증, 텍스처=품질 향상의 2중 구조.

**연출 원칙**:
- 단계 전환은 **부드럽게**(스냅 전환 금지). 추위는 점진적이어야 한다(추위 기획 §7).
- 펄스는 3단계에서만. 1·2단계는 정적 — 펄스를 일찍 쓰면 위급 신호가 닳는다.
- 회복 시(모닥불/낮) 오버레이는 **차오를 때보다 약간 빠르게** 걷힌다 — 안도감을 즉각 보상.
- **시각 단계 = 데미지 단계 단일 소스(client v0.1 결정)**: 화면이 파래지는 임계(30/60/90)와 서버 데미지 임계를 하나의 소스로 통일한다. "화면은 멀쩡한데 죽는다"가 발생하면 코어 텐션 전달이 정면 훼손되므로, 단일 소스화는 연출 품질이 아니라 **기획 요구사항**이다.

### 5.3 낮밤 환경 톤 — 4페이즈 색 전환 (G-3)

시간 기획 §5-2 환경 단서표를 **화면 톤 연출로** 확정. 디지털 타이머 절대 금지, 하늘/조명 색만으로.

| 페이즈 | 하늘/환경 톤 | 정서 | 시야 |
|---|---|---|---|
| 낮 | 빛바랜 회청색(흐린 날) | 안도+바닥 긴장 | 100% |
| 황혼 | 짙은 붉은 주황("피처럼 물든") | 불안·귀환 압박 | 75% |
| 밤 | 어두운 남색/검정(먹구름) | 고립·외로움 | 40%(모닥불/횃불 주변 제외) |
| 새벽 | 희뿌연 회색 | 차가운 안도 | 60%→100% |

**연출 원칙**:
- 페이즈 전환은 **점진 페이드**. 특히 낮→황혼은 천천히 물들어야 "어두워지는 줄 몰랐다"의 불완전 정보가 산다(시간 기획 §5-2). 스냅 전환은 시간을 너무 명확히 알려줘 텐션을 죽인다.
- 황혼의 붉음은 **아름다운 노을이 아니라 불길한 경고색**(시간 기획 §7-4). 채도를 낮춰 "예쁜 석양"이 되지 않게.
- 밤의 시야 축소는 이미 시스템에 있음 — 이 기획은 그 어둠의 **색·질감**(낭만적 밤하늘 아님, 먹구름 어둠)만 톤으로 입힌다.
- HUD의 페이즈 아이콘 1개(시간 기획 §5-2)는 유지하되, **환경 톤이 주 채널**이고 아이콘은 보조. 아이콘에 의존하면 환경 톤 연출이 실패한 것.

### 5.4 합성 레이어 우선순위 — 세 연출이 안 싸우게

세 연출이 같은 화면에 겹칠 때의 톤 충돌을 방지하는 **정서적 우선순위**(구현상 레이어 순서가 아니라, "무엇이 화면을 지배해야 하는가"의 기획 의도).

| 상황 | 지배 채널 | 이유 |
|---|---|---|
| 밤 + 추위 누적 | **추위(청)** 위로 | 추위는 즉각적 죽음 위협. 밤 톤(남색) 위에 청색 서리가 얹혀 "밤이 추위로 깊어진다" |
| 밤 + 추위 + 도면 발견 | 토스트는 **잠깐 떴다 사라짐** | 해금 보상은 일시적 알림. 추위/밤 배경 톤을 가리지 않고 위에 스쳐 지나감 |
| 황혼 + 추위 시작 | 둘 다 약하게 공존 | 황혼은 아직 추위 본격 전. 붉은 환경 + 옅은 청 비네트가 "전환의 불안"을 만듦 |

> 핵심: **밤·추위는 배경 톤(지속), 해금은 전경 알림(순간).** 지속 채널끼리는 색 대비로 공존(남색 배경 + 청색 위협), 순간 채널은 배경을 안 가린다.

#### 합성 z-order 확정 (client v0.2 — designer 배치 기준)

아래→위 레이어 순서:

> **밤 톤 틴트 → 추위 오버레이 → 적색 HP 비네팅 → 안개(백로그) → 도면 토스트 / 사망 암전(최상위)**

- **밤(최하)→추위(위)**: §5.4 "밤 톤 위에 청색 서리 얹힘" 그대로. 환경 위에 위협이 쌓임.
- **적색 HP 비네팅이 추위 위**: 즉각적 죽음(피격)이 환경 위협(추위)보다 주의 우선. 둘 다 짙어도 색이 달라 구분(§5.1) + 전투가 위라 응급 신호 우세 — 게임적으로 타당.
- **도면 토스트 최상위**: 순간 전경 알림이 배경 톤을 안 가림(§5.4). 사망 암전도 최상위(모든 위협을 덮는 종결 연출).
- 안개는 백로그(§11 날씨)지만 z-order 자리를 미리 비워둠 — 향후 추위와의 합성 규칙용.

### 5.5 도면 해금 발견 연출 — 건조한 보상 (G-9)

도면 기획 §11 UI 시각 의도를 **연출 타이밍·정서로** 재확정.

- **형태**: 절제된 토스트 한 줄. 풀스크린 팝업·파티클·파팡파레 금지(도면 기획 §7).
- **문구 톤**: "○○ 도면을 익혔다" 류 건조한 1줄. 느낌표 남발 금지. "버려진 잔해에서 건진 지식"의 톤.
- **타이밍**: 줍는 즉시 → 짧게(3초 내외) → 자동 소멸. 입력 차단 없음(탐험 흐름 끊지 않음).
- **제작창 연동**: 해금 직후 해당 칸이 회색/실루엣 → 정상색 전환 + **테두리 한 번 반짝**(과하지 않게). 잠긴 칸은 숨기지 말고 계속 보이게 — "보이지만 못 만든다"의 갈증이 탐험 동기(도면 기획 §11.2).
- **정서**: 보상이되 외롭고 건조하게. 어두운 화면에 잠깐 스민 따뜻함이 아니라, 차가운 지식의 획득. 화사한 골드/네온 금지.

## 6. 콘텐츠 분량 (수치 포함)

연출 단위로 센 분량 — 신규 콘텐츠가 아니라 **연출 상태 수**.

| 연출 축 | 연출 상태 수 | 세부 |
|---|---|---|
| 추위 오버레이 (G-2) | **3 단계 상태** + 회복 트랜지션 1 | 1/2/3단계 + 걷힘 |
| 낮밤 환경 톤 (G-3) | **4 페이즈 톤** + 전환 페이드 3 | 낮/황혼/밤/새벽 + 페이즈간 전환 |
| 도면 해금 연출 (G-9) | **토스트 1종** + 제작창 잠금/해금 상태 2 | 토스트 + 잠김/해금 표시 |
| 합성 우선순위 규칙 | **3 케이스** | §5.4 표 |

- 전환 타이밍 권장: 추위 단계 전환 ~1초 페이드 / 낮밤 페이즈 전환 수 초 페이드(낮→황혼은 더 느리게) / 추위 3단계 펄스 1.5초 주기 / 해금 토스트 3초 표시.
- 위 타이밍은 **방향성 초기값**(§8 튜닝 대상). 정확 프레임·알파 곡선은 client/designer 협의.

## 7. 톤 가이드

세 연출을 관통하는 단일 톤 — 감사 §0 비주얼 정체성 준수.

- **2대 정서 축 위에**: 추움(청)↔따뜻함(주황 모닥불), 밤(위협)↔낮(빌린 시간). 모든 연출이 이 축 어딘가에 위치.
- **따뜻함은 희소**: 모닥불 주황은 추위 청색의 **대비 신호**로만. 추위 오버레이가 짙을수록 모닥불 빛이 더 강하게 느껴지게(대비 강화).
- **절제·침묵**: 화면을 정보로 채우지 않는다. 추위 3단계도 완전 불투명 금지, 해금도 토스트 한 줄. "조용한 위협"이 WES 톤.
- **점진·부드러움**: 모든 전환이 서서히. 갑작스러운 플래시·스팅 금지. 추위는 "서서히 파래지는 세상", 밤은 "천천히 물드는 하늘".
- **질감**: 거칠고 닳음 — 서리 결정, 먹구름, 바랜 양피지. 깔끔/미니멀/네온 금지.
- **외로움**: 보상(해금)조차 건조하게. 따뜻한 축하가 아니라 "혼자 건진 지식". 밤의 어둠은 낭만이 아니라 고립.

## 8. UI 시각 의도 (디자이너 → 와이어프레임/자산 변환용)

> 본 섹션은 디자이너가 자산·와이어프레임으로 변환할 시각 의도 가이드. 톤·색·타이밍·레이아웃 방향 제시(픽셀/코드 스펙 아님).

### 8.1 추위 오버레이 (G-2) — 색·알파·타이밍

| 단계 | 색조 방향 | 알파 방향 | 형태 | 추가 연출 |
|---|---|---|---|---|
| 1단계 경고 | 청색 `(0.1, 0.3, 0.8)` 계열 | ~0.15 | 화면 가장자리 비네트(중앙 깨끗) | 정적, 펄스 없음 |
| 2단계 약틱 | 짙은 청색 | ~0.30 | 비네트가 안쪽으로 번짐 + 외곽 어두워짐 | 정적 |
| 3단계 강틱 | 흰빛 서리 `(0.8, 0.9, 1.0)` 계열 | ~0.50~0.60 | 화면 전체 서리 + 가장자리 성에 결정 | 1.5초 주기 느린 펄스 |

- **레이아웃**: 화면 가장자리→중앙으로 번지는 비네트 구조. 중앙 시야는 마지막까지 일부 남겨 "조작 가능"의 여백 유지(완전 차폐 금지).
- **전환**: 단계간 ~1초 페이드. 회복 시 약간 빠르게 걷힘.
- **색 약속 고정**: 청색 계열 전용. 적색(전투)과 절대 안 섞임(§5.1).

### 8.2 낮밤 환경 톤 (G-3) — 페이즈 색·전환

| 페이즈 | 환경/하늘 톤 방향 | 분위기 키워드 |
|---|---|---|
| 낮 | 빛바랜 회청색 (흐린 날, 쾌청 금지) | 평온, 바닥 긴장 |
| 황혼 | 짙은 붉은 주황, **저채도**(불길한 경고색, 예쁜 노을 금지) | 압박, 불안 |
| 밤 | 어두운 남색/검정 (먹구름, 별·달 거의 없음) | 고립, 외로움 |
| 새벽 | 희뿌연 회색 → 낮 톤으로 복귀 | 차가운 안도 |

- **전환 타이밍**: 페이즈간 점진 페이드. **낮→황혼이 가장 느리게**(어두워지는 걸 눈치 못 채게 = 불완전 정보).
- **레이아웃**: 화면 전역 톤 그레이딩이 주 채널. HUD 페이즈 아이콘 1개는 보조(우상단 등 비침습 위치). 디지털 시계 금지.
- **모닥불/횃불**: 밤 톤 위에서 주황 광원이 "유일한 안전 색"으로 도드라지게.

### 8.3 도면 해금 토스트 (G-9) — 색·타이밍·레이아웃

- **위치**: 화면 상단 또는 중앙 하단의 절제된 토스트 한 줄. 풀스크린/중앙 모달 금지.
- **색상**: 바랜 양피지색 아이콘 + 무채색/저채도 배경. 화사한 골드·네온 금지.
- **타이밍**: 줍는 즉시 표시 → 3초 내외 → 자동 페이드 아웃. 입력 차단 없음.
- **제작창(CraftPopup) 잠금/해금**:
  - 잠긴 칸: 회색/실루엣 + 자물쇠 또는 "도면 필요" 라벨. **숨기지 않음**(목록 재배치 금지, 오버레이 레이어로만).
  - 해금 칸: 정상 컬러 전환 + 테두리 한 번 반짝(절제).

### 8.4 톤 일관성 고정점 (designer 우선 고정)

1. **추위 청색 ↔ 전투 적색**의 2색 위험 약속 — 절대 안 섞임.
2. **모닥불 주황 = 유일한 따뜻함** — 추위/밤 톤 위에서 희소 안전 신호로만 도드라짐.
3. **모든 전환 = 점진 페이드** — 스냅 전환·플래시·스팅 금지가 WES 톤의 기준.

## 9. 밸런스/튜닝 포인트

연출은 밸런스를 안 건드리지만, **연출 강도**가 체감 난이도에 영향을 준다.

| 파라미터 | 초기 방향 | 튜닝 방향 |
|---|---|---|
| 추위 오버레이 최대 알파(3단계) | 0.50~0.60 | 너무 짙으면 조작 불가(불공정), 너무 옅으면 위협 전달 실패. 0.50~0.65 범위 |
| 추위 단계 전환 페이드 | ~1초 | 너무 빠르면 스냅 느낌, 너무 느리면 단계 변화 인지 실패 |
| 낮→황혼 전환 속도 | 가장 느리게 | 느릴수록 불완전 정보↑(어두워지는 줄 모름). 단 너무 느리면 황혼 시작을 영영 못 알아챔 |
| 밤 환경 톤 어둠 정도 | 시야 40% 연동 | 시스템 시야값과 색 어둠을 분리 — 색이 너무 어두우면 시야값과 무관하게 안 보임 |
| 해금 토스트 지속 | 3초 | 짧으면 놓침, 길면 화면 점유. 2~4초 |
| 합성 시 추위 우선 강도 | 밤 톤 위 청색 우세 | 밤 남색과 추위 청색이 너무 비슷하면 단계 변화 인지 실패 — 채도/명도로 구분 보장 |

> **핵심 튜닝 목표**: "위협은 명확히 전달되되 조작은 끝까지 가능하다." 연출이 정보를 주되 게임을 막지 않는다.

## 10. 엣지 케이스 (게임플레이 차원)

| 케이스 | 연출 처리 방향 |
|---|---|
| 밤 + 추위 3단계 + 저체력(전투) 동시 | 청(추위)·적(전투) 두 채널이 동시에 떠도 **서로 다른 색**이라 구분 가능해야(§5.1). 둘 다 짙으면 화면이 위험으로 가득 — 의도된 절박함이되 조작 여백은 유지 |
| 황혼 전환 중 도면 발견 | 환경 톤 전환과 토스트가 겹침 — 토스트는 전경 순간 알림이라 배경 전환을 안 가림(§5.4) |
| 모닥불 옆에서 밤을 남(추위 0 유지) | 추위 오버레이 없음. 밤 환경 톤만 — "안전하지만 외로운 밤"의 정서. 모닥불 주황이 청색 부재 속에 따뜻함을 채움 |
| 멀티: 한 명만 추위 3단계 | 추위는 개인 수치(추위 기획 §9). 각 플레이어 화면에 자기 오버레이만. 동료 화면은 안전할 수 있음 — 협동의 비대칭 긴장 |
| 같은 종류 도면 재획득(이미 해금) | 토스트 안 띄움 또는 "이미 익힌 도면" 절제 표기. 중복 보상감 인플레 방지(도면 기획 §9) |
| 색약 플레이어 | 추위(청)·전투(적) 채널이 색에만 의존하면 구분 실패 위험 — **형태 차이**(추위=서리/외곽 비네트, 전투=심박 비네트)로도 구분되게 이중화 권장 |

- **사운드 연동(스케줄 G)**: 연출의 절반은 소리. 추위 호흡음·황혼 바람 고조·해금 종이 소리 — 사운드 기획과 짝. 본 기획은 시각만, 사운드는 별도 호출.
