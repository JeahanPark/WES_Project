## 0. 인벤토리 대조 핵심 결론

| 사실 | 내용 |
|---|---|
| **현 UI = 전부 그레이박스** | 모든 Popup/HUD가 Unity 내장 sprite(UISprite 단색 패널)만 사용. 다크그레이(0.196 회색·0.2 alpha)로 기능만 동작. |
| **실 커스텀 sprite 2건뿐** | `lock_overlay.png`, `blueprint_highlight_frame.png` (blueprint 시스템용). 나머지 배경·로고·프레임·게이지스킨·버튼 전무. |
| **ItemIcon만 충실** | AI 생성 완료 ~20종(wood/stone/herb/leather/bone/ironore/sword/ironsword/shield/leatherarmor/torch/bandage/potion_hp/potion_cold/campfire/waterskin/rope/stone_chunk + blueprint 3종). 라인/팔레트 통일성은 G-5에서 재점검 필요. |

**생성 방식 표기**:
- **AI텍스처** = Gemini로 2D 생성 (0단계 최우선). 배경·로고·프레임·스킨·아이콘.
- **Procedural** = Texture2D/머티리얼/셰이더 코드. 단색 오버레이·그라데이션·비네팅·환경톤.
- **차용** = Synty/polyperfect 기존 자산 복제·변형.
- **코드영역(client)** = 리소스 자체보다 셰이더/런타임 로직 비중. 디자이너는 머티리얼/텍스처만 제공, 와이어링은 client.

## 8. director 확인 필요 / 메모

- **G-2 추위 오버레이, G-3 낮밤 톤**: 단색 청/그라데이션 텍스처는 디자이너가 생성하나, 단계 전환·펄스·포스트프로세스 합성은 **client 코드영역**. 리소스 인터페이스(머티리얼 슬롯명)만 디자이너가 제공.
- **G-5 아이콘 통일성**: 기존 ItemIcon ~20종이 이미 AI 생성됨 → 신규 아이콘은 동일 스타일 프리픽스로 이어 생성(세션 일관성). 기존 세트 라인/팔레트가 ★2 기준선이 되는지 director 1회 확인 권장.
- **9-slice 다수**: 버튼/슬롯/프레임은 9-slice 테두리 텍스처로 생성해 스케일 자유도 확보(개별 크기 생성 회피).
