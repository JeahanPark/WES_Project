## 0. 전제 — WES의 비주얼 정체성

이 감사의 모든 시각 판단 기준.

| 항목 | 방향 |
|---|---|
| **톤** | 어둡고 외로운 야생 생존. 난파 후 버려진 섬. 희망은 멀리 있고(반대편 마을), 지금은 춥고 위험하다. |
| **레퍼런스** | Don't Starve(고딕 손그림·흑백 잉크 톤), Escape from Duckov(그리드 인벤토리 UX), 워크3 야생탈출 UMS(생존 긴박감) |
| **컬러 베이스** | 저채도·어두운 청록/회갈색. 따뜻함(모닥불 주황)은 **희소한 안전 신호**로만 등장 → 대비로 더 강하게 느껴지게 |
| **금지** | 밝고 명랑한 카툰, 네온, 채도 높은 RPG식 UI, 깔끔한 SF/미니멀 톤 |
| **핵심 정서 축** | **추움 ↔ 따뜻함**, **밤(위협) ↔ 낮(빌린 시간)**. 모든 화면이 이 두 축 어딘가에 위치해야 함 |

## 1. Intro 단계

플레이어가 게임을 켜고 처음 보는 화면. **첫인상이 톤 전체를 약속한다.** 현재 LoginPopup만 존재 — 기능적이지만 정서 전달이 비어 있음.

### 디자인 필요 지점

| # | 지점 | 현재 상태 | 우선순위 | WES 톤 시각 의도 |
|---|---|---|---|---|
| I-1 | **타이틀 배경 이미지** | 없음/단색 추정 | **꼭 필요** | 폭풍 친 뒤 새벽 해변, 부서진 부두·썩은 어망 잔해, 멀리 불 꺼진 등대. 저채도 청회색, 안개. 스토리 첫 문장("검은 파도에 삼켜지듯")의 시각화. 정적이되 외로움이 깔린 화면 |
| I-2 | **게임 로고** | 없음/텍스트 | **꼭 필요** | "야생 생존 탈출" 로고. Don't Starve식 거칠고 긁힌 손글씨/판각 느낌. 흰빛 서리 또는 빛바랜 흰색. 깔끔한 산세리프 금지 |
| I-3 | **START / OPTIONS 버튼 스타일** | 기본 UI 추정 | **꼭 필요** | 나무판·밧줄·금속못 모티프의 거친 버튼. 호버 시 주황 불빛이 스며드는 느낌(따뜻함=상호작용 신호). 톤 일관성의 기준점 |
| I-4 | **로그인/입력 필드 스킨** | 기본 인풋 | 있으면 좋음 | 닉네임/접속 입력. 양피지·낡은 명판 위 글씨 느낌. 기능 화면이라 과투자 불필요하나 버튼과 톤은 맞춰야 함 |
| I-5 | **앰비언트 연출 단서** | 없음 | 있으면 좋음 | 바람·파도 소리(사운드 팀), 시각적으로는 안개 흐름·등대 너머 희미한 빛 깜빡임 정도. 정적인 화면에 "살아있는 외로움" 부여 |

**Intro 한줄 의도**: 켜는 순간 "여긴 따뜻한 게임이 아니다"가 전달돼야 한다.

## 2. Lobby 단계

방을 만들고 동료를 기다리는 공간. **혼자/함께의 정서**가 이 화면의 주제. 너무 게임 메뉴처럼 차갑지 않게 — 곧 함께 떠날 동료들이 모이는 곳.

### 디자인 필요 지점

| # | 지점 | 현재 상태 | 우선순위 | WES 톤 시각 의도 |
|---|---|---|---|---|
| L-1 | **로비 배경** | 없음/단색 추정 | **꼭 필요** | 모닥불 가에 모인 생존자들의 야영지 느낌, 또는 출항 직전 부두. 따뜻함(모닥불 주황)이 처음 등장하는 화면 — 인트로의 추위와 대비. "함께라서 조금 덜 춥다" |
| L-2 | **플레이어 슬롯 (최대 4)** | 기능만 추정 | **꼭 필요** | 빈 슬롯=비어있는 침낭/모닥불 빈 자리(외로움), 채워진 슬롯=실루엣+닉네임. 캐릭터 선택 없으므로 동일 기본 캐릭터의 색/번호 구분만 |
| L-3 | **Ready / Start 버튼 상태** | 기본 UI | **꼭 필요** | Ready=불씨 붙는 느낌(꺼짐→켜짐), Start(방장 전용)=출발 신호. 방장/비방장 권한 차이가 시각적으로 명확해야(비활성 버튼 톤) |
| L-4 | **방 생성/참가 진입 UI** | 기본 UI | 있으면 좋음 | 방 목록·코드 입력. 낡은 게시판·항해 일지 모티프. 인트로/인게임과 버튼 톤 통일이 핵심 |
| L-5 | **로딩/매칭 대기 연출** | 없음 추정 | 있으면 좋음 | 게임 진입 전환. 지도가 펼쳐지며 목표 마을 표식이 보이는 연출 → 스토리의 "단 하나의 표식" 환기. 동기 부여 Beat |

**Lobby 한줄 의도**: 인트로의 추위 다음에 오는 **희소한 온기** — 동료가 모이는 잠깐의 안도.

## 3. Ingame 단계

플레이 시간의 대부분. 여기 HUD/UI는 **장식이 아니라 생존 정보**다. 가독성 최우선이되, 정보 하나하나가 톤을 입어야 한다. 특히 **추위·낮밤·도면해금**이 코어 텐션이므로 그 피드백 비주얼이 빈약하면 게임의 핵심 재미가 전달 안 됨.

### 3-A. 상시 HUD

| # | 지점 | 현재 상태 | 우선순위 | WES 톤 시각 의도 |
|---|---|---|---|---|
| G-1 | **PlayerStatusHUD (HP/Cold 바)** | 기능 동작 | **꼭 필요** | HP=마른 피/녹슨 붉은색, Cold=서리 푸른색. 깔끔한 게이지 금지 — 거칠고 닳은 프레임. Cold가 차오를수록 바 주변에 서리 결정이 끼는 표현 |
| G-2 | **화면 추위 오버레이 (Cold 단계)** | 기획 확정 [[밤_추위_데미지]] | **꼭 필요** | 1단계 청색 알파 0.15 → 2단계 짙은 청색 0.30 → 3단계 흰빛 서리 0.5~0.6 + 펄스. **게임의 죽음 직전 정서를 만드는 핵심 연출**. 화면 가장자리 성에/혈관 수축 비네팅 |
| G-3 | **낮밤/시간대 환경 톤** | 기획 확정 [[시간_낮밤_시스템]] | **꼭 필요** | 낮(빛바랜 청회) → 황혼(주황·보라 경고색) → 밤(짙은 남색·시야 축소) → 새벽(차가운 안도). **디지털 타이머 금지** — 하늘색/조명만으로 시간 인지. 황혼=귀환 압박 신호의 색 |
| G-4 | **QuickSlotHUD (하단 8칸)** | 기능 동작 | **꼭 필요** | 닳은 가죽 파우치·나무 슬롯 8칸. 선택 슬롯 하이라이트=주황 불빛. 빈 슬롯과 채워진 슬롯 대비 명확. 키 1~8 각인 |
| G-5 | **아이템 아이콘 세트 (15종+)** | 부족 추정 | **꼭 필요** | 자원/소비/장비/건물 전 아이템 아이콘. Don't Starve식 손그림·잉크 라인. 자원(나무·돌·약초·가죽·뼈·철광석)부터 우선. 통일된 라인 두께·팔레트 |
| G-6 | **CraftHUDTab (제작 진입 탭)** | 기능 동작 | 있으면 좋음 | HUD에서 제작 여는 탭/버튼. 모루·작업대 모티프 아이콘 |

### 3-B. 팝업/오버레이

| # | 지점 | 현재 상태 | 우선순위 | WES 톤 시각 의도 |
|---|---|---|---|---|
| G-7 | **InventoryPopup (그리드)** | 기능 동작 | **꼭 필요** | Escape from Duckov식 그리드 UX. 비주얼은 낡은 배낭 내부·천 칸막이. 드래그 중 슬롯 하이라이트, 장착 슬롯 구분. 차갑고 실용적이되 거친 질감 |
| G-8 | **CraftPopup (레시피·재료·상세)** | 기능 동작 | **꼭 필요** | 레시피 카드=낡은 도면/설계도 종이. 재료 보유/부족을 색으로(충분=흐린 흰색, 부족=빛바랜 적색). 제작 버튼=불씨 |
| G-9 | **도면해금 발견 연출** | 기획 확정 [[도면해금]] | **꼭 필요** | 월드에서 도면 습득 시 "새 제작 해금" 피드백. 테크=옵션을 사는 코어 분기이므로 발견의 보상감이 시각적으로 분명해야. 종이 펼쳐지는 토스트+잠금 해제 표현 |
| G-10 | **BlueprintToast / 알림 토스트** | 기능 추정 | 있으면 좋음 | 해금·획득·경고 알림. 화면 점유 최소, 양피지 쪽지가 잠깐 떴다 사라지는 톤 |
| G-11 | **WorldUI (월드 체력바·상호작용)** | 기능 동작 | 있으면 좋음 | 몬스터 머리 위 체력바, 수집/상호작용 프롬프트. 월드에 떠도 톤 깨지지 않게 — 가는 거친 라인, 저채도 |
| G-12 | **건물 배치 모드 인디케이터** | 기능 동작 (반경 1.5m 녹색 원) | 있으면 좋음 | 배치 프리뷰·범위 표시. 녹색=가능/적색=불가의 거친 룬/마킹 느낌. 기본 유니티 원형 금지 |

### 3-C. 위협/사망 피드백

| # | 지점 | 현재 상태 | 우선순위 | WES 톤 시각 의도 |
|---|---|---|---|---|
| G-13 | **피격/저체력 화면 비네팅** | 없음 추정 | **꼭 필요** | HP 위급 시 적색 비네팅·심박. 추위 오버레이와 구분되는 "전투 위협" 채널. 외로운 생존의 절박함 |
| G-14 | **플레이어 사망/다운 연출** | 없음 추정 | 있으면 좋음 | 화면 암전·탈색·서리. 협동에서 동료 다운 표시(멀리 쓰러진 동료 마커). 죽음이 가볍지 않게 |

**Ingame 한줄 의도**: HUD는 정보지만 **모든 정보가 "지금 얼마나 위험한가"를 톤으로 말해야** 한다. 추위 오버레이와 낮밤 톤이 빈약하면 코어 텐션이 죽는다.

## 4. Clear 단계

세션의 끝. 탈출 성공(목표 도달) 또는 전멸. **저장 없는 세션**이라 이 화면이 "한 판의 의미"를 봉인한다. 결과를 건조하게 보여주되, 생존의 무게가 남게.

### 디자인 필요 지점

| # | 지점 | 현재 상태 | 우선순위 | WES 톤 시각 의도 |
|---|---|---|---|---|
| C-1 | **ResultPopup — 탈출 성공 화면** | 기능 동작 | **꼭 필요** | 반대편 마을 도착=스토리의 "단 하나의 표식" 완수. 멀리 불 켜진 마을·따뜻한 빛(인트로 꺼진 등대와 대비). 안도하되 화려하지 않게 — 살아남았다는 조용한 무게 |
| C-2 | **ResultPopup — 전멸(게임오버) 화면** | 부족 추정 | **꼭 필요** | 섬에 삼켜짐. 흑백·서리·암전. 차갑고 외롭게. 좌절감을 주되 "다시"를 유도(세션형). 성공 화면과 명확히 다른 정서 |
| C-3 | **결과 요약 비주얼** | 부족 추정 | 있으면 좋음 | 탈출 인원/생존 시간/도달 거리 등 한 판 기록. 낡은 항해 일지에 기록되는 톤. 숫자가 차갑게 박히되 톤 유지 |
| C-4 | **로비 복귀 전환 연출** | 없음 추정 | 있으면 좋음 | 결과 → 로비 전환. 지도가 접히고 다시 야영지로 돌아가는 느낌. 다음 판으로의 리듬 |

**Clear 한줄 의도**: 성공은 **조용한 온기**, 실패는 **차가운 침묵**. 둘의 정서 대비가 명확해야 세션형의 무게가 산다.

## 6. UI 시각 의도 요약 (designer 전달용)

- **컬러 베이스**: 저채도 청회·회갈. 따뜻함(주황)은 희소 안전 신호로만.
- **2대 정서 축**: 추움(청)↔따뜻함(주황), 밤(위협)↔낮(빌린 시간). 모든 화면이 이 축 위에.
- **질감**: 거칠고 닳음 — 나무·밧줄·가죽·녹·서리·낡은 종이. 깔끔/미니멀/네온 금지.
- **로고·텍스트**: Don't Starve식 긁힌 판각/손글씨.
- **상호작용 신호**: 호버/선택은 주황 불빛이 스미는 느낌.
- **HUD 위험 채널 2종 분리**: 추위(청색 서리 오버레이) vs 전투(적색 비네팅).
- **낮밤은 하늘/조명 톤으로만** — 디지털 타이머 금지.
- **결과 화면 정서 대비**: 성공=조용한 온기, 실패=차가운 침묵.
