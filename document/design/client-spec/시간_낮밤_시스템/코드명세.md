## 2. 재활용 가능한 기존 시스템

| `PlayerCharacter.m_Cold / SetCold()` | Cold 감소율 적용 시 그대로 사용. 감소량 계산만 외부에서 수행 |

## 3. 신규 필요 클래스/스크립트

### 3-1. `DayNightWorker` — 핵심 페이즈 타이머 (신규, Worker)

- **위치**: `Assets/Scripts/Worker/DayNightWorker.cs`
- **역할**: 페이즈 타이머 진행, 페이즈 전환 이벤트 발행, `NetworkVariable<int> m_Phase` 동기화
- **권한**: 서버에서 타이머 진행 및 페이즈 결정. NetworkVariable로 모든 클라에 동기화
- **주요 시그니처 (가이드)**:
  ```
  public static event Action<DayPhase, DayPhase> OnPhaseChanged;  // (이전, 현재)
  public DayPhase CurrentPhase => (DayPhase)m_Phase.Value;
  public float GetColdRateMultiplier()  // 현재 페이즈의 Cold 감소율 배수
  ```

### 3-2. `DayNightRenderWorker` — 클라이언트 시각/사운드 전환 (신규, Worker)

- **위치**: `Assets/Scripts/Worker/DayNightRenderWorker.cs`
- **역할**: `OnPhaseChanged` 구독 → 하늘색 Lerp, 시야 마스크 강도, Directional Light 강도, 앰비언스 트랙 전환, 진입 SE 재생
- **권한**: 클라이언트 전용 (시각/사운드는 동기화 불필요)
- **의존**: `DayNightWorker.OnPhaseChanged`, Unity `RenderSettings`, `AudioSource`

### 3-3. `DayNightConfig` — 페이즈 수치 데이터 (ScriptableObject)

- **위치**: `Assets/GameResource/Config/DayNightConfig.asset`
- **판단**: ScriptableObject로 시작. 튜닝 빈도가 높은 수치(지속 시간, Cold 배수, 시야 배수)는 Inspector에서 직접 조정 가능. 추후 CSV 이전 가능

### 3-4. `NightVisionComponent` — 시야 마스크 처리 (신규, Component)

- **위치**: `Assets/Scripts/Component/NightVisionComponent.cs`
- **역할**: 야간 시야 마스크(어둠 오버레이) 렌더링. UI 오버레이 마스크 방식(옵션 A) 확정
- **구현 방식**: 전체 화면 반투명 검정 Image + 모닥불/횃불 스크린 좌표 기반 원형 컷아웃
- **슬라이스**: 슬라이스 2에서 구현

### 3-5. `NightMonsterComponent` — 야간 몬스터 전용 행동 (신규, Component)

- **위치**: `Assets/Scripts/Component/NightMonsterComponent.cs`
- **역할**: 모닥불 회피 반경 감지, 야간/새벽 페이즈에서만 활성화, 어둠 접근 패턴
- **의존**: `DayNightWorker.CurrentPhase`, `WorldBuildingObject` 모닥불 위치 목록

## 5. 네트워크 동기화 항목

| 항목 | 방식 | 권한 |
|------|------|------|
| 현재 페이즈 (`m_Phase`) | `NetworkVariable<int>` in `DayNightWorker` | 서버 Write / Everyone Read |
| 페이즈 전환 이벤트 | `NetworkVariable.OnValueChanged` → 클라 이벤트 발행 | 자동 (NetworkVariable 콜백) |
| Cold 감소 적용 | 서버에서만 `PlayerCharacter.SetCold()` 호출 | 서버 전용 |
| 시야/렌더링 | 동기화 불필요 — 클라이언트 각자 페이즈 값 기반으로 처리 | 클라이언트 로컬 |
| 야간 몬스터 스폰 | 기존 `InGameSpawnWorker.SpawnObject()` 패턴 재사용 | 서버 전용 |
| 모닥불 위치 | `WorldBuildingObject`가 이미 NetworkObject로 동기화됨 — 추가 동기화 불필요 | 기존 |

## 6. 상태 머신 — 페이즈 FSM

```
[낮] ---(낮 종료)--> [황혼] ---(황혼 종료)--> [밤] ---(밤 종료)--> [새벽] ---(새벽 종료)--> [낮]
```

| 전이 | 조건 |
|------|------|
| 낮 → 황혼 | 서버 타이머 360초 경과 |
| 황혼 → 밤 | 서버 타이머 120초 경과 |
| 밤 → 새벽 | 서버 타이머 240초 경과 |
| 새벽 → 낮 | 서버 타이머 120초 경과 |
| 세션 시작 | 항상 낮(Day)에서 시작 |

**불가능한 전이** (코드에서 명시적 차단):
- 낮 → 밤 (황혼 건너뜀 금지)
- 역방향 전이 (밤 → 황혼 금지)
- GameState가 Playing이 아닐 때 페이즈 진행 중단

## 11. 엣지 케이스 (구현 차원)

| 케이스 | 처리 방향 |
|--------|-----------|
| 클라이언트 접속 중 페이즈 전환 발생 | `NetworkVariable.OnValueChanged`는 접속 후 초기값도 콜백 발생 → 자동 처리됨 |
| 늦게 접속한 클라이언트의 페이즈 동기화 | `NetworkVariable` 초기값이 현재 페이즈값 → 접속 즉시 정상 상태로 수신 |
| 호스트 디스커넥트 시 페이즈 타이머 정지 | 기존 NGO 동작과 동일. 별도 처리 불필요 |
| `GameState`가 Clear/GameOver일 때 페이즈 진행 | `DayNightWorker` Update에서 `InGameController.Instance.GameState == Playing` 조건 확인 필요 |
| 모닥불이 디스폰된 경우 회피 반경 목록 갱신 | `WorldBuildingObject`의 OnNetworkDespawn에서 회피 목록 해제 필요 |
| 모닥불이 여러 개 배치된 경우 | `NightMonsterComponent`가 모든 모닥불 위치를 순회하여 최근접 거리 체크 — 비용 낮음 |
| 야간 몬스터가 새벽으로 전환 시 디스폰 처리 | `OnPhaseChanged`(Night→Dawn) 수신 → 서버에서 야간 몬스터만 선별 디스폰 |
| 솔로 플레이 야간 균형 | 이 케이스는 구현이 아닌 수치 밸런스 문제. 구현 차원에서는 별도 처리 없음 |
| 황혼/새벽의 Cold 배수가 "서서히" 복귀 | DayNightWorker에서 새벽 구간 내 경과 시간 비율로 Lerp 처리 필요 (미세 구현 복잡성 있음) |
| 중복 페이즈 전환 호출 | 타이머는 서버 단일 진행이므로 중복 없음. 방어 코드는 `if (nextPhase == m_Phase.Value) return;` 1줄 |

## 12. 권고

### 권고 1 — 시야 마스크 구현 방식 선택 (디렉터 의견 필요)

세 가지 옵션이 있으며 비용 차이가 크다:

| 옵션 | 설명 | 비용 | 권고 |
|------|------|------|------|
| **A. UI 오버레이** | 반투명 검정 Image를 전체 화면 덮고, 모닥불/횃불 위치에 원형 마스크(UI Mask 또는 Shader) | M | 권고. 구현 명확하고 성능 안정적 |
| **B. Post Processing 비네팅** | 화면 가장자리 어둠만. 모닥불/횃불 주변 밝기 구현 어려움 | S | 디렉터의 "단순화 허용" 옵션과 일치. 가장 빠름 |
| **C. 3D 조명 기반** | Directional Light 끄고 Point Light (모닥불/횃불)만 켜는 방식 | XL | 씬 전체 조명 설계 변경 필요. 초기 구현 비권고 |

**권고**: 옵션 A(UI 오버레이 마스크)를 기본으로 구현. 디렉터가 단순화를 원하면 옵션 B로 대체.

### 권고 2 — 야간 몬스터 모닥불 회피 방식

NavMesh Obstacle 방식과 트리거 콜라이더 방식을 비교:

| 방식 | 설명 | 비용 |
|------|------|------|
| **NavMesh Obstacle** | `NavMeshObstacle` 컴포넌트를 모닥불에 동적 추가/제거 | M. NavMesh carving 성능 이슈 가능 |
| **트리거 콜라이더** | 모닥불 주변에 Sphere Trigger. `NightMonsterComponent`에서 회피 목표 조정 | S. 기존 Physics 패턴 재사용 |

**권고**: 트리거 콜라이더 방식. NavMesh carving은 런타임 비용이 불확실하고 현재 NavMesh 구조와 충돌 가능성 있음.

### 권고 3 — 1라운드 구현 범위 제안 (MVP 슬라이스) [확정됨]

디렉터 동의 완료. 아래 슬라이스 순서로 진행:

1. **슬라이스 1**: `DayNightWorker` (타이머 + 페이즈 동기화 + Cold 가속) → 핵심 기능 즉시 플레이어블
2. **슬라이스 2**: `NightVisionComponent` (UI 오버레이 마스크) + HUD 페이즈 아이콘 + `DayNightRenderWorker`
3. **슬라이스 3**: 야간 몬스터 스폰 분기 + `NightMonsterComponent` (모닥불 트리거 회피)
