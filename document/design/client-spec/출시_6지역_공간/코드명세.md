# 출시_6지역_공간 — 코드명세

기준 기획: `document/design/game-design/출시_6지역_공간/기획.md`
판정: 경계=좌표 임계값(a) 채택 / 맵=절차생성 확장(종단축 6구간) / 큰줄기 고정·지역 내부만 시드변주.
작성: client. 게임가치 판단 없음(director 영역).

**[확정] SetArea 기준 플레이어 = 선두(종단축 진행 깊이 최대인 살아있는 플레이어).** 개별 분포 불가(전역 1분포 의도 유지). 선두 진입 → 전원 그 환경 공유 → 뒤처진 인원에 뭉치기 압박. 다수결/호스트 기각. (director 확정, 기획서 §5.3)
**[확정] 경계 단일 진실원 = CSV(WorldAreaInfo AxisMin/AxisMax). MapGenerator가 그 값을 읽음. 이중 진실원 금지.** (기획서 §5.3)
**[확정] 펌프 = 좌표게이트 + 전원통과까지 관문 유지. 지형붕괴 연출은 백로그. 1차 맵 = 곧은 회랑 종단 + 키큰 지형벽 시야차폐, S/L자 굴곡은 2차 백로그.**
**[확정] m_CurrentAreaId = 전진만(단조증가 clamp). 선두 사망으로 선두가 얕은 인원으로 승계돼도 활성 지역 후퇴 안 함(도달한 가장 깊은 지역 환경 유지). 후퇴 허용 시 "선두 자살로 환경 리셋"이 최적플레이가 돼 비가역·후회 코어 위배. 펌프와 일관.** (director 확정, 기획서 §9) → **R2 기획 미결 0건.**

## 2. 재활용 가능한 기존 시스템

- **`MoveCostWorker.Update`**: 이미 서버측 `registry.GetAlivePlayers()`를 매 프레임 순회. 종단축 구간 판정을 이 순회에 얹으면 신규 추적 루프 불필요. (단 SRP상 별도 Worker로 분리 권장 — §3 참조)
- **`WeatherWorker.SetArea(int)` / `MoveCostWorker.SetArea(int)`**: 서버권위, 이미 구현됨. 호출만 신설하면 됨.
- **`WorldAreaInfo` / `WorldAreaWeatherInfo` / `WorldAreaMonsterInfo`**: AreaId 키 구조. 6지역=행 추가만, 코드 0 (Weather/Monster). WorldAreaInfo만 컬럼 추가.
- **`InGameAreaWorker` + `MonsterSpawnArea`**: AreaId별 몬스터 스폰 그대로 재사용. 6지역분 MonsterSpawnArea를 씬에 추가 배치(level-design)하면 됨.
- **`DayNightWorker.OnPhaseChanged` 이벤트 패턴**: 지역전환 이벤트도 동일 static event 패턴으로 발신(`OnAreaChanged`).
- **`InGameController` Worker 와이어링**: `[SerializeField]` Worker 슬롯 + `Initialize()` 호출 패턴 동일하게 신규 Worker 등록.
- **`PopupManager`/토스트**: 지역명 표시는 기존 UI 알림 경로 재사용(신규 팝업 만들지 않음).

## 3. 신규 필요 클래스/스크립트

### 3.1 `InGameAreaBandWorker` (신규, Worker)
- 위치: `Assets/Scripts/Worker/InGameAreaBandWorker.cs`
- 역할: 서버 전용. 종단축(예 Z) 임계값으로 **선두 플레이어**(살아있는 플레이어 중 종단축 진행 깊이 최대)의 현재 지역을 판정 → 전역 SetArea. [확정] 개별 아님, 전역 1분포. 선두 = `max(GetAlivePlayers() 의 종단축 좌표)`.
- 부착: `DayNightWorker` GameObject(WeatherWorker/MoveCostWorker와 동일 오브젝트 — SetArea 타깃과 같은 곳).
- 핵심:
  - `NetworkVariable<int> m_CurrentAreaId` (서버쓰기/전원읽기) — 현재 활성 지역.
  - 매 프레임(또는 N틱마다) 기준 플레이어 종단좌표 → `ResolveAreaId(z)` → 변경 시 `WeatherWorker.SetArea` + `MoveCostWorker.SetArea` + `OnAreaChanged` 발신.
  - 히스테리시스(§Q3): 경계마다 데드존 폭 `m_BandHysteresis`. 직전 지역 캐싱 후, 경계를 데드존 이상 넘어야 전환.
- 메서드 가이드(시그니처 수준):
  - `public void Initialize()` — InGameController가 호출.
  - `private int ResolveAreaId(float _axisPos)` — CSV 경계좌표로 구간 매핑.
  - `private void ApplyAreaChange(int _newAreaId)` — SetArea×2 + 이벤트.

### 3.2 `AreaGateComponent` (신규, Component) — 관문 펌프(일방향)
- 위치: `Assets/Scripts/Component/AreaGateComponent.cs`
- 역할: 관문 5~6개 각각에 부착. 통과 후 역행 차단(펌프). 서버권위.
- 구현 권고 = **좌표게이트 푸시백** (지형 붕괴보다 저비용):
  - 관문 종단좌표 `m_GateAxis`, 통과 방향 고정.
  - 서버가 "이미 통과한 플레이어"가 역방향으로 게이트 좌표를 넘으려 하면 위치를 게이트 앞으로 clamp(서버 NetworkTransform 보정) 또는 보이지 않는 BoxCollider 벽 활성화.
  - **전원통과 유지(Q2)**: `m_OpenUntilAllPassed=true`면 관문 직전 플레이어 전원이 통과할 때까지 벽 비활성(뒤처진 인원 통과 허용), 전원 통과 후 벽 활성. 통과상태 = `HashSet<int> m_PassedPlayers`.
- 비용: 좌표게이트 자체 S, 전원통과 추적 M.

### 3.3 (선택) `InGameMapGenerator` 종단화 — 기존 MapGenerator 개편
- 위치: 기존 `Assets/Scripts/Editor/MapGenerator.cs` 개편 (신규 파일 아님).
- §5(상태) 아님 — §9 구현단계 참조.

## 4. 데이터 레이어 (CSV / 상수)

### 4.1 WorldAreaInfo.csv — 컬럼 신설 + 6행
현재: `Id,Name,MaxCount,RespawnDelay,MoveCostMultiplier` (3행)
제안 헤더 추가: `AxisMin.FLOAT,AxisMax.FLOAT` (종단축 구간 경계)

```
Id,Name,MaxCount,RespawnDelay,MoveCostMultiplier,AxisMin,AxisMax
1,해안가,...,1.0,<beachMin>,<beachMax>      # 넓고 짧음
2,숲,  ...,1.0,...
3,산지,...,1.2,...                          # 최장
4,설원,...,~2.5,...                          # ④이동비용 충돌 지역
5,폐허,...,...                               # 좁고 압축
6,<6번째 지역>,...
```
- `AxisMin/AxisMax`: level-design 페이싱 튜닝용 (면적/체류시간). **단일 진실원 문제는 §Q1 참조** — MapGenerator는 에디터 도구라 InfoManager 미로드. 권고: CSV를 진실원으로 두되 MapGenerator가 빌드 시 CSV를 직접 파싱(또는 const 자동생성)하는 경로 추가. 미도입 시 CSV(런타임 판정용)와 MapGenerator const(지오메트리용)가 이중 — 둘이 어긋나면 지역명과 실제 날씨/이동비용 경계가 불일치(버그). **반드시 한쪽으로 통일 필요.**
- WorldAreaInfo.cs는 자동생성 partial — CSV 헤더에 컬럼 추가하면 재생성 시 `public float AxisMin; public float AxisMax;` 자동 추가. 수동 편집 불필요/금지.

## 6. 상태 머신 (지역 전이)

- 유효 상태: AreaId 1~6 + 마을(종착, 별도 AreaId 또는 7).
- 가능 전이: **m_CurrentAreaId d → d+1만 (전진 전용, 단조증가).** [확정] 후퇴 없음.
- **불가능 전이(차단)**: d → d-N 후퇴 일체 차단 (선두 사망 승계로 선두가 얕아져도 활성지역 유지). 지역 스킵 d → d+2 (선형 종단 지오메트리상 물리적 불가; 단 선두가 데드존 통째로 건너뛸 만큼 빠르면 d+2 점프 가능 → ResolveAreaId는 선두 절대좌표 기반이라 자연 허용, 차단 불요).
- 히스테리시스: 경계 데드존 안에서는 직전 AreaId 유지(전이 보류). 단조증가와 결합 — 데드존+단조 둘 다 적용.

## 11. 엣지 케이스 (구현 차원)

- **기준 플레이어**: [확정] 선두 = 살아있는 플레이어 중 종단축 진행 깊이 최대. [확정] **m_CurrentAreaId는 단조증가** — 선두 사망 승계로 선두가 얕아져도 활성지역 후퇴 안 함(ResolveAreaId 결과가 현재보다 작으면 무시). 살아있는 플레이어 0명이면 게임오버 경로라 판정 자체 무의미(GetAlivePlayers 빈 리스트 → no-op).
- **경계 연타(왔다갔다)**: 히스테리시스 데드존으로 해결(§4.4 Q3). 매우 간단.
- **디스커넥트/재접속**: m_CurrentAreaId는 NetworkVariable라 재접속 클라에 자동 동기. 푸시백 m_PassedPlayers는 재접속 시 PlayerIndex 키로 복원 필요(서버 유지 시 OK).
- **관문 데드존 안에서 디스커넥트**: 전원통과 카운트에서 빠진 인원 처리 — 연결끊긴 플레이어는 통과 카운트에서 제외(`GetAlivePlayers` 기준).
- **게임 시작 시 초기 AreaId**: 시작지점(해안가=1)로 명시 초기화. SetArea(1) 1회 강제.
- **빈 데이터(분포/몬스터 없는 지역)**: WeatherWorker는 ladder 비면 no-op(기존 가드 존재), MoveCost는 배수 1f fallback(기존). 안전.
- **중복 호출**: SetArea는 같은 값이면 m_AreaId만 갱신(부작용 없음). OnAreaChanged는 변경 시에만 발신(가드).
- **푸시백 vs 클라 이동예측 충돌**: 서버 위치보정이 클라 NavMeshAgent 예측과 싸우면 떨림 — 보이지 않는 콜라이더 벽 방식이 떨림 적음(권고).

## 12. 권고 (director에게)

1. **추가질문 답 (Q1~Q3)**:
   - **Q1(경계 CSV화)**: 컬럼 추가 자체는 S(자동생성). 단 **MapGenerator const와 이중 진실원**이 핵심 리스크. 권고 = CSV를 단일 진실원으로 두고 MapGenerator가 빌드 시 그 값을 읽게(또는 const 자동생성). 안 그러면 "지역명·날씨 경계 ≠ 실제 지형 경계" 버그. 통일 비용 S~M, **반드시 포함 권고.**
   - **Q2(펌프)**: 좌표게이트 푸시백/콜라이더벽 = S~M. 지형 붕괴(무너지는 갱도) = 비용 L+연출, 백로그 권고. 전원통과 유지(뒤처짐) = M, 가능. 종합 권고 = **좌표게이트로 펌프, 연출은 designer/level-design 백로그.**
   - **Q3(히스테리시스)**: (a)방식이라 데드존 1파라미터로 끝. 거의 무비용.
2. **단일 최대 비용 = MapGenerator 종단화(L).** 출시 일정 압박 시 단순화 옵션: 1차로 큰줄기만 일자 회랑(곧은 종단) + S/L자 굴곡은 2차. 시야 차폐(앞뒤 지역 안보임)는 굴곡 없이 "키 큰 지형벽"으로도 일부 달성 가능(level-design).
3. **기준 플레이어 정책(§11 첫 줄)은 명세 확정 전 director 답 필요** — 이것만 추측 불가.
