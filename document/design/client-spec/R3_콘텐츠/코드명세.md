# R3 — 콘텐츠 채우기 — 코드명세 (v0.2 — 확정)

> 상태: **director3 R3 스코프 기획.md(v0.2) 반영 / 확정**. 슬라이스 R3-A·B·C 매핑. director3가 client 4미결을 모두 결정·내 권고 수용(도구 quick win, Chase+Attack 2상태+BehaviorType enum, DetectRange=0 비파괴, leash≈영역반경×1.5, 피격 시 반격, 도면 중복무효).
> 상위 기획: `document/design/game-design/R3_콘텐츠/기획.md` (§5.2, §5.2.1)

## 0. 슬라이스 매핑 (구현 순서 = 가치/비용비)

| 슬라이스 | 코드 작업 | 비용 | 산출 게이트 |
|---|---|---|---|
| **R3-A (quick win)** | 도구 등급 실아이템(코드 0, ItemInfo/Craft 행) + 해변·숲(d0~d1) 최소 수치 컬럼(몬스터 ATK·드롭확률) | **S** | "효율 vs 전투" 첫 결정 작동 |
| **R3-B (행동 코어)** | MonsterInfo 행동컬럼 확장 + `MonsterPerceptionComponent` + `MonsterChaseState`/`MonsterAttackState` + BehaviorType=Pack/Charge(숲·늪) + leash + 피격반격 | **M** | 평화배회→실제위협, 무리/돌진 |
| **R3-C (확장)** | BehaviorType=Poison/Ranged/Stealth/WeatherBuff/Boss 분기 + 도면 월드스폰 경로 + 산~폐허(d3~d5) 수치(날씨분포·이동비용·도면확률) + 제작조건(날씨/지역) | **M+** | 탐험보상·게이트·보스·최종베팅 |

## 1. 영향 범위 요약

R3 콘텐츠는 3개 축. 각 축의 현재 코드 준비도(R1/R2 위) 가 크게 다르다.

| 축 | 현재 상태 | R3에서 client 코드 작업 | 슬라이스 |
|---|---|---|---|
| A. 몬스터 행동패턴 | FSM에 **추격·공격 상태 자체가 없음** (Idle/Walk/Hit/Death = 평화 배회만). MonsterInfo에 행동 파라미터 컬럼 없음 | **신규 — 비용 큼** | B(Pack/Charge)·C(나머지) |
| B. 도구등급 실아이템 | R1-T3 시스템(ToolTierSystem·채집배수·ItemInfo.ToolTier) **이미 연결됨**. 실아이템 행/레시피만 없음 | **CSV 위주 — 코드 거의 없음** | A |
| C. 도면 월드스폰 | 도면 **획득→해금** 경로는 연결됨(WorldDropItem). **자연 월드스폰(SpawnAreaId/SpawnChance) 경로는 미연결** | **신규 스폰 경로 — 중간** | C |

## 1.1 확정된 설계 결정 (director3 §5.2.1)

- **행동 종수·지역**: 총 6+보스, 슬라이스 분산. R3-B=무리(Pack)·돌진(Charge) 2종(숲·늪). R3-C=독·은신·공중·날씨강화·보스. 전부 `BehaviorType` enum 분기.
- **추격 leash**: 있음. 스폰영역 + DetectRange 이탈 시 배회 복귀. leash 거리 ≈ 영역반경 × 1.5. (무한추격 금지 — 도주=이동분기 전술 보존)
- **도면 중복**: 중복 무효 — 이미 해금된 종류는 효과없음/줍히지 않음. (`RecipeUnlockRegistry.Unlock`은 이미 멱등이므로 코드 무해. 월드스폰 단계에서 "이미 해금분은 스폰 후보 제외" 옵션 — §3-C)
- **평화몬스터 반격**: 피격 시 Chase 전환(반격). 단 DetectRange=0이라 먼저 안 덤빔. 비파괴 디폴트와 양립 — 기존 `OnDamaged` 훅에서 타깃 세팅 후 Chase 전이.

## 2. 재활용 가능한 기존 시스템

- **몬스터 FSM**: `MonsterStateMachine` + `MonsterStateBase`(Enter/Update/Exit) + `MonsterStateType` enum + `Dictionary<type,state>`. 신규 상태를 Dictionary에 추가하면 그대로 동작. `NavMeshAgent` 이동(`MoveTowardsTarget`/`SetDestination`)·서버권위 가드(`!IsServer return`) 재사용.
- **전투 경로**: `CharacterBase.TakeDamage(dmg, attacker)` → 서버RPC → DEF/크리 적용 → 사망. 몬스터→플레이어 공격은 이걸 그대로 호출하면 됨(신규 데미지 파이프 불필요).
- **애니메이션**: `AnimationType.Attack` enum **이미 존재**. `StateAnimationComponent.PlayAnimation(AnimationType.Attack)` 호출만.
- **드롭/채집**: `WorldEntityBase` 드롭 + `ToolTierSystem.GatheringMultiplier` 채집배수 이미 적용. 도구 실아이템(ToolTier>0)만 추가하면 즉시 활성.
- **도면 해금**: `WorldDropItem.AddItemClientRpc` 가 `GetBlueprintByItemId` → `RecipeUnlockRegistry.Unlock` 처리. 획득 경로 완비.
- **스폰 인프라**: `InGameSpawnWorker.SpawnObject<T>`/`DespawnObject`, `InGamePlayWorker.SpawnDropItem(itemId,count,pos)`, `InGameAreaWorker`의 area별 스폰/리스폰/주야 로직.
- **CSV 파이프라인**: `*Info.csv` → InfoConvert → Reflection 파싱. 컬럼 추가/행 추가는 데이터 작업(level-design).

## 3. 신규 필요 클래스/스크립트 (축별)

### A. 몬스터 행동패턴 (director 확정 후 범위 결정)
신규 상태 후보 (각각 `MonsterStateBase` 상속, `MonsterStateType`에 enum 값 추가, Dictionary 등록):
- `MonsterChaseState` — 타깃 플레이어 추격 (감지범위·이탈범위 필요). **다수 패턴의 공통 전제**.
- `MonsterAttackState` — 사거리 진입 시 `TakeDamage` 호출 + 쿨다운 + Attack 애니. 근접 공통.
- 패턴별 분기는 **신규 상태 대신 파라미터화 권장** (아래 §12):
  - 직선 돌진 = Chase의 변형(가속·직선 고정·충돌데미지)
  - 독 DoT = Attack의 후처리(피격 대상에 DoT 틱) — `ApplyEnvironmentDamage` 반복 재사용
  - 원거리 = Attack의 투사체 변형(투사체 프리팹 필요 → designer)
  - 광역강타 = Attack의 범위판정 변형(`Physics.OverlapSphere`)
  - 은신 = Idle/Chase에 가시성 토글(Renderer)
  - 무리 추격 = Chase 진입 시 같은 area 동료에게 타깃 전파(AreaWorker 경유 브로드캐스트)
- 신규 컴포넌트(필요 시): `MonsterPerceptionComponent`(감지/타깃선정, 서버전용) — Update마다 OverlapSphere로 플레이어 탐색. Chase/Attack의 입력원.

### B. 도구등급 실아이템
- **신규 코드 0**. ItemInfo 행(나무/돌/철 도끼, ToolTier 1/2/3) + CraftInfo/CraftMaterialInfo 레시피 행만. (단 §11 엣지 참조)

### C. 도면 월드스폰
- `InGameAreaWorker` 또는 신규 `BlueprintSpawnWorker` 에 area별 도면 스폰 로직 추가. `BlueprintInfo.SpawnAreaId/SpawnChance` 순회 → 확률 판정 → `SpawnDropItem(BlueprintItemId,1,pos)`. **중복방지(이미 해금/이미 월드존재) 정책은 director 결정 필요**(§11).

## 4. 데이터 레이어

### MonsterInfo 컬럼 확장 (행동패턴 도입 시 필수)
현재 컬럼: `Id,Name,MaxHP,PrefabKey,DropTableId,ATK`. **행동에 필요한 컬럼 전무.** 추가 후보:
| 컬럼 | 타입 | 용도 | 기본값 |
|---|---|---|---|
| MoveSpeed | FLOAT | 현재 `const MOVE_SPEED=2f` 하드코딩 → 몬스터별 | 2.0 |
| DetectRange | FLOAT | 추격 감지 반경 | 0(=비공격, 기존 평화) |
| AttackRange | FLOAT | 공격 사거리 | 1.5 |
| AttackCooldown | FLOAT | 공격 간격(초) | 1.5 |
| BehaviorType | ENUM | None/Melee/Charge/Poison/Ranged/Stealth/Smash/Pack 등 | None |

> DetectRange=0 / BehaviorType=None 이면 **기존 평화 배회와 동일** → 비파괴. 모든 행동 수치는 CSV(level-design). client는 컬럼 추가 + 파싱 소비만.

### 도구 실아이템 (ItemInfo) — R3-A **적용 완료(골격)**
ID 대역: 도끼 **501~503**(tier1/2/3), 곡괭이 **511~513**(tier1/2/3). 도면 401~403과 분리.
컬럼 변경 없음(기존 ItemInfo 스키마에 행 추가만) → ConvertInfo 재생성 불필요.

| ItemId | 이름 | ToolTier | IconKey(요청) |
|---|---|---|---|
| 501/502/503 | 나무/돌/철 도끼 | 1/2/3 | axe_wood/stone/iron_icon |
| 511/512/513 | 나무/돌/철 곡괭이 | 1/2/3 | pickaxe_wood/stone/iron_icon |

**레시피(CraftInfo Id 9~14, CraftMaterialInfo Id 17~26)** — 재료 수량은 **골격값**(나무→돌→철 점진). 실수치(재료수량·게이트 곡선)는 level-design 확정 대상.
- 9 나무도끼=나무3 / 10 돌도끼=나무2+돌3 / 11 철도끼=나무2+철광석3
- 12 나무곡괭이=나무3 / 13 돌곡괭이=나무2+돌3 / 14 철곡괭이=나무2+철광석3

**designer 리소스 요청**: axe_wood_icon / axe_stone_icon / axe_iron_icon / pickaxe_wood_icon / pickaxe_stone_icon / pickaxe_iron_icon (placeholder 임시 아이콘 OK). 미존재 시 기존 sword_icon 등으로 폴백해도 기능 무관.

> **현 한계(기능 사실)**: ToolTierSystem.GetCurrentToolTier()는 "보유 최대 ToolTier"만 본다 → 도끼/곡괭이를 **종류 구분 없이** 단일 채집배수에 합산. "도끼는 벌목·곡괭이는 채광"식 종류별 배수 분리는 현재 코드에 없음(R5 도구종류 확장 백로그와 일치). R3-A는 등급만 활성 — director 의도(§6 도끼·곡괭이 라인)와 부합하나 종류별 분리가 필요하면 별도 비용(S~M).

### 도면 (BlueprintInfo) — 이미 존재
SpawnAreaId/SpawnChance 컬럼 이미 있음. R3는 행 추가(level-design) + §3-C 스폰 코드.

## 5. 네트워크 동기화 항목

- 몬스터 행동: 상태머신은 **이미 서버권위**(`!IsServer return`). 신규 Chase/Attack도 서버에서만 구동. 위치는 기존 NetworkTransform. **추가 NetworkVariable 불필요**(공격은 `TakeDamage` 서버RPC 재사용).
- 은신 가시성: 클론에도 보여야 함 → `NetworkVariable<bool>` 또는 ClientRpc 토글 1건 필요(은신 채택 시).
- 도면 월드스폰: 서버에서 `SpawnDropItem`(이미 NetworkObject.Spawn 경유). 추가 동기화 없음.
- 도구 실아이템: 인벤토리 기존 동기화 재사용. 추가 없음.

## 6. 상태 머신 (몬스터 — 행동 도입 시)

확장 후 유효 상태: Idle, Walk(배회), **Chase, Attack**, Hit, Death.
- 전이: Idle/Walk → (플레이어 DetectRange 진입) → Chase → (AttackRange 진입) → Attack → (쿨다운/사거리이탈) → Chase → (DetectRange 이탈) → Idle.
- Hit는 어느 상태에서나 진입(피격 시), 종료 후 Chase 복귀(타깃 있으면) / Idle(없으면).
- Death는 종착(전이 불가).
- **차단**: DetectRange=0 몬스터는 Chase/Attack 진입 금지(평화 유지). Death 중 다른 상태 진입 금지(기존과 동일).

## 7. 이벤트 / Manager 계약

- `Managers.Info.MonsterInfoList` (확장 컬럼 소비), `WorldAreaMonsterInfoList`, `BlueprintInfoList`.
- `InGameController.Instance.SpawnWorker / PlayWorker / AreaWorker`.
- `CharacterBase.TakeDamage` (몬스터→플레이어 공격).
- `StateAnimationComponent.PlayAnimation(AnimationType.Attack)`.
- 무리추격 채택 시 AreaWorker에 "동료 타깃 전파" 헬퍼 신설.

## 8. 의존성 / 선행 작업

- **director**: R3 스코프(어느 패턴 몇 종 / 어느 지역 몬스터에 / 도면 신규 수 / 도구 라인업) 결정 필요.
- **level-design**: MonsterInfo 신규 컬럼 값, 도구 ItemInfo·Craft 행, BlueprintInfo 신규 행.
- **designer**: 도구 아이콘(axe/pickaxe 등), (원거리 채택 시) 투사체 프리팹, (몬스터 메쉬는 백로그 확정).
- **PrefabKey**: 현재 모든 몬스터가 `Test01Monster` 단일 프리팹. 행동패턴별 컴포넌트/콜라이더 차이가 있으면 프리팹 분화 필요(designer/client 협의).

## 11. 엣지 케이스 (구현 차원)

- **타깃 디스커넥트**: Chase/Attack 중 타깃 플레이어 퇴장 → 서버에서 null 체크 후 Idle 복귀.
- **동시 다중 플레이어**: Perception이 최근접 선택. 무리추격은 area 단위 공유 타깃.
- **area 경계 추격**: 플레이어가 area 벗어나면 추격 유지할지/포기할지 → **director 결정**(스폰 area 종속? 자유?). 기본 권고: leash 거리 초과 시 스폰지점 복귀.
- **권한 부재(클론)**: 상태머신·Perception·공격 모두 `IsServer` 가드. 클론은 NetworkTransform만.
- **도면 중복**: 이미 해금된 도면을 또 스폰? 이미 월드에 같은 도면 존재? → **director 정책 결정**(§3-C).
- **도구 중복 보유**: ToolTierSystem은 보유 최대 tier만 사용 → 중복 무해(기존 설계).
- **DetectRange=0 몬스터 피격**: Hit는 진입하나 Chase 미진입 → 반격 안 함(평화 유지). director가 "피격 시 반격" 원하면 별도 플래그 필요.
- **Death 중 공격틱**: AttackCooldown 코루틴/타이머는 Death Enter에서 중단해야 함.

## 12. 권고

1. **행동패턴은 신규 상태 남발 대신 파라미터화**: Chase+Attack 2상태 + `BehaviorType` enum 분기로 6패턴 대부분 흡수. 상태 클래스 폭발 방지, 비용 L→M.
2. **DetectRange=0 디폴트로 비파괴 도입**: 컬럼 추가해도 기존 평화 몬스터 안 깨짐. director가 지역별로 점진 활성.
3. **B축(도구 실아이템) 먼저**: 코드 0, R1-T3 즉시 활성. R3 빠른 가시성과(quick win). director 승인만 받으면 level-design+designer로 바로 진행.

# 부록 R3-B — 몬스터 추격/공격 행동 코어 (확정 명세)

> 범위: Chase+Attack 2상태 + `MonsterPerceptionComponent` + `MonsterInfo` 행동컬럼 + leash + 피격반격. BehaviorType=**Pack(무리)·Charge(돌진)** 만 R3-B(숲·늪). 나머지 4행동+보스는 R3-C. 비용 **M**.

## RB.1 신규 enum / 컬럼

`MonsterStateType` 에 `Chase`, `Attack` 추가 (GameEnum.cs).
신규 enum `MonsterBehaviorType` (GameEnum.cs):
```
None, Pack, Charge, Poison, Ranged, Stealth, WeatherBuff, Boss
```
> R3-B는 None/Pack/Charge만 코드 처리. 나머지 값은 enum에 미리 선언만(R3-C 분기 추가 시 컬럼 재파싱 불필요).

`MonsterInfo` 컬럼 추가(§4 표) — `MoveSpeed.FLOAT, DetectRange.FLOAT, AttackRange.FLOAT, AttackCooldown.FLOAT, BehaviorType.MONSTERBEHAVIORTYPE`.
**컬럼 추가이므로 ConvertInfo(MonsterInfo.cs) 재생성 필요** (InfoConvertEditor 메뉴). ENUM 컬럼 타입명 = enum명 대문자(`MONSTERBEHAVIORTYPE`) — 기존 `CRAFTCATEGORYTYPE` 등과 동일 규약.
기존 14행은 신규 컬럼 **빈칸=기본값**(DetectRange 0, BehaviorType None) → 전부 평화 유지(비파괴).

## RB.2 신규 클래스 / 수정

| 대상 | 종류 | 역할 |
|---|---|---|
| `MonsterPerceptionComponent` | **신규** Component (`Assets/Scripts/WorldBaseObject/Monster/`) | 서버전용. Update마다 `Physics.OverlapSphere(pos, DetectRange, playerLayer)` 로 최근접 플레이어 타깃 산출. `CurrentTarget` getter. DetectRange=0이면 비활성(탐색 안 함). |
| `MonsterChaseState` | **신규** State | 타깃 향해 `MoveTowardsTarget` 추격. AttackRange 진입 → Attack. **leash**: 스폰지점 거리 > SpawnRadius×1.5 또는 타깃 null/DetectRange 이탈 → Walk(스폰지점 복귀). |
| `MonsterAttackState` | **신규** State | AttackRange 유지 시 쿨다운마다 `target.TakeDamage(ATK, owner)` + `PlayAnimation(Attack)`. 사거리 이탈 → Chase. 타깃 null → Idle. Charge면 진입 시 1회 가속 직선 이동(파라미터 분기). |
| `MonsterStateMachine` | **수정** | Dictionary에 Chase/Attack 등록. `MOVE_SPEED`/`WANDER_RADIUS` const → MonsterInfo값 소비(MoveSpeed). 스폰지점/Perception 접근 헬퍼 추가. NavMeshAgent.speed = MoveSpeed 세팅. |
| `MonsterBase` | **수정** | MonsterInfo 행동컬럼 캐싱(DetectRange/AttackRange/AttackCooldown/BehaviorType getter). `OnDamaged`에서 **피격반격**: 타깃 미설정 시 공격자를 타깃으로 Perception에 주입 후 Hit→Chase(DetectRange와 무관하게 반격 1회 발동). |
| `MonsterSpawnArea` | **수정** | `SpawnRadius` getter 노출(현재 private) — leash 거리 계산용. |

## RB.3 상태 전이 (R3-B 확정)

```
Idle/Walk ──(Perception 타깃 감지, DetectRange>0)──▶ Chase
Chase ──(AttackRange 진입)──▶ Attack ──(쿨다운/사거리이탈)──▶ Chase
Chase ──(타깃 null | DetectRange 이탈 | leash 초과)──▶ Walk(스폰지점 복귀)──▶ Idle
Attack ──(타깃 null)──▶ Idle
임의상태(피격) ──▶ Hit ──(종료)──▶ Chase(타깃 있음) | Idle(없음)
임의상태 ──(사망)──▶ Death(종착)
```
**차단**: DetectRange=0 → Perception 비활성 → 자발적 Chase 불가(평화). 단 피격반격은 OnDamaged가 타깃 강제주입하므로 가능. Death 진입 후 모든 전이·AttackCooldown 타이머 중단.

## RB.4 네트워크

- Perception/Chase/Attack 전부 **서버에서만**(상태머신 기존 `!IsServer return` 가드). 위치=NetworkTransform 기존. 공격=`TakeDamage` 서버RPC 재사용. **신규 NetworkVariable 0**.
- Pack(무리) 타깃 전파: 서버 내 로직(`InGameAreaWorker`에 "같은 area 몬스터에 타깃 공유" 헬퍼). 네트워크 추가 없음(서버 권위 내부).

## RB.5 BehaviorType 분기 (R3-B 2종)

- **Pack(무리)**: Chase 진입 시 같은 SpawnAreaId 몬스터들에게 동일 타깃 전파(AreaWorker 경유). 단독 추격 대비 코드 +S.
- **Charge(돌진)**: Attack 진입 시 1회 가속 직선 돌진(MoveSpeed×배수, 고정방향). 충돌 또는 시간 만료 후 일반 Attack. 돌진 배수는 MonsterInfo 추가 컬럼 또는 상수(level-design 협의).

## RB.6 의존 / 리소스 요청

- **level-design**: MonsterInfo 신규 컬럼 값(숲·늪 몬스터의 DetectRange/AttackRange/Cooldown/MoveSpeed/BehaviorType), Charge 돌진배수.

- **PrefabKey 단일(Test01Monster)**: R3-B는 단일 프리팹 + 로그/색으로 검증 가능. Perception 컴포넌트는 프리팹에 1개 부착(MCP `u_editor_prefab`로 와이어링).

## RB.7 엣지 케이스 (R3-B)

## RC.3 행동별 구현 요지

- **Poison**: 근접 적중 후 `target.ApplyPoison(3,3,1f)` — 환경데미지(DEF무시) 틱 3회. 재적중 시 직전 코루틴 갱신.
- **Ranged**: 신규 상태 없음. AttackRange 넓게(6) → Chase가 먼 거리에서 Attack 진입 → 히트스캔(`TakeDamage`). **투사체 = 백로그.**
- **Stealth**: `m_IsVisible` NetworkVariable. 평시 false(알파 0.18), 추격중/근접(2.5m)/안개 시 true(알파 1.0). 서버 Tick.
- **WeatherBuff**: `WeatherWorker.GlobalWeather`가 눈보라/안개면 이동 ×1.4(변화 시에만 재적용). ATK 강화 = 백로그.
- **Boss**: HP 구독으로 66%/33% 통과 시 `m_BossPhase` 단조 상향 → 이동가속(×1.25/페이즈) + 데미지 배수(×1.3/페이즈). Death에서 구독 해제. **모델·연출 = 백로그.**

## RC.4 도면 월드스폰 (director 결정 A)

- 세션 시작(`Initialize`) 1회 `SpawnAreaBlueprints`: `BlueprintInfoList` 순회 → `SpawnAreaId` 활성 area면 `SpawnChance` 롤 → `SpawnDropItem(BlueprintItemId,1,pos)`.
- **중복무효/결핍**: area당 세션 1회(`m_SpawnedBlueprintIds`), 리스폰 없음. 시작 시 아무 도면도 미해금 → 중복 해금 문제 자체가 없음(가장 단순·비파괴). SpawnChance(<1)로 세션당 일부 누락 = 결핍설계 충족.
- 줍는 즉시 해금: 기존 `WorldDropItem` → `RecipeUnlockRegistry.Unlock`(멱등) 그대로.
