# campfire-cold — 코드명세

## 1. 영향 범위 요약

밤+모닥불꺼짐 조건에서 Cold가 누적되고, Cold 단계에 따라 HP 틱 데미지를 주는 시스템.

영향 파일:
- `WorldBuildingObject.cs` — `m_IsLit` NetworkVariable 추가, 꺼짐 시 Cold 회복 중단
- `DayNightWorker.cs` — Cold 누적 틱 추가 (기존 Cold 감쇠와 병렬)
- `PlayerCharacter.cs` — `TakeEnvironmentDamage()` 메서드 추가 (DEF/크리 무시)
- `DayNightConfig.cs` — Cold 단계 수치 파라미터 추가
- 신규: `ColdDamageWorker.cs` — Cold 단계 판정 + HP 틱 데미지 주관

## 3. 신규 필요 클래스/스크립트

### `ColdDamageWorker` (신규)
- 위치: `Assets/Scripts/Worker/ColdDamageWorker.cs`
- 역할: 서버 전용. 매 틱 Cold 단계 판정 후 HP 데미지 적용
- 부착: `InGameController` GameObject (DayNightWorker와 동일 GameObject 권장)
- 주요 메서드 시그니처:
  ```
  private void UpdateColdAccumulation(float _deltaTime)   // 밤+꺼짐 조건 Cold 누적
  private void UpdateColdDamageTick(float _deltaTime)     // 단계별 HP 틱
  private ColdStage GetColdStage(int _cold)               // 단계 판정
  private bool IsNearActiveCampfire(PlayerCharacter _player)  // 켜진 모닥불 근접 여부
  ```

### `ColdStage` enum (신규, `GameEnum.cs`에 추가)
```
None      // Cold 0~29
Warning   // Cold 30~59
WeakDot   // Cold 60~89
StrongDot // Cold 90~100
```

### `WorldBuildingObject` 수정
- `NetworkVariable<bool> m_IsLit` 추가 (Server Write, Everyone Read)
- `ApplyCampfireEffect()`: `m_IsLit.Value == false`이면 스킵
- `SetLit(bool _lit)` public 메서드 추가

### `PlayerCharacter` 수정
- `TakeEnvironmentDamage(int _damage, bool _allowDeath)` 추가
  - 서버 전용, DEF/크리티컬 무시
  - `_allowDeath == false`이면 `Mathf.Max(1, HP - damage)` 적용 (HP 1 보호)
  - 내부에서 `SetHP` 직접 호출 후 `OnDamagedClientRpc` 발화

## 4. 데이터 레이어

`DayNightConfig.cs` ScriptableObject에 항목 추가 (CSV 불필요 — 단계 수치는 씬당 1개 고정값):

## 5. 네트워크 동기화

| 항목 | 방식 | 권한 |
|---|---|---|
| `WorldBuildingObject.m_IsLit` | `NetworkVariable<bool>` | Server Write / Everyone Read |

별도 새 Rpc 없음. 기존 `OnDamagedClientRpc` 재활용.

## 6. 상태 머신

`ColdStage` 전이 (플레이어당 Cold 값 기준):

```
None(0~29)  → Warning(30~59)  조건: Cold >= 30
Warning     → WeakDot(60~89)  조건: Cold >= 60
WeakDot     → StrongDot(90~100) 조건: Cold >= 90
StrongDot   → WeakDot         조건: Cold <= 89
WeakDot     → Warning         조건: Cold <= 59
Warning     → None            조건: Cold <= 29
```

불가능한 전이: None → StrongDot (누적은 반드시 순서대로 진행).

## 11. 엣지 케이스

| 케이스 | 처리 방침 |
|---|---|
| 플레이어가 모닥불 근처에 있지만 꺼진 경우 | `m_IsLit == false` → 보호 없음, Cold 누적 계속 |
| 플레이어가 모닥불 근처에 있고 켜진 경우 | `IsNearActiveCampfire == true` → Cold 누적 스킵. `ApplyColdDecay`(DayNightWorker)는 밤에도 계속 작동하므로 누적 스킵 중에도 감쇠는 진행됨 |
| 밤 + 모닥불 근처(켜짐) 플레이어 Cold 방향 | 누적 없음 + 감쇠 있음 → Cold 자연히 줄어듦 |
| 강틱(StrongDot) 중 HP가 1일 때 | `Mathf.Max(1, 1 - 5) = 1` → HP 유지, 죽지 않음 |
| 플레이어 사망 후 Cold 틱 발동 | `CharacterBase.IsDead` 체크 후 스킵 |
| 낮(Day/Dusk/Dawn) 중 Cold 누적 | `CurrentPhase != Night` → 누적 안 함 |
| 플레이어 disconnect 중 Cold 틱 | **보장됨**: `CharacterBase.OnNetworkDespawn()`에서 `UnregisterFromCharacterRegistry()` 호출 → disconnect 시 NetworkObject despawn → registry에서 제거됨. `GetAlivePlayers()` 반환 목록에 포함되지 않음 |
| `m_IsLit` 초기값 | `true`로 스폰 (모닥불은 설치 시 켜진 상태) |
| Cold 누적 중 낮으로 전환 | 누적 중단, Cold는 기존 `ApplyColdDecay`로 자연 감쇠 |
| 여러 모닥불 중 하나라도 근처에 켜진 것이 있으면 | 보호 적용 (OR 조건) |

## 12. 권고

- **연료/내구도 소모 시스템**: `m_IsLit = false` 전환 트리거가 없으면 모닥불이 영원히 켜진 상태. 이번 슬라이스에서 연료 소모까지 포함할지 director 확인 필요. 포함하지 않으면 `SetLit(false)`는 테스트/QA 전용으로만 노출.
- **Cold 스케일**: 옵션 B 확정 — `DEFAULT_MAX_COLD = 100` 유지, 단계 임계값 30/60/90 적용.

## 13. 구현 확정 결정 (디렉터, 2026-06-03)

team-lead가 코드 검증 중 발견한 모델 충돌·미결 3건을 디렉터 권한으로 확정한다. client는 이 섹션을 우선 따른다.

### 결정 1 — 켜진 모닥불 근처 효과: **B (누적 스킵 + 자연 감쇠가 줄임)**

- **확정**: 켜진 모닥불 근처(`IsNearActiveCampfire == true`)에서는 **Cold 누적만 스킵**한다. 별도의 적극적 Cold 감소는 추가하지 않고, 기존 자연 감쇠(`ApplyColdDecay`)가 Cold를 줄인다.
- **기존 코드 처리**: `WorldBuildingObject.ApplyCampfireEffect()`의 `SetCold(Cold + 2)`는 옛 체온모델 잔존이며 추위 모델과 정반대다. **이 누적 호출은 제거**한다 (불 옆에서 추위가 오르면 안 됨). 모닥불의 역할은 "추위 누적을 막는 안전지대"로 단일화.
- **근거**: 명세 §11이 이미 채택한 모델(누적 스킵+감쇠 진행)과 일치. 최소 슬라이스 — 새 감소 경로를 만들 필요 없이 기존 감쇠를 재활용. ③ 분기("모닥불=지금 안전을 산다")는 "추위가 안 쌓이는 안전지대" 만으로 충분히 성립.

### 결정 2 — 밤 ×2 감쇠 멀티플라이어: **A (1.0 고정)**

- **확정**: `DayNightWorker.GetColdMultiplier`의 밤 ×2 가속(Night 2.0)은 옛 체온모델 잔존으로 추위 모델과 거꾸로다(밤에 추위가 더 빨리 사라지면 안 됨). **밤 멀티플라이어를 1.0으로 고정**한다. 밤의 추위 누적은 `ColdDamageWorker`가 전담한다(누적 ↑, 감쇠는 평탄).
- **근거**: 책임 분리 명확 — 감쇠(평탄)와 누적(밤 전담)을 한 곳에서 안 섞음. 밤이 압박 장치가 되려면 추위가 빠지지 말고 쌓여야 함(기획 §2·톤). 모델 방향 일관성 확보.

### 결정 3 — 연료 시스템 범위: **B (SetLit은 TestManager/QA 전용, 연료는 백로그)**

- **확정**: 이번 0순위 슬라이스에 **연료/내구 소모 시스템은 포함하지 않는다**. `SetLit(false)` 트리거는 TestManager/QA에서만 호출해 추위 누적 발동을 실증한다. 연료 소모 → 자동 소등은 별도 기획서(범위 L)로 분리, 백로그.
- **근거**: 갭로드맵 §2 "최소 슬라이스로 빠르게 실증" + §7 안티골(한 번에 다 도입 금지, 1인 개발 슬라이싱 필수). 0순위의 목표는 "추위→데미지→방어" 루프의 실증이지 연료 경제 구축이 아니다. 연료는 ③ 트레이드오프를 완성하는 후속 작업으로 그 자체가 별도 기획 가치가 있음.

> **client 주의**: 결정 1·2는 기존 코드의 *제거/수정* 항목이다. `ApplyCampfireEffect()`의 `SetCold(Cold+2)` 제거, `GetColdMultiplier` 밤 값 1.0 고정 — 둘 다 옛 체온모델 잔존 제거이지 신규 추가가 아님.
