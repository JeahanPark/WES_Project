## 2. 재활용 가능한 기존 시스템

| 시스템 | 재활용 방식 |
|---|---|
| `CoreTensionOverlayWorker` | 오버레이 레이어·페이드 코루틴·Cold/HP/Phase 구독 골격에 날씨/이동 레이어 추가 |
| `WeatherWorker.OnWeatherChanged` (static event) + `GlobalWeather` (static) | 오버레이가 구독 → 날씨별 연출. 서버권위·전원동기 완료, 신규 Rpc 불필요 |
| `PlayerCharacter` transform (오너 로컬) | 매 프레임 위치 델타로 이동 중/정지 판정(로컬). NavMeshAgent는 오너만 활성 |
| `PlayerCharacter.SubscribeOnColdChanged` | Cold 변화량/방향으로 "추위 가속(이동중)/회복(정지·모닥불)" 구간 판정 |
| `DayNightWorker.OnPhaseChanged` (static event) | 이미 오버레이가 구독 중(낮밤 틴트) |
| `BlueprintToast` (`ShowMessage(string)`) | 지역진입 토스트로 일반화 재활용(도면 토스트와 동형). 신규 ToastWorker 불필요 |
| `InGameHUDWorker` (`m_BlueprintToast`·`m_CoreTensionOverlay`·`SetLocalPlayer`) | 토스트·오버레이 보유·로컬플레이어 연결 진입점 |

| `ResourceManager.LoadAddressable<T>` / `Load<T>` | AudioClip 로드에 그대로 사용. 신규 로더 불필요 |
| `Managers` 싱글톤 패턴 (`MonoSingleton<T>`) | AudioManager를 `Managers.Audio`로 등록(Init/Clear 라인 추가) |

## 3. 신규 필요 클래스/스크립트

### 3-1. `AudioManager` (Manager)
- 위치: `Assets/Scripts/Manager/AudioManager.cs`
- 상속: `MonoSingleton<AudioManager>`
- 역할: 4채널 사운드 재생 골격. 음원 0개여도 동작(null 가드).
- 가이드 시그니처(public):
  - `void PlayBgm(string _key, bool _loop = true)` — 단일 BGM 소스, 키 교체 시 크로스페이드
  - `void PlayAmbient(string _key)` — 환경음 루프(날씨/페이즈 연동)
  - `void StopAmbient()`
  - `void PlaySfx(string _key)` — 원샷 풀(N소스 라운드로빈)
  - `void PlayStinger(string _key)` — 사건 강조 원샷(우선순위 최상)
  - `void SetChannelVolume(AudioChannel _channel, float _volume01)`
  - `void SetMasterVolume(float _volume01)`
- 내부: 채널별 AudioSource(BGM 1·Ambient 1·SFX 풀 N·Stinger 1), 클립 캐시 `Dictionary<string,AudioClip>`, 로드 실패 시 null 스킵.

### 3-2. `AudioChannel` (enum) — `Assets/Scripts/GameEnum.cs`에 추가
- `Bgm`, `Ambient`, `Sfx`, `Stinger`

### 3-3. `WeatherOverlay` (enum, 내부 or GameEnum) — 선택
- 오버레이가 날씨를 자체 분류. WeatherType 직접 사용해도 됨(신규 enum 불필요).

### 3-4. 이벤트 트리거 UI — 신규 위젯 (designer 프리팹 의뢰)
- 위치(스크립트): `Assets/Scripts/UI/HUD/EventDialogueWorker.cs` (가칭)
- 역할: 관문/이벤트 대사 6종 1줄 표시(토스트와 다르게 트리거 시점·약간 더 긴 표시). **재활용 권고**: BlueprintToast와 거의 동형 → 별 위젯 대신 BlueprintToast를 `ShowMessage(string, float _displayTime)` 오버로드로 일반화하고 동일 프리팹 인스턴스 1개 추가로 처리 가능. (director 결정 필요 — 12절 참조)

> 신규 NetworkBehaviour 없음. AudioManager·토스트·오버레이 전부 로컬.

## 4. 데이터 레이어

### 4-1. NarrationData (ScriptableObject 신설) — `Assets/Scripts/Config/NarrationData.cs` + `.asset`
> CSV 파이프라인은 STRING 콤마 금지 고정 → 콤마 포함 내레이션/이벤트/엔딩이 다수라 **CSV 대신 SO** 채택(director 정리: 이 텍스트들은 CSV 무관, 원문 유지). CSV 신설 취소.
- 구조: `[Serializable] class NarrationEntry { string Key; [TextArea] string Text; NarrationCategory Category; }` + `List<NarrationEntry> m_Entries`.
- `NarrationCategory` enum: AreaEnter / Notify / Event / Ending.
- 분량: 지역진입12 + 통지2 + 이벤트6 + 엔딩5 = **25엔트리**. (플레이버13은 ItemInfo.Description로 별도)
- 조회: `string Get(string _key)` (없으면 빈문자열) / `IReadOnlyList<NarrationEntry> GetByCategory(NarrationCategory)`.
- 콤마 자유(인스펙터 TextArea), 빌드 포함, director/story 편집 가능.
- AudioManager·오버레이처럼 로컬 전용, 동기화 무관.

### 4-2. ItemInfo.csv Description (기존 컬럼 채움)
- 13종 아이템의 Description을 story v0.1 플레이버로 교체. 스키마 변경 없음(이미 STRING 컬럼 존재).
- **콤마 금지** 제약 → story 텍스트에 콤마 있으면 client가 반영 시점에 우회표기(마침표/공백) 확인 필요.

### 4-3. 사운드 키 (인라인/SO 아님 → 상수 모음)
- 트리거 키는 `AudioKey` 정적 상수 클래스 권고: `Assets/Scripts/Manager/AudioKey.cs`
  - `public const string SFX_HIT = "sfx_hit";` 등. (키-음원 매핑은 sound가 Addressable 등록)

## 5. 네트워크 동기화 항목

- **신규 NetworkVariable: 없음.**
- **신규 ServerRpc/ClientRpc: 없음.**

- 권한: 모든 R4 연출/사운드/텍스트는 **각 클라이언트 로컬**. 동료 사망 stinger/텍스트도 이미 동기화된 사망 상태를 각 클라가 읽어 로컬 발화(기획 §9).

## 6. 상태 머신 (오버레이 블렌드)

오버레이는 **우선순위+블렌드**(누적 가산 아님, 기획 §5-1):
- 채널 분리: 추위(G-2)·날씨(비/안개/눈보라)·낮밤틴트(G-3)·저체력비네팅(G-13)·사망(G-14)·앰비언트안개(I-5).
- 합산 후 최종 화면 가시도가 `m_OverlayVisibilityFloor` 아래로 내려가지 않게 클램프(절대 차단 금지).
- 날씨 전이: WeatherWorker가 한 단계씩만 이동(기존). 오버레이는 OnWeatherChanged에서 목표 알파로 페이드(기존 CoFade 패턴 재사용).
- **불가능 전이(차단)**: 어떤 조합도 "전체 화면 알파 1.0(완전 암전)"에 도달 금지(사망 G-14 의도적 암전 제외 — 그건 PlayDeathFade 독립 경로).

## 11. 엣지 케이스 (구현 차원)

- **오버레이 과중첩** → m_OverlayVisibilityFloor 클램프로 완전차단 불가(§6).
- **음원 0개(④ 백로그)** → 모든 Play*가 null 가드로 무음 통과. 에러 없음.
- **클립 로드 실패** → 캐시에 null 저장 후 재시도 안 함(스팸 방지), 1회 경고 로그.
- **비오너 클론에서 이동 판정** → 오버레이는 로컬플레이어(오너)만 SetLocalPlayer로 연결 → 비오너 영향 없음.
- **동료 사망 stinger 중복** → 이미 동기화된 사망 상태 전이 시 1회만(상태 변화 이벤트 기준).
- **빠른 날씨 전이(페이즈마다)** → CoFade가 진행 중 코루틴 Stop 후 재시작(기존 패턴).
- **텍스트 40자 초과** → TMP 줄바꿈 또는 … (designer 협의, 기획 §9: 첫 줄에 핵심).
- **AudioManager 씬 전환** → MonoSingleton DontDestroyOnLoad, BGM 유지/교체는 PlayBgm 키 비교.
- **빈 NarrationInfo 조회(Key 없음)** → GetNarration null 반환, 호출부 빈 문자열 가드.
- **중복 PlaySfx(동프레임)** → SFX 풀 라운드로빈으로 잘림 없이 다중 재생.

4. **콤마 우회표기 [director 제공].** CSV(ItemInfo.Description) 콤마 2건만 director가 우회표기 확정:
