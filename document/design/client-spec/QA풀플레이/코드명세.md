## 2. 재활용 가능한 기존 시스템

| 시스템 | 재사용 방식 |
|---|---|
| `InGameController.m_TestMode` + `CoInitializeTestMode()` | Managers init + Info 로드 + Relay Host + 스폰 셋업 100% 재사용. 신규 Host 로직 0줄 |
| `GameSceneManager.LoadScene(SCENE_INGAME)` | Intro→Ingame 씬 전환. (Host 아닌 상태라 일반 SceneManager 경로, Ingame에서 TestMode가 Host 시작) |
| `InGameController.OnPlayerReachedEscape(player)` (public) | Warp 폴백 시 트리거 직접 발동에 사용 가능 (단 정상경로는 트리거 콜라이더로) |
| `CharacterBase.MoveWithDirection(Vector2)` | EscapePoint 방향 실제 이동 |
| `NavMeshAgent.Warp()` (player.GetComponent) | 60초 타임아웃 시 폴백. `CoTestTerrainSlope`가 이미 동일 패턴 사용 |
| `CharacterBase.SetMaxHP/SetHP`, `PlayerCharacter.SetCold` | 무적/저체온 동결 (TestManager 기존 관행) |
| `EscapePoint` (Ingame 씬 1개 배치) | `FindFirstObjectByType<EscapePoint>()`로 목표 위치 획득 |
| `ResultPopup` + `InGameController.GameState` | 클리어 판정: `GameState == GameState.Clear` 또는 `Managers.Popup.FindOpen<ResultPopup>()` |

## 3. 신규 필요 클래스/스크립트

신규 클래스 없음. 메서드 추가만:

### 3.1 `TestManager.TestFullPlayClear()` / `CoTestFullPlayClear()`
- 위치: `Assets/Scripts/Manager/TestManager.cs` (#if UNITY_EDITOR 블록 내)
- 역할: E2E 단일 진입점. QA가 `u_play_invoke`로 `TestFullPlayClear` 호출.
- 시그니처(가이드):
  ```
  public void TestFullPlayClear()                 // StartCoroutine(CoTestFullPlayClear())
  private IEnumerator CoTestFullPlayClear()       // 본체
  ```

### 3.2 `InGameController` 진입 플래그 (#if UNITY_EDITOR)
- 역할: Intro에서 Ingame으로 점프할 때, Ingame의 InGameController가 Start에서 자동으로 TestMode 셋업을 타게 한다.
- 타이밍 사실: InGameController는 `Awake`(싱글톤 등록)→`Start`(m_TestMode 체크)가 같은 프레임. 씬 로드 후 인스턴스 플래그를 켤 틈이 없음 → **static 플래그**가 안전.
- 시그니처(가이드):
  ```
  #if UNITY_EDITOR
  private static bool s_ForceTestModeOnLoad = false;        // private static (WES: public 직접접근 금지)
  public static void RequestTestModeOnLoad()  => s_ForceTestModeOnLoad = true;
  public static bool ConsumeTestModeRequest() { bool v = s_ForceTestModeOnLoad; s_ForceTestModeOnLoad = false; return v; }
  #endif
  ```
- `Start()` 수정: `if (m_TestMode)` → `#if UNITY_EDITOR if (m_TestMode || ConsumeTestModeRequest()) #else if (m_TestMode) #endif`. (플래그는 1회 소비 후 false로 리셋 — 다음 정상 플레이 오염 방지)

## 4. 데이터 레이어 (CSV / ScriptableObject / 인라인 상수)

신규 CSV/SO 없음. 튜닝 파라미터는 TestManager const (테스트 전용, 진짜 상수):

| 상수 | 기본값 | 범위 | 의미 |
|---|---|---|---|
| `FULLPLAY_TOTAL_TIMEOUT` | 120f | 60~300 | 전체 타임아웃(초). 초과 시 FAIL |
| `FULLPLAY_MOVE_TIMEOUT` | 60f | 30~120 | EscapePoint 이동 타임아웃(초). 초과 시 Warp 폴백 |
| `FULLPLAY_ARRIVE_DISTANCE` | 1.5f | 0.5~3 | EscapePoint 도달 판정 거리(트리거 콜라이더 진입 보조) |
| `FULLPLAY_INVINCIBLE_HP` | 999999 | — | 무적용 HP (기존 TestMonsterRespawnDamage가 99999 사용 — 동일 관행) |

UPPER_SNAKE_CASE, `public const` (WES 상수 규칙).

## 5. 네트워크 동기화 항목

- **NetworkVariable 추가/변경: 없음.**
- **신규 RPC: 없음.** 클리어는 기존 `NotifyEscapeServerRpc`→`TriggerClearClientRpc` 흐름 그대로.
- 권한: 솔로 Host이므로 로컬=서버=오너. `SetHP/SetMaxHP/SetCold`는 IsServer 가드가 이미 있고 Host라 통과. Warp는 오너 로컬에서 수행.
- 주의: `OnPlayerReachedEscape`는 IsOwner 플레이어 기준. 솔로 Host의 LocalPlayer가 곧 오너라 문제 없음.

## 6. 상태 머신 (E2E 진행 단계 — 코루틴 내부 순차 상태)

기획 5단계에 대응하는 체크포인트. 각 단계 진입 시 로그 1줄.

```
[S0 Intro시작] InGameController.RequestTestModeOnLoad() → LoadScene(Ingame)
      ↓ (Ingame 로드, m_TestMode 경로로 CoInitializeTestMode 자동 실행)
[S1 Ingame진입·스폰] InGameController.Instance.IsGameStarted == true && PlayWorker.LocalPlayer != null 대기
      ↓ (무적/Cold 동결 적용)
[S2 이동개시] EscapePoint 위치 획득 → 매 프레임 MoveWithDirection(목표방향)
      ↓
[S3a EscapePoint도달]  도달거리 이내 + 트리거 발동(GameState==Clear 전이) → 정상
[S3b Warp폴백]         이동 60초 초과 → NavMeshAgent.Warp(escapePos) + 트리거 보조 → "이동실패→Warp폴백" 로그
      ↓
[S4 클리어화면] GameState==Clear && ResultPopup 열림 확인 → PASS
```

### 가능한 전이
- S0→S1: 씬 로드 완료 + IsGameStarted (CoWaitForGameStart 통과)
- S2→S3a: `Vector3.Distance(player, escape) <= ARRIVE_DISTANCE` 또는 `GameState==Clear`
- S2→S3b: `elapsed > FULLPLAY_MOVE_TIMEOUT`
- S3a/S3b→S4: `GameState==Clear` (TriggerClearClientRpc 수신)

### 불가능한/차단 전이
- 전체 elapsed > FULLPLAY_TOTAL_TIMEOUT: 어느 단계든 즉시 FAIL 중단(이동 정지 + 로그).
- S1 미도달 상태에서 S2 진입 금지 (LocalPlayer null이면 이동 명령 X — 기획 9 순서보장).

## 7. 이벤트 / Manager 계약

호출하는 것:
- `InGameController.RequestTestModeOnLoad()` (신규, static)
- `GameSceneManager.Instance.LoadScene(GameSceneManager.SCENE_INGAME)`
- `Object.FindFirstObjectByType<InGameController>()` / `<EscapePoint>()`
- `controller.PlayWorker.LocalPlayer` → `MoveWithDirection` / `GetComponent<NavMeshAgent>().Warp`
- `player.SetMaxHP/SetHP/SetCold`
- `controller.GameState`, `Managers.Popup.FindOpen<ResultPopup>()`

수신 이벤트: 없음(폴링 기반 WaitUntil). 의존 Worker: `InGamePlayWorker`(LocalPlayer). 의존 Component: `NavMeshAgent`(Warp 폴백).

## 8. 의존성 / 선행 작업

- EscapePoint가 Ingame 씬에 NavMesh 도달 가능 위치에 배치돼 있어야 함(현재 1개 배치 확인됨).
- PlayerCharacter 프리팹에 NavMeshAgent 부착(확인됨, CharacterBase.InitializeNavAgent).
- Relay Host 가능 환경(m_TestMode가 HostRelayAsync 사용 — 네트워크 필요). **오프라인/CI에선 Host 실패 가능 — 리스크 11 참조.**
- 디자이너 의뢰: 없음(신규 프리팹/리소스 0).

---

## 9. 구현 단계 (순서)

1. `InGameController`: static 플래그 + `RequestTestModeOnLoad`/`ConsumeTestModeRequest` 추가, `Start`의 m_TestMode 조건 확장.
2. `TestManager`: const 4개 추가.
3. `TestManager.CoTestFullPlayClear` 작성:
   - S0: `InGameController.RequestTestModeOnLoad()` → `LoadScene(Ingame)` → 로그.
   - S1: `WaitUntil(InGameController.Instance?.IsGameStarted == true && LocalPlayer != null)` (TOTAL_TIMEOUT 동시 감시) → 무적/Cold 동결 → 로그.
   - S2: EscapePoint 위치 획득, 이동 루프 시작 → 로그.
   - S3: 도달/타임아웃 분기, 폴백 시 Warp+로그.
   - S4: `WaitUntil(GameState==Clear)` → PASS 로그. 타임아웃 시 FAIL 로그.
4. 개발 테스트: refresh → 2~3초 → console error 0 확인.
5. QA 의뢰.

## 11. 엣지 케이스 (구현 차원)

| 케이스 | 처리 |
|---|---|
| Ingame 로드 후 InGameController 못 찾음 | WaitUntil 내 null 가드, TOTAL_TIMEOUT으로 FAIL |
| Relay Host 실패(오프라인) | CoInitializeTestMode가 LogError, IsGameStarted 영원히 false → S1 TIMEOUT FAIL("Host 셋업 실패" 식별). **자동 복구 불가 → 콘솔 로그로 명확화** |
| LocalPlayer 스폰 전 이동 명령 | S1에서 LocalPlayer null 차단, 스폰 확인 후에만 S2 진입 |
| EscapePoint 없음 | S2에서 null 가드 → 즉시 FAIL 로그 |
| NavMesh 막힘(나무/슬로프) | MOVE_TIMEOUT 60초 → Warp 폴백 + "이동실패→Warp폴백" 로그(회귀 신호) |
| 이동 중 사망(몬스터/Cold) | S1에서 SetMaxHP/SetHP 무적 + SetCold(0) 동결 → 사망 차단. (혹시 죽으면 MoveWithDirection이 IsDead로 막힘 → MOVE_TIMEOUT→Warp폴백→트리거. OnPlayerReachedEscape는 IsDead 가드로 거부될 수 있음 → 폴백 직전 SetHP 재적용) |
| 클리어 RPC 도달 후 ResultPopup 미표시 | S4는 GameState==Clear 우선 판정(RPC 수신=PASS), ResultPopup은 보조 로그 |
| 중복 호출(코루틴 재진입) | TestManager 코루틴 1회성. 진행 중 재호출 가드 필요 → 진행 플래그(bool m_FullPlayRunning) 1개로 차단 |
| 전체 타임아웃 발동 | 이동 루프 정지(MoveWithDirection(zero)) + FAIL 단계명 로그 + 코루틴 종료 |

## 12. 권고

- **무적/스탯 동결**: 신규 무적 NetworkVariable 만들지 말고 `SetMaxHP/SetHP/SetCold` 재사용(비용 0, 네트워크 영향 0). 동결은 S1 1회 적용. 만약 HP regen/Cold tick이 무적을 침식하면 이동 루프에서 주기 재적용(저비용).
- **로비 UI 회귀**: 이번 범위 제외(합의). Backlog "Intro→Lobby UI 자동통과(Relay 풀흐름)"로 분리.
- **Host 실패 가시화**: 오프라인 환경에서 Relay Host가 막히면 이 E2E는 S1에서 FAIL — 이는 도구 한계가 아니라 환경 의존. QA 실행 시 네트워크 연결 전제임을 QA에게 명시 필요.
