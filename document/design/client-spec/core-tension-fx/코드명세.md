## 3. 신규 필요 클래스/스크립트

**원칙: 신규 클래스 0개 목표.** 기획 충족에 필요한 단 하나의 신규 동작은 도면해금 "잠긴 칸 → 정상색 전환 + 테두리 반짝"이며, 이는 **신규 클래스 없이 `CraftScrollCell`에 메서드 추가**로 해결한다.

| 항목 | 위치 | 역할 | 형태 |
|---|---|---|---|
| `CraftScrollCell.PlayUnlockFlash()` | 기존 `CraftScrollCell.cs`에 추가 | 잠금 오버레이 페이드 아웃 + 테두리 1회 반짝 코루틴(`CoPlayUnlockFlash`) | 기존 클래스 메서드 추가 |
| (선택) 테두리 반짝용 `[SerializeField] private Image m_UnlockFlashFrame;` | `CraftScrollCell.cs` | designer가 프리팹에 반짝용 프레임 Image 슬롯 제공 시 연결 | SerializeField 슬롯 |

> 합성/오버레이는 신규 클래스 불필요 — §5.4 합성은 기존 `CoreTensionOverlayWorker`의 레이어 sibling order로 충족.

### 3.1 결정1·2 반영 — `CoreTensionOverlayWorker` 수정 (신규 클래스 아님)

| 변경 | 내용 |
|---|---|
| **알파 캡(결정1)** | const `COLD_OVERLAY_MAX_ALPHA = 0.85f` 추가. `ApplyColdStage`/`CoFadeColdStage`/`UpdateColdStage3Pulse`의 단계3 목표·펄스 상한을 1.0 → `COLD_OVERLAY_MAX_ALPHA`로 캡. 어떤 텍스처가 와도 중앙 시야 여백 보증 |
| **단일 소스화(결정2)** | `[SerializeField] private DayNightConfig m_Config;` 추가. `EvaluateColdStage`가 `m_ColdStageWarning/Weak/Strong` 인스펙터 필드 대신 `m_Config.ColdStageWarning/Weak/Strong`를 읽음. 인스펙터 임계 3필드 **제거**. `m_Config` null이면 안전 폴백(상수 30/60/90) |

> `DayNightConfig`는 이미 `ColdStageWarning/Weak/Strong` public 프로퍼티 노출(`DayNightConfig.cs:66-68`). 추가 노출 작업 불필요. 서버 `ColdDamageWorker`와 **같은 SO 에셋**을 인스펙터에서 연결하면 단일 소스 성립.

## 5. 네트워크 동기화 항목

**신규 NetworkVariable / Rpc: 0개.** 기획의 "신규 시스템 0개"가 네트워크 차원에서도 성립.

| 채널 | 권위(authority) | 동기화 경로 |
|---|---|---|
| 추위 단계 | 서버(`ColdDamageWorker`/`DayNightWorker`가 `PlayerCharacter.Cold` write) → 클라 NetworkVariable 동기화 | 연출은 로컬 플레이어 Cold 구독, 화면연출만 |
| 낮밤 페이즈 | 서버(`DayNightWorker.m_Phase` NetworkVariable, Write=Server) | `OnPhaseChanged` static event 전원 발화 → 각 클라 톤 전환 |
| 도면 해금 | **로컬**(`RecipeUnlockRegistry`, 클라이언트별 세션 로컬) | 동기화 없음. 줍는 클라만 해금·연출(설계상 의도, §11-4) |

> 모든 연출은 read-only 소비자. 권위 위반 없음.

## 6. 상태 머신 (추위 단계만 해당)

추위 오버레이는 4상태 머신(`ColdStage`): `None → Warning → WeakDot → StrongDot`.

| 전이 | 조건 | 처리 |
|---|---|---|
| 상향(None→Warning→Weak→Strong) | Cold 임계 도달 | `CoFadeColdStage`로 누적 알파 페이드 |
| 하향(Strong→…→None) | Cold 감쇠(자연/모닥불) | 동일 페이드, 알파 감소 |
| StrongDot 진입 | Cold≥Strong | 펄스 시작(`UpdateColdStage3Pulse`) |
| **차단**: 완전 불투명 | — | **(v0.2 확정)** 단계3 목표·펄스 상한 = `COLD_OVERLAY_MAX_ALPHA(0.85)` 캡. 1.0 도달 불가 |

> **(v0.2)** 결정1로 단계3 목표 알파를 0.85로 코드 캡 — 어떤 텍스처가 와도 완전 차폐 불가, 중앙 시야 여백 보증. designer 텍스처 투명여백은 그 위 품질 향상(선택).

5. §9 수치 튜닝(펄스 1.4→1.5, 틴트 색/알파, 전환시간)

## 11. 엣지 케이스 (구현 차원)

1. **서버/클라 추위 임계 불일치**: **(v0.2 해소)** 결정2로 `DayNightConfig` 단일 소스화 — 동일 SO 참조라 영구 일치. 단, **인스펙터에서 `m_Config`에 서버와 같은 SO 에셋을 연결**해야 함(다른 SO 연결 시 어긋남 → §8 의존성·QA 체크 항목). `m_Config` null이면 폴백 상수 30/60/90.
2. **추위 단계3 완전 차폐**: **(v0.2 해소)** 결정1로 `COLD_OVERLAY_MAX_ALPHA(0.85)` 코드 캡. 1.0 도달 불가.
3. **디스커넥트 / 로컬 플레이어 교체**: `SetLocalPlayer`/`ClearLocalPlayer`가 구독 해제·재구독 처리. 재진입 시 `SetLocalPlayer`가 초기 Cold/HP/페이즈 즉시 반영(instant). 누락 없음.
4. **도면해금 — 줍는 클라만 해금**: 로컬 레지스트리라 동기화 없음. 다른 클라는 토스트·반짝 안 뜸(설계 의도). director 확인 필요 — "멀티에서 도면 발견은 개별인가 공유인가". 기획서가 개별 전제면 OK.
5. **CraftPopup 닫힌 상태 해금**: 반짝 연출 생략, 다음 Open 시 `RefreshLockState()`로 정상색만 표시. 기획 양보가능 항목으로 처리.
6. **동시 페이즈 전환 + 추위 단계 변경**: 두 코루틴 독립 채널(틴트 vs 추위 오버레이)이라 색이 합성될 뿐 충돌 없음(청 위에 남색). §5.4 색 대비 공존 성립.
7. **사망 암전 중 추위/페이즈 변경**: `PlayDeathFade`는 별도 death_overlay(최상위 레이어). 하위 연출이 깔려도 암전이 덮음. 중복 트리거 가드(`m_DeathTriggered`) 있음.
8. **다중 풀스크린 합성 가능성(질문2)**: 전부 UI `Image`/`RawImage` alpha 레이어. 별도 셰이더 불필요 — `NightDarknessOverlay` 셰이더 전제는 **불필요**(UI Image 합성으로 이미 동작). 렌더 구조상 동시표시 가능 확인.

## 12. 권고 (v0.2 — 1·2 director 결정 반영 완료)

1. ~~추위 임계 단일 소스화~~ → **결정2 확정·반영.** `DayNightConfig` 단일 소스. (§3.1, §4, §11-1)
2. ~~단계3 완전차폐 방지 방식~~ → **결정1 확정·반영.** 알파 캡 0.85 + designer 텍스처 여백 병행. (§3.1, §6, §11-2)
3. **§5.4 합성 우선순위 (잔여 권고)**: 코드 레이어 sibling order로 충족 가능하나, 기획서에 z-order를 1줄 못박아 주면 designer 프리팹 배치 기준이 명확해짐. 제안 순서(아래→위): **밤 톤 틴트 → 추위 오버레이 → 적색 HP 비네팅 → 안개 → 도면 토스트/사망 암전(최상위)**. director 확정만 주면 designer에게 그대로 전달.
4. **질문 3개 답 요약**:
   - Q1(훅 존재?): **3개 전부 이미 발화 중.** Cold=`SubscribeOnColdChanged`, 낮밤=`OnPhaseChanged`, 해금=`OnUnlockChanged`. 신규 훅 불필요.
   - Q2(다중 풀스크린 합성?): **가능. UI Image alpha 레이어로 이미 동작.** 청/적 채널 분리도 코드상 이미 분리됨(`m_ColdOverlay*` vs `m_HpVignette`). 별도 셰이더 전제 불필요.
   - Q3(칸별 잠금/해금 토글?): **수용 가능.** `CraftScrollCell.m_LockOverlay` + `RefreshLockState()` 보유, 목록 재배치 없음. "해금 시 그 칸만 반짝"만 메서드 1개 신규.
