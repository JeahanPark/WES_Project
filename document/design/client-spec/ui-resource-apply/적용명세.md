## 1. 코어텐션 오버레이 — 신규 GameObject 생성 (G-2/G-3/G-13/G-14/I-5)

코드는 단일 Worker `CoreTensionOverlayWorker` (`Assets/Scripts/Worker/CoreTensionOverlayWorker.cs`)가 5개 오버레이를 SerializeField로 구동. designer-b는 **풀스크린 오버레이 GameObject들을 만들고 sprite 연결 + 이 Worker의 슬롯에 드래그**.

### 1-A. 부착 위치
- `CoreTensionOverlayWorker`는 **인게임 HUD Canvas 하위**에 단일 GameObject로 부착(예: `InGameHUD/CoreTensionOverlay` 빈 오브젝트).
- 이 GameObject를 `InGameHUDWorker`의 신규 슬롯 **`m_CoreTensionOverlay`** 에 드래그 연결(필수 — 미연결 시 오버레이 전부 미동작, 단 에러는 없음).
- 오버레이 Image들의 **렌더 순서(형제 순서)** 권장 — 위에서 아래로 그려지는 순서:
  1. daynight_tint (최하단, 가장 약함)
  2. ambient_fog
  3. cold_overlay_1 → 2 → 3
  4. vignette_red
  5. death_overlay (최상단, 사망 시 전부 덮음)

### 1-B. 생성할 Image/RawImage GameObject 목록

전부 **풀스크린 stretch**(anchor min 0,0 / max 1,1 / offset 0), `raycastTarget = false`, 초기 alpha 처리는 아래 표대로.

| GameObject 이름 | 컴포넌트 | sprite/texture | Image Type | 초기 alpha | Worker 슬롯 |
|---|---|---|---|---|---|
| `ColdOverlay1` | Image | `cold_overlay_1` | Simple | **0** | `m_ColdOverlay1` |
| `ColdOverlay2` | Image | `cold_overlay_2` | Simple | **0** | `m_ColdOverlay2` |
| `ColdOverlay3` | Image | `cold_overlay_3` | Simple | **0** | `m_ColdOverlay3` |
| `HpVignette` | Image | `vignette_red` | Simple | **0** | `m_HpVignette` |
| `DeathOverlay` | Image | `death_overlay` | Simple | **0** | `m_DeathOverlay` |
| `DayNightTint` | Image | `daynight_gradient` | Simple | color.a = **0.18**(시작값, 코드가 덮어씀) | `m_DayNightTint` |
| `AmbientFog` | **RawImage** | `ambient_fog` | (RawImage, uvRect 스크롤) | texture 알파 그대로 | `m_AmbientFog` |

주의:
- alpha=0 처리 = **Image.color의 a를 0으로**. (GameObject active는 켜둔 채로 — 코드가 color.a만 조절)
- `AmbientFog`는 **RawImage** (Image 아님). texture import는 이미 Repeat. 코드가 uvRect.x를 가로 스크롤.
- `DayNightTint`의 sprite는 `daynight_gradient`지만 코드는 **color 틴트만** 바꾼다(단색 곱). sprite는 흰색/중립 단색이면 충분 — 굳이 그라데이션 LUT일 필요 없음. designer-b 판단: 단색 흰 1px sprite로 대체 가능(틴트색은 코드 제공). 그라데이션 LUT를 쓰면 색이 이중 적용되니 **흰색 단색 sprite 권장**.

### 1-C. 코드가 자동 구동하는 동작 (designer-b 참고용, 설정 불필요)
| 오버레이 | 트리거 | 동작 |
|---|---|---|
| cold_overlay_1/2/3 | 로컬 Cold ≥ 30/60/90 | 단계별 누적 페이드 인. 90단계는 펄스 |
| vignette_red | 로컬 HP ≤ 30% | 심박 펄스(alpha 0.35~1.0) |
| death_overlay | 로컬 HP ≤ 0 | 암전 페이드(1.8s). 부활 시 해제 |
| daynight_tint | DayNightWorker phase 전환 | 낮/황혼/밤/새벽 틴트 크로스페이드(1.5s) |
| ambient_fog | 상시 | 가로 UV 스크롤 |

## 2. 게이지 스킨 — PlayerStatusHUD (G-1)

| 슬롯 | sprite | Image Type | 비고 |
|---|---|---|---|
| HP StatBar 프레임 배경 | `panel_frame`/`slot_frame` 계열 9-slice | **Sliced** | HP·Cold 공통 프레임 |
| HP fill (`StatBar.m_BarImage`) | `gauge_fill_hp` | **Filled Horizontal** *또는* 현행 anchorMax 방식 유지 | ※ 아래 주의 |
| Cold fill | `gauge_fill_cold` | 동일 | |

**주의 — fill 구동 방식 충돌:** 현행 `StatBar.cs`는 fill을 **rectTransform.anchorMax.x** 로 조절(Image.fillAmount 미사용). designer-b가 `gauge_fill_*`를 **Filled Horizontal**로 세팅해도 `StatBar`가 fillAmount를 안 건드리므로 시각 게이지가 안 줄어든다.
→ **옵션 A(권장, 코드무변경):** fill sprite를 **Simple**로 두고 현행 anchorMax 마스킹 방식 그대로. 프레임만 9-slice 입힘.
→ 옵션 B: `StatBar`를 fillAmount 방식으로 교체(client 작업, 비용 S). director/team-lead 지시 시 진행.
→ **이번 패스 기본 = 옵션 A.** gauge_fill sprite는 Simple로 배치.

## 3. 버튼 프레임 + SpriteSwap 3-state (I-3 / L-3)

대상: `LoginPopup`, `LobbyPopup`, `LobbyRoomPopup`의 Button GameObject.

| 항목 | 설정 |
|---|---|
| Button `targetGraphic` Image sprite | `btn_frame_normal` (9-slice), Image Type **Sliced** |
| Transition | **Sprite Swap** |
| Highlighted/Pressed Sprite | `btn_frame_hover` |
| Disabled Sprite | `btn_frame_disabled` |

주의:
- WES 규칙상 **onClick은 코드(`AddListener`)로 바인딩**돼 있으므로 designer-b는 sprite/transition만 만진다. onClick 이벤트는 건드리지 말 것.
- 버튼 루트 Image를 Sliced로 바꾸면 코드 변경 불필요.

## 5. ResultPopup 성공/전멸 분기 (C-1/C-2)

`ResultPopup` 확인 필요 — 성공/전멸 배경 Image 슬롯 분기 존재 여부. (client 미확인, designer-b가 프리팹 열어 확인)
- 성공 배경 Image ← `bg_result_success` (Simple)
- 전멸 배경 Image ← `bg_result_defeat` (Simple)
- 슬롯이 한 개면 코드 분기 필요할 수 있음 → 그 경우 client에 회신.

## 6. designer-b → client 회신 요청 항목

작업 후 아래만 알려주면 client가 슬롯 연결 검증 보조:
1. `CoreTensionOverlay` GameObject 경로 + `InGameHUDWorker.m_CoreTensionOverlay` 연결 완료 여부
2. ResultPopup 성공/전멸 배경 슬롯이 분리돼 있는지 (단일이면 코드 분기 필요)
3. gauge fill을 옵션 A(Simple+anchorMax)로 갔는지 — 다르게 가야 할 이유 있으면 회신
