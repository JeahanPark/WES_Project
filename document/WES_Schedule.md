M3에서 한 UI 마감·QA 도구·코어텐션 연출은 그대로 출시본 기반으로 계승.

## 출시본 로드맵 (B안 — 9시간 6지역, 2026-06-08 ~)

> 의존성 순서로 5단계(R1→R5). **공수(주차)는 client 추정 영역** — director는 구조·순서·우선순위까지.
> 기반: M1~M3(현 콘텐츠·UI·QA도구·코어텐션 연출) 위에 구축. 기획 전체는 `design/game-design/출시_9시간_관통구조/` 외 출시 문서군.

### R1 — 스키마·시스템 토대 (최우선, 모든 것의 전제)
- **신규 CSV 스키마**(client): 몬스터 ATK · 드롭 확률% · 날씨 분포 · 이동비용 배수 · 도면 등장 확률 (현 스키마에 없음)
- **난이도 공식 코드화**(client): `D = TP/CAP` 계산·적용
- **4분기 미구현 시스템 구현**: ① 도구 등급 / ③ 덫·표지 / ④ 이동비용(체온 가속)
- **0순위 날씨 시스템 구현**(불완전정보 척추 — 없으면 4분기 의미 약화)
> R3 수치 입력이 이 스키마에 의존. 최우선.

### R2 — 6지역 공간
- **MapGenerator 6지역 종단 확장**(client): 원형 섬 → 선형 종단
- **지역 지형 에셋**(designer): 늪·설원·폐허 (Synty 차용 / 백로그)
- 지역 전환·관문·랜드마크 배치

### R3 — 콘텐츠 채우기
- **몬스터 12~16종**: 모델(designer)·행동 패턴(client)·수치(level-design)
- 자원 ~9 · 제작 ~25~32 · 도면 5~10
- **level-design 수치 CSV 입력** ([수치명세](design/level-design/출시_지역별_수치명세/수치명세.md) → CSV)

### R4 — 감각
- **사운드**(sound): 지역 BGM·SFX·앰비언스 + AudioManager 신설(client)
- **인게임 텍스트 반영**(story→client): 플레이버·내레이션·통지·엔딩
- 코어텐션 연출 날씨·이동 연동

### R5 — 밸런스·QA·출시
- **난이도 튜닝**: QA 모드 C 효과측정 = [수치명세 §9 합격 기준](design/level-design/출시_지역별_수치명세/수치명세.md)
- 멀티 동기화·디스커넥트 (mppm-qa 활용)
- Steam 연동·페이지·빌드

> **순서 철칙**: R1(토대) 없이 R3(콘텐츠) 못 채운다. R2(공간) 없이 R3 배치 못 한다. 병렬 가능: R4 사운드·텍스트는 R2/R3와 일부 병행.

## 코어 비전 갭 보완 로드맵 — 2026-06-03 (B안으로 격상)

> 게임의 "재미 핵심" = **자원투자 4분기 트레이드오프**. **B안 전환으로 2~4순위가 "출시 후"→"출시본 R1 포함"으로 격상됨.** 상세: [CORE_갭보완_로드맵.md](design/CORE_갭보완_로드맵.md)

| 우선순위 | 항목 | 분기 | 상태 |
|---|---|---|---|
| 0 | 시간/낮밤 시스템 | ④ + 불완전정보 | ✅ 구현 (페이즈 사이클+렌더) |
| 0 | Cold 실질화 | ③ | ✅ 구현·검증 (QA PASS 20/0, 2026-06-03) |
| 1 | 도면 해금 | ② | ✅ 구현·검증 (QA PASS 16/0 + 심화 3건, 2026-06-03) |
| 2 | 덫(능동 방어) | ③ | 미착수 → **출시본 R1** (설계 ✅ 건물효과) |
| 3 | 도구 등급 | ① | 미착수 → **출시본 R1** (설계 ✅ 아이템·제작) |
| 4 | 이동 비용 | ④ | 미착수 → **출시본 R1** (설계 ✅ 이동비용) |
| 0+ | 날씨(불완전정보) | 0순위 | 미착수 → **출시본 R1** (설계 ✅ 날씨_시스템) |

> **2026-06-03**: 출시 전(M4) 최소 슬라이스 = 0순위(Cold/시간)+1순위(도면) **완료**. 2~4순위는 출시 후. 잔여 M4 = mppm-qa V2~V7 멀티 검증.

> **전제**: 0순위(시간낮밤·Cold)가 없으면 1~4를 추가해도 분기 결정의 긴장감이 안 생긴다(코어 비전 우선순위 0).
