# mppm-qa 멀티QA 도구 (2026-06-03)

MPPM(Multiplayer Play Mode) 기반 멀티플레이 동기화 QA를 **per-run 사용자 개입 0**으로 자동화하는 도구 세트를 Ingest. 메인 에디터=Host, 가상 플레이어(클론)=Client로 로컬 직결 후 양측 상태를 파일 공유로 수집하고, 클론까지 QA 트리거를 보낼 수 있다.

## 구성

| 구성요소 | 파일 | 역할 |
|----------|------|------|
| 부트스트랩 | `Assets/Scripts/Worker/MppmBootstrapWorker.cs` | Ingame 진입 시 `CurrentPlayer.IsMainEditor`로 역할 분기 → Relay 없이 127.0.0.1 직결 자동 Host/Join. `BootstrapState` 상태머신. `GameNetworkManager.IsNetworkConfigured` 대기로 레이스 방지. 클론은 타임아웃까지 재시도 |
| 프로브 | `Assets/Scripts/Component/MultiplayerQaProbe.cs` | 양측 스냅샷을 시스템 temp 공유폴더(`wes-mppm-qa`)에 JSON 기록(`probe_host.json`/`probe_clone.json`, temp→rename 원자 쓰기). 클론 한정 `cmd_clone.json` 역방향 커맨드 폴링 |
| 수집 | McpBridge `mppm_collect` | 양측 파일을 1콜로 `{ host, clone }` 반환 |
| 트리거 | `TestManager.TestMultiSpawnForSync` + `TestMpV2~V7` | 멀티 동기화 검증 시나리오 트리거 |

## 흐름

```
Play 1회
  └ 메인 에디터  → MppmBootstrapWorker: Host (StartLocalHostAsync)
  └ 가상 플레이어 → MppmBootstrapWorker: Client (StartLocalClientAsync, 재시도)
        ↓ 양측 MultiplayerQaProbe가 0.5s 주기로 스냅샷 기록
  QA → mppm_collect (host/clone 스냅샷 비교)
  QA → 호스트 트리거: u_play_invoke(TestMp*)
  QA → 클론 트리거: cmd_clone.json seq++ (TestMp* 화이트리스트) → 프로브가 폴링 호출
```

- **개입 0(per-run)**: 가상 플레이어 최초 1회 활성만 수동(디스크 영속). 이후 Play 1회로 Host/Join·스냅샷·정리 전 자동.

## 검증 상태

- **V1(부트스트랩/접속/스폰 동기화)**: 검증됨 — 양측 직결 + 스폰 오브젝트 스냅샷 수집 동작 확인.
- **V2~V7**: 트리거·수집 데이터블록(스냅샷 포맷·클론 커맨드 채널) 준비 완료. 시나리오별 PASS 판정은 후속 사이클.

## 수정한 버그 2건

1. **클라 스폰 오브젝트 미복원** — 동적 생성 NetworkManager의 NetworkConfig에 NetworkPrefabsList가 비어 호스트/클라 프리팹 해시 불일치. `GameNetworkManager.RegisterNetworkPrefabs`로 런타임에 `Resources/DefaultNetworkPrefabs` 등록해 일치(직전 커밋 `048ade4` 라인). 설정 완료를 `IsNetworkConfigured`로 노출해 부트스트랩 레이스 차단.
2. **InGameController NGO 직렬화 충돌** — 자체 `m_NetworkObject` 필드가 베이스 `NetworkBehaviour.NetworkObject` 프로퍼티와 이름 충돌(NGO 직렬화 규칙 위반). 필드 제거 후 베이스 `NetworkObject`/`IsSpawned` 직접 사용.
