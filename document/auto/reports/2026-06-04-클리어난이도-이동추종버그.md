# 게임 클리어 난이도 적용 + 이동 경로추종 근본 버그 수정

## 의뢰
클리어 조건이 테스트용으로 스폰 옆 4u만 움직이면 바로 깨짐 → 목표지점·몬스터를 두고 **플레이해야 클리어**되는 테스트 난이도(거리+몬스터 돌파)로 변경.

## 한 일
- **EscapePoint 이동**: `(-3,1,-38)` → `(0,1.5,35)`. 스폰(z-42)에서 종단 **77u**, Beach·Forest 몬스터 Area를 관통해야 도달. (클리어 로직 자체는 정상이었고 씬 배치 문제였음)
- 몬스터 스폰 시스템(`InGameAreaWorker` + `WorldAreaInfo` CSV, 3영역)은 이미 경로 전체에 깔려 있어 추가 작업 없음.

## 검증 중 발견한 근본 버그 — MoveTo 경로추종 무효화
EscapePoint를 멀리 두니 자동 종단(`TestFullPlayClear`)이 매번 **Warp 폴백**으로만 클리어. 체계적 디버깅으로 추적:

| 단계 | 측정 결과 |
|------|-----------|
| NavAgent 상태 | enabled/isOnNavMesh/updatePosition 전부 정상 |
| 평지 직접 이동 | `MoveWithDirection`·`NavAgent.Move` 둘 다 정상(5u/s) |
| `MoveTo` 경로추종 | **True 반환하지만 0.4u 만에 추종 종료** |

**원인**: [PlayerCharacter.cs:385](../../WES/Assets/Scripts/WorldBaseObject/Player/PlayerCharacter.cs#L385) `HandleInput()`이 매 프레임 `MoveWithDirection(moveInput)` 호출 → 입력이 0이어도 [CharacterBase.cs](../../WES/Assets/Scripts/WorldBaseObject/CharacterBase.cs)의 `if(m_IsFollowingPath) StopMove()`로 **경로추종이 첫 프레임에 즉시 취소**. → 플레이어 거의 안 움직임 → FullPlay 직선폴백 → 나무(z-13)에 막힘 → Warp.

## 수정 (CharacterBase.cs 3건)
1. **`MoveWithDirection`** — 실제 입력(`sqrMagnitude > 0`)일 때만 경로추종 취소. 0 입력에는 추종 유지. *(근본)*
2. **`MoveTo`** — 목적지를 `NavMesh.SamplePosition`으로 표면에 투영(트리거 콜라이더 중심 y=1.5가 PathPartial 유발하던 것 차단) + `PathComplete`만 추종 시작.
3. **`UpdateMovement` 경계 게이트** — `SamplePosition` 반경 0.5→3.0(언덕 표면 높이차 흡수) + 샘플점과의 **수평** 거리 0.6 이내만 이동 허용(바다 외곽 차단 유지). 언덕 경사 통과 가능.
