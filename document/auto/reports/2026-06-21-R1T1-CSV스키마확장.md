# R1-T1 — CSV 스키마 확장 + 파싱

> 로드맵 [출시본 R1](../../design/work-plans/R1_스키마_시스템_토대/계획.md) T1.
> DoD: 신규 컬럼이 스키마에 추가되고 InfoLoader가 파싱(컴파일 0). 수치 튜닝은 R5.

## 한 줄
9시간 출시본 시스템(T2~T6)이 의존하는 **데이터 스키마 토대**를 추가하고 런타임 파싱까지 검증.

## 변경 내역

### 신규 컬럼 (기존 CSV)
| CSV | 추가 컬럼 | 용도 |
|---|---|---|
| MonsterInfo | `ATK.INT` | 몬스터 공격력 (피해공식 ATK−DEF) |
| DropTableItemInfo | `Chance.FLOAT` | 드롭 확률(0~1). 기존 행 1.0(R5 튜닝) |
| WorldAreaInfo | `MoveCostMultiplier.FLOAT` | 지역별 이동 체온 소모 배수(T5) |
| BlueprintInfo | `SpawnAreaId.INT`, `SpawnChance.FLOAT` | 도면 등장 지역·확률(도면해금) |

### 신규 CSV (테이블)
| CSV | 컬럼 | 용도 |
|---|---|---|
| WeatherInfo | Id, WeatherType(ENUM), Name, ThreatMul, VisionMul, ColdDrainMul, MoveCostMul | 날씨 5종별 계수(weatherMul 실체, T6) |
| WorldAreaWeatherInfo | Id, AreaId, WeatherType(ENUM), Chance | 지역별 날씨 확률 분포(T6) |

### 코드
- `GameEnum.cs` — `WeatherType { Clear, Cloudy, Rain, Fog, Snowstorm }` 추가.
- `InfoConvertEditor.cs` — 생성 클래스를 **`partial class`** 로 변경(재생성 시 커스텀 멤버 보존).
- `Info/InfoCustom.cs`(신규) — 자동생성 클래스 커스텀 멤버 보존용 partial. `ItemInfo.IsBuilding` 이전.
- 자동 재생성: ConvertInfo/*.cs (WeatherInfo·WorldAreaWeatherInfo 신규, 전 클래스 partial화), InfoLoader.cs(신규 2종 LoadAllInfo 등록).

## 회귀 1건 (해소)
InfoConvert 1차 실행이 ConvertInfo/ItemInfo.cs의 수동 프로퍼티 `IsBuilding`를 덮어써 컴파일 5에러.
→ 근본수정: converter가 `partial class` 생성 + 커스텀을 InfoCustom.cs로 분리 → **재발 방지**(T2~T6에서 InfoConvert 재실행 안전).

## 검증
- 컴파일 0 에러.
- 플레이 진입 → 인게임 로드(`LoadAllInfo`) → **파싱 예외 0**(enum `Enum.Parse`·float `float.Parse` 전부 성공). 플레이어 스폰·게임 시작 정상.
