### 시나리오 3 — 야간 몬스터 스폰/디스폰: 코드 구조 PASS / 데이터 레이어 SKIP

`NightMonsterComponent` 코드 구조 정상 (Night 페이즈 진입 시 스폰, Dawn/Day 진입 시 디스폰).

**SKIP 사유**: `WorldAreaMonsterInfo.csv`의 `PhaseCondition` 필드에 데이터 미입력으로 실측 스폰 0건. 데이터 레이어 설정은 별도 작업으로 분리.

**디렉터 결정 사항**: 기획서 5-3에 명시된 "기존 몬스터 야간 WANDER_RADIUS 확장"은 이번 슬라이스 범위에서 제외. 신규 기능 추가에 해당하며, 기존 몬스터 낮 시간대 밸런스 기준 보호를 위해 별도 슬라이스에서 수치와 함께 정밀 설계 후 구현 예정 (Backlog 등록).

### 시나리오 4 — 야간 시야 반경: 코드 구조 PASS / 시각 수동 재검증 필요

`NightVisionComponent` 수치가 기획서와 일치.

| 항목 | 기획서 | 코드 상수 |
|------|--------|-----------|
| 밤 시야 반경 | 40% | VisionRadiusNight = 0.4f |
| 횃불 원형 컷아웃 반경 | +30% (3m) | TorchLightRadius = 3f |
| 모닥불 원형 컷아웃 반경 | +40% (5m) | CampfireLightRadius = 5f |

| 밤 어둠 오버레이 알파 | 최대 | m_NightOverlayAlphaMax = 0.85f |

**시각 미반영 원인 (구조적 한계)**: 에디터 직접 실행 환경에서 `DayNightWorker.OnNetworkSpawn`이 호출되지 않아 `m_Phase.OnValueChanged` 등록 누락 → `OnPhaseChanged` 이벤트 미발행 → `NightVisionComponent`에 페이즈 변화 전달 불가. `NightVisionComponent` 자체 로직(어둠 오버레이·LightCircle 컷아웃)은 코드상 정상.

- **증상**: TestDayNightCycle 코루틴 완료 후 `m_EditorPhaseOverride = 0`(Day) 잔류 → NGO spawn 상태에서 독립 `ForcePhase(Night)` 호출 시 `CurrentPhase`가 Day 반환 → `NightVisionComponent`/`DayNightRenderWorker`가 이벤트는 수신해도 재조회 시 Day가 반환되어 시각 효과 미적용.
- **수정 (client)**: `DayNightWorker.ForcePhase` 의 IsSpawned 분기에 `m_EditorPhaseOverride = -1` 초기화 추가.
- **상태**: 코드 수정 완료, **시각 캡처 1회 미완** (작업 중단 시점). 정상 진입 경로 + ForcePhase(Night) 5초 대기 후 `u_screenshot` 1세트만 추가 캡처하면 시나리오 4 시각 확인 완료.

## 라운드4 결과 (셰이더 기반 컷아웃 재작성)

### 원인 — 라운드3 모닥불 컷아웃 FAIL
UI Alpha-Blend로는 어둠 오버레이 위에 원형 구멍을 뚫을 수 없는 **구조적 결함**. Inspector 슬롯은 모두 정상 연결돼 있었음.
