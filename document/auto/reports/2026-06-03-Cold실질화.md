# Cold 실질화 (추위 위협) 구현·검증

코어 비전 ③ 분기(기술 사용=지금의 안전을 산다)를 활성화하는 0순위 작업. 옛 "체온(↑=안전)" 잔존 모델을 **추위 누적(↑=위험, 0=정상)** 모델로 전환하고, 밤 Cold 누적 + 단계별 HP 틱 위협을 신규 구현.

## 디렉터 확정 (§13, 2026-06-03)

| # | 결정 | 근거 |
|---|------|------|
| 1 | 켜진 모닥불 = **누적 스킵만**(적극 감소 X), 자연 감쇠가 줄임 | 명세 §11 모델, 최소 슬라이스 |
| 2 | 밤 감쇠 멀티 **1.0 고정**(옛 2.0 제거) | 밤은 추위가 쌓여야 압박 |
| 3 | 연료 시스템 제외, `SetLit`은 Test/QA 전용 | 갭로드맵 최소 슬라이스 |

## 구현 (client)

- **신규**: [[ColdDamageWorker]](밤 누적 +10/s, 단계 30/60/90, WeakDot -2/3s·StrongDot -5/2s, HP1 보호, 켜진불 보호), `ColdRealizationSetup.cs`(씬 와이어링 메뉴), `ColdStage` enum(GameEnum).
- **수정**: [[DayNightConfig]](Cold Damage 파라미터 + 밤멀티 1.0) · [[WorldBuildingObject]](옛 `SetCold(+2)` 제거) · [[CharacterBase]](`ApplyEnvironmentDamageServer` — DEF/크리 무시, HP1 보호) · [[PlayerCharacter]](`TakeEnvironmentDamage`) · [[InGameController]](m_ColdDamageWorker 슬롯).

## 발견·수정 버그 (QA 회귀가 포착)

1. **밤 감쇠 멀티 2.0 잔존** — 코드 디폴트는 1.0이나 `DayNightConfig.asset` 직렬화 인스턴스가 2.0 → 에셋 1.0으로 수정(결정2).
2. **`BuildingInfo not found:1`(모닥불 스폰 불가)** — 데이터 누락 아님. MPPM Host 부트스트랩이 `LoadAllInfo` 미호출 → 서버 `BuildingInfoList` 빈 상태. [[InfoManager]]에 `LoadAllInfoOnce()` 멱등 래퍼 추가, [[MppmBootstrapWorker]]·[[InGameController]]가 호출하도록 일원화(경로 무관 1회 로드).

## 검증 (QA, mode: function)

`TestManager.TestColdRealization` 10분기 → **최종 PASS 20 / FAIL 0**. 시1~7(누적/낮중단/임계/약틱/강틱/HP1보호/None무틱) + 시8·9(켜진불 보호·소등 재개=결정1) + 시10(밤 ×1=결정2) + 멱등가드 회귀 전부 PASS.
