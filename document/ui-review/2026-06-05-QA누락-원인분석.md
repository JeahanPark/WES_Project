# QA 누락 원인 분석 — UI 리소스 패스 (2026-06-05)

> LoginPopup(Intro 화면)에서 **로고 투명 체커보드 박스 · ✦ 워터마크 잔존 · 버튼 normal-state 미스타일**이 QA "모드B 통과" 후에도 살아 나갔다. 자산 수정은 쉽지만, **검수 게이트가 뚫린 구조**가 진짜 문제다.

## 새어나간 결함 3종
| 결함 | 성격 |
|---|---|
| 로고에 회색 체커보드 박스 | logo_main.png의 "투명 배경"이 진짜 alpha가 아니라 **체커 무늬가 픽셀로 구워짐** → 게임 합성 시 격자 박스로 렌더 |
| 로고 우하단 ✦ Gemini 워터마크 잔존 | 로고는 크롭 파이프라인을 안 탐(원본 그대로) |
| 버튼 normal-state 단색(나무프레임 안 보임) | SpriteSwap의 hover/disabled만 와이어링되고 base Image.sprite=btn_frame_idle 미설정 추정 → 평상시 평면 |

## 왜 QA가 통과시켰나 — 근본 원인 3가지

### 1. 검사 항목(B1~B10)이 전부 "기하/직렬화"뿐, "콘텐츠 진위"가 없다
B1~B10 = 정렬·크기·겹침·폰트·극단값·상태색·전환·해상도·비율·반복균등. **전부 RectTransform 수치 / 컴포넌트 직렬화 검사.**
- 체커보드 로고: rect 크기·위치·aspect 전부 정상 → B2/B9 통과. **"이미지 픽셀이 맞는가"를 보는 항목이 없음.**
- 버튼 SpriteSwap: transition=2 + hover/disabled sprite 직렬화 정상 → 통과. **"normal-state가 실제로 프레임으로 보이는가"를 보는 항목이 없음.**
- 워터마크: 어떤 수치 검사에도 안 걸림.
→ **결함이 "수치 시그니처"가 없으면 B1~B10을 그냥 통과한다.**

### 2. "수치 백업 필수 / 스크린샷 단독 검수 금지" 규칙이 오히려 시각결함을 밀어냄
qa.md B-3: *"스크린샷 단독 검수 금지. 비율·크기·정렬 의심은 반드시 수치 백업 필수."*
- 의도(레이아웃 깨짐 오탐 방지)는 옳지만, **백업할 수치가 없는 순수 시각결함**(구운 체커·워터마크·평면버튼)은 "근거 부족"으로 플래그가 안 됨.
- 즉 프로세스가 검수자를 **수치로 환원되는 결함**으로만 유도 → 순수 픽셀/렌더 결함은 사각.

### 3. 라이브 렌더가 아니라 정적 표면을 봤다 (포트7777 플레이모드 차단이 가중)
- 모드B 시각검수 시점에 **포트 7777 누수로 호스트 기동 실패 → 플레이모드에서 LoginPopup이 실제로 안 떴을 가능성**. 그 경우 판정 근거가 프리팹 직렬화 / 씬뷰 / PNG Read로 후퇴.
- 결정적: **투명 PNG는 어떤 뷰어/Read에서도 체커로 보이는 게 정상** → 파일을 아무리 들여다봐도 "구운 체커" 결함을 절대 못 가린다. **오직 게임에 합성된 라이브 렌더에서만** 드러난다.
- 사용자가 실제로 본 결함 화면은 Unity 재시작·포트 해제 **이후** 라이브로 뜬 LoginPopup. QA의 모드B는 그 이전 정적 상태였다.

## 한 줄 결론
> **QA가 "구조(직렬화·수치·와이어링)"는 검증했지만 "사람 눈에 보이는 렌더 결과"는 검증하지 못했다.** B1~B10이 기하 전용이고, 수치백업 규칙이 시각결함을 밀어냈고, 플레이모드 차단으로 라이브 렌더 대신 정적 표면을 봤다 — 세 개가 겹쳐 체커·워터마크·평면버튼이 동시에 통과했다.

## 재발 방지 — qa.md 모드B 보강안 (제안)
1. **C-시리즈 "콘텐츠/렌더 진위" 검사 신설** (수치 아닌 시각 판정, 라이브 렌더 필수):
   - C1 투명/알파: 게임 합성 상태에서 투명 격자(체커)·의도치 않은 단색 박스 노출 0 (PNG Read의 체커는 판정 근거 금지).
   - C2 잔존 마크: AI생성 자산에 워터마크(✦)·서명·잡티 0.
   - C3 적용 완료성: 위젯이 placeholder 단색/평면이 아니라 의도 자산으로 보이는가 (버튼 normal-state 포함).
   - C4 화면=의도 일치: 플레이어가 실제 보는 화면이 명세 의도와 일치하는가.
2. **라이브 렌더 강제**: 시각검수는 **플레이모드로 각 화면 실제 진입 후 Game View 캡처**가 원칙. 정적 프리팹/PNG Read만으로 "통과" 판정 금지.
3. **플레이모드 진입 불가 시 "통과" 금지**: 호스트/씬 진입이 막히면 해당 화면은 **"런타임 미검증(BLOCKED)"**으로 명시 보고하고, 환경 해소 후 재검증 — 정적 표면으로 갈음 금지.
4. **AI생성 자산 특화 체크리스트**: 투명 격자·워터마크·미적용 평면은 AI-UI 패스의 단골 결함 → 별도 1차 스윕.

## 심층 갱신 (2026-06-05, 2차 — 버튼 텍스트 대비결함 발견 후)

수정(체커·워터마크·버튼프레임) 후에도 사용자가 **버튼 텍스트가 나무 프레임 색과 묻혀 안 보임**을 또 발견. 이걸로 더 깊은 공통 원인이 드러났다.

### 공통 분모 (모든 결함의 단일 뿌리)
> **생산자(designer)도 검증자(qa)도 "플레이어가 실제로 보는 합성된 렌더 화면"을 끝까지 안 본다. 모든 작업·검증이 고립된 아티팩트 단위(프리팹 직렬화·개별 sprite·rect 수치·PNG 파일).** 합성에서만 드러나는 결함은 사람(사용자)이 볼 때까지 전부 생존한다.

| 결함 | 고립 단위로 본 것 | 놓친 합성 결과 |
|---|---|---|
| 체커보드 | "투명 배경"이라는 말 | alpha 진위(모서리 불투명=체커 baked) |
| 워터마크 | 크롭 파이프라인 존재 | 로고만 파이프라인 안 거침(예외 누락) |
| 평면버튼 | 스프라이트 박힘 여부 | normal 상태 렌더 |
| **텍스트 대비** | 글자색 정상·프레임색 정상 | **둘이 겹친 합성 결과** |

### 핵심: "창발적(emergent)" 결함
텍스트 대비는 **글자색 자체도 정상, 프레임색 자체도 정상, 겹쳐야만 문제**. 어떤 단일 아티팩트 검사로도 원리적으로 못 잡는다. 오직 합성 렌더를 눈으로 봐야 잡힌다. → "각 자산이 명세대로인가"만 검사하는 파이프라인은 창발적 결함에 구조적으로 눈멀어 있다.

### 구조적 예방 — 3층 게이트 (인코딩 완료)
| 층 | 어디 | 무엇 | 상태 |
|---|---|---|---|
| 생산 | `designer.md` "합성 렌더 자가검수" | 자산 적용 후 합성 라이브 렌더 캡처·화면전체 자가검수(투명·워터마크·대비·적용·톤) 후에만 보고 | ✅ 추가 |
| 검증 | `qa.md` C-시리즈 | C5 대비/가독성(창발적) 추가 + 라이브렌더 필수 | ✅ 추가 |
| 소스 | AI 파이프라인 | ①다운로드 후 alpha 히스토그램 검증 ②워터마크 제거 무조건 경유(예외 0) | 원칙 문서화(코드 가드는 후속) |

핵심 한 줄: **"합성 렌더를 본다"를 생산 단계부터 의무화 → QA까지 안 가고 designer가 먼저 잡는다.**
