# 외부 자산 백로그

> **관리자**: 디자이너 에이전트
> **사용자 의뢰**: 본 폴더의 `<주제>.md` 파일들이 사용자가 외부에서 마련해야 할 자산 목록

## 구조

각 주제(시스템·콘텐츠)마다 별도 .md 파일:
- `시간_낮밤_시스템.md`
- `cold_실질화.md`
- ...

## 파일 표준 양식

```markdown
# <주제> 외부 자산 백로그

## <자산 항목 1> (우선순위: 상/중/낮)

### 현재 상태
- 임시 처리: <어떤 placeholder 사용 중인지>

### 정식 자산 요구사항
- 종류: 메쉬 / 텍스처 / sprite / 애니메이션 / 사운드
- 크기·해상도·톤
- 참조 이미지·게임 (있다면)
- 기타 메모

### 대체 출처 후보
- (구매·무료·AI 생성 등)

### 도착 시 자동화 (디자이너 처리)
- 사용자가 `Assets/GameResource/...` 에 드롭
- 디자이너가 import + 머티리얼 정렬 + 프리팹 교체

### 상태
- [ ] 외부 자산 도착 대기
- [ ] (도착 시) ✅ 처리 완료
```

## 자동 집계

이 폴더의 모든 `.md` 파일은 `WES/Tools/Generate Resource Usage Report` 메뉴 실행 시 자동으로 `document/RESOURCE_USAGE_REPORT.md`의 "외부 자산 백로그" 섹션에 통합된다.
